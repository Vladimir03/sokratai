| Страница | Содержимое | Проблемы |

## Изменения

### 1. TutorLayout — оставить как есть
Навигация уже работает корректно:
- Дашборд → `/tutor/dashboard`
- Ученики → `/tutor/students`  
- Оплаты → `/tutor/payments`

### 2. TutorDashboard — полностью переписать

**Было:** Заглушка с "Скоро появится"

**Станет:** Информативная стартовая страница с:

**A) Приветственная секция:**
- "Добро пожаловать, [Имя]!" с датой
- Краткое описание возможностей

**B) Сводные карточки (3 штуки в ряд на десктопе):**
```text
+---------------+  +---------------+  +---------------+
| 👥 Ученики    |  | ⏳ Ожидается  |  | 🔴 Просрочено |
|     5         |  |   15 000 ₽    |  |    3 000 ₽    |
| активных      |  |  2 записи     |  |   1 запись    |
+---------------+  +---------------+  +---------------+
```

**C) Быстрые действия:**
- Кнопка "Добавить ученика" (открывает модалку с invite-ссылкой)
- Кнопка "Добавить оплату" (переход на `/tutor/payments`)
- Кнопка "Все ученики" (переход на `/tutor/students`)

**D) Секция "Требуют внимания" (если есть проблемы):**
- Список учеников с просроченной оплатой
- Список учеников без активности > 7 дней

### 3. Использовать TutorLayout везде единообразно
Сейчас TutorDashboard НЕ использует TutorLayout (имеет свой header) — исправить для консистентности.

## Детали реализации

### TutorDashboard.tsx

**Импорты:**
- Добавить `useTutorStudents`, `useTutorPayments` для получения данных
- Использовать `TutorLayout` вместо кастомного хедера
- Иконки: `Users`, `CreditCard`, `AlertCircle`, `UserPlus`, `Plus`, `ChevronRight`

**Структура компонента:**
```typescript
function TutorDashboardContent() {
  const { tutor } = useTutor();
  const { students, loading: studentsLoading } = useTutorStudents();
  const { payments, loading: paymentsLoading } = useTutorPayments();
  
  // Вычислить статистику
  const stats = useMemo(() => {
    const activeStudents = students.filter(s => s.status === 'active').length;
    const pendingPayments = payments.filter(p => p.status !== 'paid');
    const overduePayments = payments.filter(p => 
      p.status !== 'paid' && p.due_date && new Date(p.due_date) < new Date()
    );
    return { activeStudents, pendingPayments, overduePayments };
  }, [students, payments]);
  
  // Ученики без активности > 7 дней
  const inactiveStudents = useMemo(() => {
    const sevenDaysAgo = new Date(Date.now() - 7 * 24 * 60 * 60 * 1000);
    return students.filter(s => 
      s.status === 'active' && 
      (!s.last_activity_at || new Date(s.last_activity_at) < sevenDaysAgo)
    );
  }, [students]);

  return (
    <TutorLayout>
      {/* Приветствие */}
      {/* Сводные карточки */}
      {/* Быстрые действия */}
      {/* Требуют внимания (если есть) */}
    </TutorLayout>
  );
}
```

**Сводные карточки:**
- Кликабельные — при клике переходят на соответствующую страницу
- Анимация hover для интерактивности

**Требуют внимания:**
- Показывать только если есть просроченные оплаты или неактивные ученики
- Компактный список с возможностью перейти к ученику

### UX-улучшения

1. **Консистентность:** Все страницы используют TutorLayout
2. **Навигация понятна:** Активная вкладка подсвечена
3. **Быстрые действия:** Основные действия доступны сразу с дашборда
4. **Информативность:** Сразу видно состояние дел без перехода на другие страницы
5. **Внимание к проблемам:** Просроченные оплаты и неактивные ученики подсвечены

### Мобильная версия

- Быстрые действия — полноширинные кнопки
- Нижняя навигация уже есть в TutorLayout
