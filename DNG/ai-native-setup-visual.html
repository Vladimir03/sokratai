Vibe Coding vs AI-Native

Vibe Coding
Идея в голове → AI пишет код → ? а зачем → AI фиксит баги → ? кому это нужно → Рефактор → 0 пользователей
AI-Native
AI Research → Сегменты → Граф работ → Wedge decision → PRD + UX + UI → Код по docs → Pilot
Vibe Coding

«AI, сделай мне приложение»

  • ! Код есть, продукта нет. Нет ответа на «кому и зачем»
  • ! AI фантазирует — нет методологии, нет фреймворка
  • ! Каждый промпт — с нуля. Потеря контекста между сессиями
  • ! Дрифт: «Помощник» превращается в generic чат
  • ! Рефакторы без конца — нет canonical source of truth
  • ! Скорость кодинга, но не скорость product-market fit
vs
AI-Native Development

AI-система от research до кода

  • ✓ Каждая строка кода привязана к Job: исследование → код
  • ✓ AJTBD-методология закодирована в промпты — AI не фантазирует
  • ✓ Результат AI = вход следующего AI. Ноль потери контекста
  • ✓ Guardrails в CLAUDE.md — агент физически не может сломать фокус
  • ✓ 20 canonical docs = source of truth. Docs рядом с кодом в Git
  • ✓ Скорость кодинга + скорость нахождения PMF
Парадокс: чем больше ограничений для AI — тем лучше результат. CLAUDE.md, canonical docs, guardrails — не бюрократия. Это операционная система, которая превращает одного человека в полноценную команду.
