# fipi-import — пилот «ФИПИ ОГЭ физика → База задач Сократа»

Конвейер: страница открытого банка ФИПИ → 10 структурированных задач с ответами/решениями →
«Черновики для Сократа» Егора (модератор физики) → его ревью → публикация в Каталог.

## Артефакты

- `out/page1.json` — 10 задач стр. 1 (текст+LaTeX, ответ, решение-эталон, классификация, КЭС).
- `out/images/p1/*` — оригинальные картинки заданий, скачанные с oge.fipi.ru (имя = номер задания ФИПИ).
- `out/refs.json` — соответствие картинка → `storage://kb-attachments/...` (появляется после загрузки).
- `out/page1-insert.sql` — SQL вставки задач (исполняется через Lovable MCP `query_database`, НЕ миграцией).

## Загрузка картинок (шаг владельца)

1. Открой **PowerShell** в папке проекта:
   - в Проводнике зайди в `C:\Users\kamch\sokratai`, в адресной строке набери `powershell` и Enter;
   - либо в уже открытом PowerShell выполни `cd C:\Users\kamch\sokratai`.
2. Запусти одну команду:

   ```powershell
   node scripts/fipi-import/upload-images.mjs
   ```

3. Скрипт спросит **email** и **пароль** от Сократа прямо в терминале (пароль скрыт звёздочками,
   никуда не сохраняется). Введи и нажми Enter.
4. Когда увидишь `Готово: 6/6 загружено` — пришли мне содержимое появившегося файла
   `scripts/fipi-import/out/refs.json`, и я подставлю ссылки в задачи.

Скрипт входит под твоим аккаунтом (политика бакета требует, чтобы путь начинался с uid
загружающего) и кладёт файлы в `kb-attachments/{uid}/fipi-oge/p1/`. Читать эти файлы может
любой авторизованный пользователь — задачи Егора ссылаются на них без проблем.

**Если входишь через Яндекс / VK / Google** (пароля от Сократа нет) — signInWithPassword
не сработает; напиши мне, загрузим картинки другим способом.

## Почему вставка задач — не миграция

Это контент-пилот: задачи должны легко правиться/удаляться при ревью Егора, а миграции в этом
репо нельзя изменять и откатывать (AGENTS.md → Database rules). На 151 страницу банка миграции
не масштабируются. SQL-файл хранится здесь для аудита; исполнение — через Lovable MCP.
`id` задач детерминированные (uuidv5 от GUID задания ФИПИ) + `ON CONFLICT DO NOTHING` —
повторный запуск безопасен, откат = `DELETE ... WHERE id IN (...)`.
