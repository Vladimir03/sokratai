### Рекомендуемый порядок реализации (9-day Pilot → Paid sprint, 6–15 апреля)

Контекст: 3 репетитора в бесплатном пилоте, платный запуск 15 апреля (2000 ₽/мес), цель — 30+ репетиторов. Live-blocker с realtime в thread viewer уже закрыт, остаётся добить качество hint-ов и срезать LLM-косты до монетизации.

**День 1–2 (6–7 апр) — Pilot blockers:**
1. **Е10 (P0)** — hint quality + запрет «перечитай условие» — **✅ DONE** (`FORBIDDEN_HINT_PHRASES` + validator + retry-once + fallback + telemetry + smoke canary)
2. **Е9 (P0)** — realtime thread viewer через Supabase Realtime на `homework_tutor_thread_messages` — **✅ DONE**

**День 3–5 (8–10 апр) — Усиление core value:**
3. **Е11 (P1)** — LLM cost optimization: bootstrap caching, history window trimming, system prompt compression, cheaper model для bootstrap/hint. Параллельно с Е10
4. **Ж2 (P1)** — статистика по ДЗ в формате ЕГЭ (баллы, номера КИМ, динамика ученика)
5. **Е3 (P1)** — уведомление о регистрации ученика

**День 6–8 (11–13 апр) — Pre-launch polish:**
6. **Ж1 (P1)** — банк задач ЕГЭ, ручное наполнение KB модераторами (Егор+Женя)
7. **Е12 (P2)** — persona presets: 3 dropdown (глубина / стиль / предпочтительный метод), без free-form prompt
8. **Е8 (P2)** — условие задачи в GuidedThreadViewer (быстрый win)
9. **Е5 (P2)** — смена папки в KB (XS, при оказии)

**После 15 апр — paid pilot:**
10. **Е6 (P2)** — AI промпты физики ЕГЭ (research-intensive, отдельный спринт)
11. **В1 (P2, Phase 1b)** — formula round как homework-linked block (кинематика + tutor assignment)
12. **Е7 (P3)** — парсер скриншотов (post-pilot)

### Паттерны между источниками

**Егор и Женя оба хотят:**
- Больше задач ЕГЭ в системе (Е1 + Ж1) — KB-наполнение критично для удержания
- Лучшую видимость результатов (Е8 + Ж2 + Е9) — репетитор хочет видеть прогресс в реальном времени, не копаться в тредах
- Формат ЕГЭ как первоклассный (Е4 + Ж2) — баллы, номера КИМ, не просто «процент»

**Только Женя:**
- Динамика ученика по времени (Ж2) — сравнение с собой, не с другими
- Целые варианты ЕГЭ как ДЗ (Ж1) — 26 задач, pre-exam тренировка

**Только Егор:**
- UX-polish guided chat (Е2, Е8, Е9) — он уже активно пользуется, видит friction points
- Модерация KB (Е5, Е7) — он наполняет базу, сталкивается с ограничениями редактора
- Качество AI-подсказок (Е10) — ученики жалуются на шаблонные «перечитай условие»

**Live signals Егора (2026-04-06, verbatim):**
- *«Хочется смотреть как ученик решает ДЗ в реальном времени, как в ВК — сейчас приходится перезагружать страницу»* → Е9
- *«Ученики ненавидят "Попробуй перечитать условие задачи и выделить ключевые данные" — AI шлёт это слишком часто. Нужны подсказки по сути задачи»* → Е10

**Статус live-проверки Е9:**
- Realtime в guided thread viewer подтверждён вручную: ученик пишет с телефона, репетитор видит новые сообщения без `F5`

**Статус live-проверки Е10:**
- В `guided_ai.ts` внедрён flow `generate -> validate -> 1 retry -> fallback` без циклов и без шаблонных hint-ов
- Literal фраза *«Попробуй перечитать условие задачи и выделить ключевые данные»* теперь режется validator'ом
- Добавлен deterministic fallback: если gateway/model вернули мусор дважды, ученик всё равно получает task-specific hint
- Добавлена telemetry без PII: `hint_rejected` и `hint_fallback_used`
- Добавлен smoke canary: regression-проверка на live pilot phrase, чтобы баг не вернулся незаметно

**Новые внутренние тикеты (Vladimir, 2026-04-06):**
- **Е11** — LLM cost reduction обязателен перед монетизацией (30–50% экономии через bootstrap cache + history trim + cheap model для hint/bootstrap)
