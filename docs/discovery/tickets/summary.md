### Рекомендуемый порядок реализации

1. **Е2 (P0)** — баг/UX-блокер guided chat, ломает решение ДЗ — **DONE**
2. **Е1 + Е4 (P1, связаны)** — формат проверки из KB + «развернутое решение» — одна фича
3. **Е3 (P1)** — уведомление о регистрации ученика
4. **Ж2 (P1)** — статистика по ДЗ в формате ЕГЭ — данные уже в БД, оба репетитора хотят
5. **Ж1 (P1)** — банк задач ЕГЭ — начать с ручного наполнения KB модераторами, не автопарсинг
6. **Е8 (P2)** — условие задачи в viewer (быстрый win, effort S)
7. **Е5 (P2)** — смена папки в KB (XS, при оказии)
8. **Е6 (P2)** — AI промпты физики (research-intensive, отдельный спринт)
9. **В1 (P2, Phase 1)** — только как homework-linked formula round: кинематика + 3 task types + tutor assignment/progress
10. **Е7 (P3)** — парсер скриншотов (post-pilot)

### Паттерны между источниками

**Егор и Женя оба хотят:**
- Больше задач ЕГЭ в системе (Е1 + Ж1) — KB-наполнение критично для удержания
- Лучшую видимость результатов (Е8 + Ж2) — репетитор хочет видеть прогресс, не копаться в тредах
- Формат ЕГЭ как первоклассный (Е4 + Ж2) — баллы, номера КИМ, не просто «процент»

**Только Женя:**
- Динамика ученика по времени (Ж2) — сравнение с собой, не с другими
- Целые варианты ЕГЭ как ДЗ (Ж1) — 26 задач, pre-exam тренировка

**Только Егор:**
- UX-polish guided chat (Е2, Е8) — он уже активно пользуется, видит friction points
- Модерация KB (Е5, Е7) — он наполняет базу, сталкивается с ограничениями редактора

**Новая внутренняя идея (Vladimir):**
- Формульный тренажёр можно встроить в pilot wedge, если продавать и строить его не как отдельную игру, а как назначаемый репетитором round внутри ДЗ по теме урока
