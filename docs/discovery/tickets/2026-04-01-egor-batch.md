## Ticket #2: Guided chat — не перебрасывать на другую задачу

### Источник
Егор: «Нужно строго оставаться в рамках диалога по одной задаче. Если я не решил третью, но дал неверный ответ на пятую или спросил подсказку по пятой — он меня кидает на диалог по третьей задаче автоматически»

### Job Mapping
- **Core Job**: S1 — Получить подсказку при застревании
- **Sub-job**: S1-2 — Выбрать правильный подход к решению
- Связь: если ученик сам выбрал задачу 5 — он хочет работать над задачей 5. Перебрасывание на задачу 3 = потеря контекста, frustration, churn

### Пилотный фильтр
- Помогает Егору и следующим 5 репетиторам? **Да** — их ученики жалуются
- Усиливает wedge? **Косвенно** — если guided mode раздражает, ученики бросают ДЗ → репетитор не видит ценности
- Блокирует пилот без этого? **Да, UX-блокер** — ломает основной flow решения ДЗ

### Решение
- [x] **GO** → P0, делать первым

### Effort: M
Контекст: свободный порядок задач уже реализован (Sprint 2026-03-19). Фронт передаёт `task_order` в check/hint. Но где-то backend или frontend всё ещё перебрасывает. Нужна диагностика: frontend `syncThreadFromResponse()` или backend `performTaskAdvance()`?

### ✅ Реализовано — 2026-04-01

**Диагноз:** проблема была на фронтенде — `syncThreadFromResponse()` безусловно перезаписывал `currentTaskOrder` из БД, игнорируя выбор ученика.

**Что сделано:**
- `syncThreadDataOnly()` — новый helper: обновляет messages/task_states/status **без** изменения навигации
- `handleCheckAnswer` и `handleHint` теперь используют `syncThreadDataOnly` вместо `syncThreadFromResponse`
- При `CORRECT` — 1200ms celebration анимация (зелёное кольцо + scale + bounce ✓ на TaskStepper), затем auto-advance на следующую active задачу
- `celebrationTimerRef` + cleanup useEffect — предотвращает memory leak при unmount
- Race guard в `handleTaskClick`: блокирует навигацию при `celebratingTaskOrder !== null`

**Файлы:** `GuidedHomeworkWorkspace.tsx`, `TaskStepper.tsx`
**Коммит:** `3a6e328`
**Спека:** `docs/delivery/features/guided-chat/task-lock-spec.md`
**AC:** AC-1…AC-7 PASS (build PASS, smoke PASS)

### Источник
Егор: «Для второй части ЕГЭ — формат проверки "развернутое решение", чтобы ученики показывали ход решения. Без решения бот не принимает задачу. В банке: КИМы 1-20 краткий ответ, 21-26 развернутое решение. Репетитор может поменять формат на финальном этапе»

- Усиливает wedge? **Да** — решает проблему «ученик угадал или списал ответ»
- Блокирует пилот? **Нет** — но сильно усиливает value prop

### Решение
- [x] **GO** → P1
- Связан с Ticket #1 (формат проверки из KB). Реализовать вместе.
- Scope: `check_format: 'short_answer' | 'detailed_solution'` в `homework_tutor_tasks` + KB `kb_tasks`
- AI enforcement: если `check_format === 'detailed_solution'` и ответ < N символов / нет шагов → AI отклоняет
