## #1: Банк задач ЕГЭ по физике в Сократе

### Discovery
- Хочет использовать задачи из neofamily.ru и РЕШУ ЕГЭ **внутри Сократа**
- Нужны **все 26 задач** КИМа ЕГЭ по физике (тестовая + развёрнутая часть)
- Нужны **десятки задач по каждому номеру КИМ** — разнообразие, не одни и те же
- Источник не принципиален (neofamily, РЕШУ ЕГЭ, ФИПИ) — важно разнообразие
- Сейчас берёт конкретные задачи, но через ~2 месяца до ЕГЭ репетиторы начинают **прорешивать целые варианты** (формат тренировки к экзамену)
- Значит нужна и поддержка **полного варианта ЕГЭ** как одного ДЗ (26 задач)

### Job Mapping
- Core Job: R4 — Сохранение контроля и качества при масштабировании
- Sub-job: R4-1 (быстро собрать ДЗ из проверенных задач), R4-3 (обновлять базу задач)
- Дополнительно: S1 — Найти и начать использовать инструмент

### Пилотный фильтр
- Помогает ли пилоту? Да — все репетиторы физики ЕГЭ используют банки задач
- Усиливает ли wedge? Да — если задачи уже в Сократе, сборка ДЗ моментальная
- Блокирует ли пилот? Частично — без банка Женя продолжит собирать ДЗ в neofamily

## #2: Статистика по ДЗ в формате ЕГЭ

### Discovery
- Результаты нужны **в формате ЕГЭ по задачам** — не просто «процент», а баллы по номерам КИМ
- По **каждому ученику отдельно** — репетитор работает индивидуально, получает деньги за результат конкретного ученика
- По **каждой задаче по каждому ученику** — краткий результат в карточке ДЗ
- Важнее **динамика ученика vs самого себя** (прогресс), а не сравнение учеников между собой
- Можно запросить у Жени скриншот статистики neofamily как референс

### Job Mapping
- Core Job: R1 — Проверка ДЗ и оценка прогресса ученика
- Sub-job: R1-2 (понять, где ученик ошибается), R1-4 (видеть прогресс по времени)
- Дополнительно: R2 — Обратная связь ученику (статистика как инструмент мотивации)

### Пилотный фильтр
- Помогает ли пилоту? Да — каждый репетитор хочет видеть прогресс учеников
- Усиливает ли wedge? Да — быстрый обзор результатов = экономия 10-15 мин на review
- Блокирует ли пилот? Нет, но сильно усиливает value prop для удержания

### Решение
- [x] **GO, P1** — начать с ручного наполнения KB модераторами, не автопарсинг
- [ ] WAIT
- [ ] KILL
