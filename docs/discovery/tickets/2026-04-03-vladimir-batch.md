## Ticket #1: Тренажёр формул механики для ЕГЭ

### Discovery
- На входе уже есть структурированная база из 35 формул по механике в 5 разделах: кинематика (12), динамика (6), законы сохранения (9), статика (5), колебания и волны (3).
- GDD описывает full vision тренажёра: раунды по 3-5 минут, 10 заданий, 3 жизни, XP, streak, карта прогресса, карточка формулы, adaptive/spaced repetition.
- После уточнения позиции идея рассматривается НЕ как standalone student-продукт, а как часть текущего tutor workflow: репетитор назначает тренажёр по теме урока, ученик проходит его внутри ДЗ, репетитор видит прогресс по формулам.
- В GDD уже зафиксирован узкий MVP, который подходит как Phase 1: только кинематика (12 формул), 3 типа заданий, 10-question round, базовый прогресс и объяснения после ответа.
- Ключевая метрика v1: ежедневные раунды ученика, а не depth самой геймификации.

### Job Mapping
- **Core Job**: R4 — Сохранение контроля и качества при масштабировании
- **Sub-job**: R4-1 — Быстро собрать качественное ДЗ по теме урока
- **Вторичные Jobs**: S1-2 — Выбрать правильный подход к решению; S1-4 — Понять ошибку и не повторить её в следующей задаче; S2-4 — Увидеть свой прогресс и почувствовать рост
- **Косвенный эффект**: P1-3 / P3-1 — между уроками появляется видимая ежедневная практика, а у родителя — ощущение, что система работает не только на уроке
- **Связь с Job**: тренажёр становится новым типом practice artifact внутри ДЗ. Репетитор после урока может за минуты назначить "раунд по формулам темы", а не только набор задач. Это усиливает workflow "урок → ДЗ → ежедневная практика".

### Пилотный фильтр
- Помогает текущему пилоту? **Да** — если назначается репетитором как часть ДЗ/практики по теме урока
- Усиливает wedge (ДЗ за 5-10 мин)? **Да** — как быстро добавляемый модуль повторения формул к теме урока
- Блокирует пилот без этого? **Нет**

### Решение
- [x] **GO** → создать PRD для **Phase 1**
- Scope guard: не строить отдельный student-продукт в первой фазе. Первая версия = tutor-assigned micro-drill внутри homework workflow.
- **Рекомендация по входу v1**: первый вход делать **внутри ДЗ**, а не отдельным экраном "Тренажёр".
- Почему внутри ДЗ лучше для v1:
  - это напрямую бьётся с wedge "собрать ДЗ и практику по теме урока";
  - репетитору проще назначать и отслеживать completion в существующем homework flow;
  - ученик получает единый контур "задачи + тренажёр" без второго параллельного продукта;
  - проще считать главную метрику v1: ежедневные раунды по назначенным темам.
- Отдельный экран "Тренажёр" имеет смысл как **Phase 2+**:
  - для самостоятельных повторов;
  - для streak/habit loop;
  - для свободного доступа к уже назначенным или пройденным темам.

### Effort
- **Phase 1: L** — кинематика, 3 типа заданий, 10-question round, базовый student progress, tutor assignment, tutor visibility
- **Full vision: XL** — все 12 типов заданий, вся механика, отдельная карта прогресса, streak, SR, boss-раунды, standalone trainer surfaces

### Открытые вопросы
- Как именно репетитор назначает раунд: выбрать тему вручную, выбрать готовый шаблон "Кинематика / Уровень 1", или прикрепить round к конкретному ДЗ?
- Ответ: выбрать тему вручную по Каталогу Сократа.
- Что репетитор должен видеть в v1: completion %, слабые формулы, ошибки по слоям, или только факт прохождения + score?
- Ответ: факт прохождения + completion %, слабые формулы.
- Нужны ли повторные попытки ученика по одному назначенному раунду или v1 достаточно кнопки "повторить ошибки"?
- Ответ: достаточно кнопки "повторить ошибки"
- Где живёт student completion UX внутри ДЗ: отдельный блок задания, отдельная вкладка, или inline-card в homework detail?
- Ответ: тут дай ответ исходя из Jobs ученика (S).
- Нужно ли в v1 связывать результаты тренажёра с guided chat / AI-подсказками, или это Phase 2?
- Ответ: это точно Phase 2.
