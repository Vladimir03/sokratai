### БЛОК A: ОРГАНИЗАЦИЯ И АДМИНИСТРИРОВАНИЕ

#### JOB A1: Управлять расписанием учеников

```
КОГДА:
  Контекст: 15-25 учеников с разным расписанием, переносы каждую неделю
  Триггер: ученик/родитель пишет "можно перенести на другой день?"
  Эмоции в точке А: раздражение ("опять жонглировать слотами")

ХОЧУ: Легко планировать и перезаписывать занятия без долгих переписок

Критерии успеха:
- Вижу свободные слоты одним взглядом
- Ученик может сам выбрать время
- Конфликты расписания автоматически блокируются
- Переносы занимают <1 минуты

Проблемы:
- "Надо по 150 раз одного подвинуть, другого засунуть" (Вячеслав)
- Calendly недоступен для оплаты из РФ
- Google Calendar — нет самозаписи

ЧТОБЫ: Не тратить 3-5 часов в неделю на переписки о времени
Сила проблемы: 7/10
```

**Работы ещё ниже уровнем:**
- A1.1: Настроить свою доступность (какие дни/часы работаю)
- A1.2: Отправить ссылку ученику для самозаписи
- A1.3: Подтвердить/отклонить запись
- A1.4: Перенести занятие на другой слот
- A1.5: Отменить занятие с уведомлением

ЧТОБЫ: Сократить no-show и повысить выполнение ДЗ
Сила проблемы: 6/10

**Работы ещё ниже уровнем:**

## ДЕТАЛИЗАЦИЯ MVP-РЕШЕНИЙ

### MVP #1: Профиль ученика (C1)

**Цель:** Репетитор видит всех учеников в одном месте

**Функционал:**
```
Список учеников:
- Имя
- Класс (9/10/11)
- Предмет
- Цель (целевой балл)
- Телефон/Telegram родителя
- Статус подписки (активен/неактивен)
- Дата последнего занятия

Карточка ученика:
- Все поля выше
- Заметки репетитора (текст)
- Результаты пробников (дата + балл)
- Ссылка на историю AI-диалогов
```

**Схема данных:**
```sql
CREATE TABLE students (
  id UUID PRIMARY KEY,
  tutor_id UUID REFERENCES tutors(id),
  name TEXT NOT NULL,
  grade INT CHECK (grade IN (9, 10, 11)),
  subject TEXT,
  target_score INT,
  parent_contact TEXT,
  status ENUM('active', 'paused', 'finished'),
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE mock_exams (
  id UUID PRIMARY KEY,
  student_id UUID REFERENCES students(id),
  date DATE,
  score INT,
  max_score INT,
  notes TEXT
);
```

**UI:** Таблица с фильтрами + модалка редактирования

**Время:** 1-2 дня

---

### MVP #2: Учёт оплат (A3)

**Цель:** Репетитор видит кто оплатил, может напомнить

**Функционал:**
```
Таблица оплат:
- Ученик (select из списка)
- Сумма
- Период (месяц/количество уроков)
- Статус: ожидает | оплачено | просрочено
- Дата оплаты

Действия:
- Добавить запись
- Отметить как оплаченное
- Кнопка "Напомнить" → отправляет сообщение в Telegram

Дашборд:
- Ожидается к получению: X руб
- Получено в этом месяце: Y руб
- Просрочено: Z руб
```

**Схема данных:**
```sql
CREATE TABLE payments (
  id UUID PRIMARY KEY,
  tutor_id UUID,
  student_id UUID REFERENCES students(id),
  amount INT NOT NULL,
  period TEXT, -- "Январь 2025" или "8 уроков"
  status ENUM('pending', 'paid', 'overdue'),
  due_date DATE,
  paid_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);
```

**Автоматизация:**
- Cron job: если due_date < сегодня и status = pending → status = overdue
- Кнопка "Напомнить" → Telegram Bot API → сообщение родителю

**Время:** 2-3 дня

### MVP #3: Подключение ученика к AI (C2.1)

**Цель:** Репетитор добавляет ученика, ученик получает доступ к AI-помощнику

**Функционал:**
```
В кабинете репетитора:
- Кнопка "Добавить ученика"
- Генерирует уникальную ссылку-приглашение
- Ученик переходит по ссылке → попадает в Telegram бот
- Бот привязывает ученика к репетитору

В Telegram боте:
- При старте по реферальной ссылке: "Вас добавил репетитор [имя]"
- Сохраняем связь tutor_id ↔ student_telegram_id
```

**Схема данных:**
```sql
ALTER TABLE students ADD COLUMN telegram_id BIGINT;
ALTER TABLE students ADD COLUMN invite_code TEXT UNIQUE;
ALTER TABLE students ADD COLUMN invite_used_at TIMESTAMP;
```

**Время:** 1 день (интеграция с существующим ботом)

### MVP #4: Напоминания (A2)

**Цель:** Автоматические напоминания о занятиях

**Функционал:**
```
Настройки репетитора:
- За сколько часов напоминать (по умолчанию: 24 и 1)
- Текст напоминания (шаблон с переменными)

Логика:
- Cron job каждые 15 минут
- Проверяет занятия из календаря
- Если до занятия осталось X часов → отправляет в Telegram

Шаблон сообщения:
"Привет! Напоминаю о занятии завтра в 16:00. 
Не забудь сделать домашку! 📚"
```

**Время:** 1-2 дня
