# PRD: Подсказки при застревании — Guided Homework для ученика

**Core Job:** S1 — Получить подсказку при застревании
**Продукт:** СократAI
**Версия:** v1.0
**Дата:** 2026-03-27
**Автор:** Product Team
**Статус:** Draft

---

## 1. Problem Statement

Ученик 10–11 класса решает ДЗ по физике вечером, застревает на задаче и не может продвинуться. Репетитор недоступен (поздно, между уроками). Существующие альтернативы — ChatGPT (даёт готовый ответ, убивает навык), РешуЕГЭ (только конечный ответ без разбора), одноклассники (готовые ответы) — не помогают ученику **научиться**, а лишь дают списать.

Проблема критична: сила 8–9/10 по всем четырём sub-jobs. Если ученик бросает задачу при застревании, он теряет 2–3 часа потенциальной практики между уроками. Для родителя, платящего 48 000 ₽/мес за репетитора, «пустота между уроками» — главный аргумент против продолжения оплаты (P3-1). Для репетитора нерешённое ДЗ = потеря контекста на следующем уроке (R2-4).

**Точка совпадения:** S1 = R2-2 (репетитор настроил AI-ассистента) = P1-1 (родитель видит, что ребёнок работает вечером). Одна фича закрывает потребности всех трёх сторон B2B2C-связки.

## 2. Аудит текущей реализации

### 2.1. Что уже реализовано (as-is)

Guided homework chat в Сократе уже существует и содержит базовые механизмы для всех четырёх sub-jobs:

| Sub-Job | Текущий механизм | Оценка |
|---------|-------------------|--------|
| S1-1: Понять условие | Условие задачи с MathText (LaTeX) + zoomable изображение + AI bootstrap-сообщение | 5/10 |
| S1-2: Выбрать подход | Поле «Обсуди с AI» (Discussion field) — свободный чат с AI | 4/10 |
| S1-3: Проверить шаг | Поле «Ответ» + AI check (CORRECT/ON_TRACK/INCORRECT) | 5/10 |
| S1-4: Понять ошибку | AI feedback с классификацией ошибки (calculation/concept) | 4/10 |

### 2.2. UX-аудит по принципам документа 16

**Принцип 3 (Recognition over recall) — ЗОНА РОСТА (P1).**
Когда ученик застрял, он видит два пустых текстовых поля («Ответ...» и «Обсуди с AI...»). Нет подсказок о том, *что* можно спросить у AI. В идеале нужны **example prompts** / **quick action chips** привязанные к текущей задаче — но это существенная UI-доработка, не приоритет для первой итерации. Вместо этого в v1 компенсируем через **улучшение system prompt**: AI сам предлагает ученику конкретные следующие шаги в тексте ответа.

**Принцип 4 (Workspace, не чат) — ОК.**
Discussion field свёрнут по умолчанию. Два раздельных поля (Answer + Discussion) — хорошее разделение. Текущая реализация адекватна для пилота.

**Принцип 5 (AI output → действие) — РЕШАЕМО ЧЕРЕЗ PROMPT.**
AI даёт текстовый фидбек, но не предлагает следующий шаг. После «INCORRECT» ученик должен сам понять, что делать. Вместо добавления UI-кнопок (дорого, риск перегрузки интерфейса) — **правим system prompt**: AI обязан в конце каждого фидбека явно предложить 1–2 конкретных следующих действия текстом. Например: «Попробуй пересчитать с учётом трения» или «Напиши мне свой промежуточный результат — проверю».

**Принцип 7 (Progressive disclosure) — ХОРОШО.**
Два поля (Answer + Discussion) — правильное разделение. Discussion свёрнут по умолчанию. Условие collapsible.

**Принцип 12 (Надёжность > эффектность) — ХОРОШО.**
AI не даёт готовых ответов. Сократовский метод через наводящие вопросы. Score-система мотивирует решать самостоятельно.

**Принцип 16 (Физика ≠ текст) — ХОРОШО.**
LaTeX-рендеринг через KaTeX в условиях, сообщениях, фидбеке. Zoomable изображения задач.

### 2.3. UX-аудит по паттернам документа 17

**Паттерн 2.1 (Один экран = одна работа) — OK.**
Экран guided homework имеет одну работу: решить текущую задачу.

**Паттерн 2.3 (Каждый AI-результат → действие) — ДОПУСТИМО НА ПИЛОТЕ.**
AI-ответы в guided chat не имеют UI action layer (кнопок после сообщения). На данном этапе это приемлемо — action layer будет реализован через **текст самого AI-ответа**: system prompt обязывает AI явно предлагать следующее действие. UI-кнопки — кандидат на P1 после обратной связи с пилота.

**Паттерн 4.1 (Jobs-first) — НЕ ПРИМЕНИМ.**
Это экран для ученика, не для репетитора. Паттерн Помощника не применяется.

## 3. Goals

| # | Goal | Метрика | Target |
|---|------|---------|--------|
| G1 | Ученик продвигается при застревании без готового ответа | % задач, где ученик перешёл от ON_TRACK/INCORRECT → CORRECT в рамках одного thread | ≥60% (vs текущие ~40%) |
| G2 | Снижение abandonment при застревании | % задач с status=active, где последнее сообщение старше 10 мин и задача не завершена | ≤25% (vs текущие ~45%) |
| G3 | Ученик использует AI как наставника, не как решатель | Среднее кол-во сообщений в discussion mode до CORRECT | 2–4 (сейчас ~1 или 0 — чаще бросают) |

| G5 | Репетитор видит, что ученик работал (P1 value) | % заданий, где thread содержит ≥3 сообщений ученика | ≥70% |

## 5. User Stories

### Persona: Школьник (11 класс, готовится к ЕГЭ по физике, 70+ баллов цель)

**S1-1: Понять условие**

- **US-1.1**: Как ученик, открывший сложную задачу с длинным условием, я хочу увидеть от AI структурированный разбор (Дано/Найти/Ситуация) в bootstrap-сообщении — чтобы понять задачу за 1 минуту, а не бросить её.
- **US-1.2**: Как ученик, видящий задачу с графиком/рисунком, я хочу получить от AI комментарий к изображению (что изображено, какие оси, что означают ключевые точки) — чтобы не гадать, что нарисовано.
- **US-1.3**: Как ученик, у которого задача с несколькими телами/системами, я хочу видеть AI-подсказку с выделением ключевых объектов и их взаимодействий — чтобы не запутаться в условии.

**S1-2: Выбрать подход**

- **US-2.1**: Как ученик, который понял условие, но не знает, какой закон применить, я хочу попросить подсказку и получить наводящий вопрос (не прямой ответ), направляющий к правильному закону — чтобы самостоятельно выбрать формулу. При повторных подсказках AI становится конкретнее.
- **US-2.2**: Как ученик, попробовавший один подход и зашедший в тупик, я хочу написать «Пробовал через энергию, не получается» и получить от AI оценку: «Энергия — правильный подход, но подумай, все ли силы учёл» или «Попробуй через импульс, подумай почему» — чтобы не тратить время на тупиковый путь.

**S1-3: Проверить промежуточный шаг**

- **US-3.1**: Как ученик, получивший промежуточный результат (например, v = 15 м/с на шаге 3 из 7), я хочу отправить этот шаг на проверку и получить ответ за 15 секунд: «Верно, продолжай» или «Проверь размерность — у тебя м/с², а должно быть м/с» — чтобы не решать 20 минут на основе ошибки на шаге 2.
- **US-3.2**: Как ученик, который не уверен в правильности подстановки чисел, я хочу сфотографировать свою запись и спросить AI «Правильно ли я подставил?» — чтобы поймать арифметическую ошибку до финального ответа.

**S1-4: Понять ошибку**

- **US-4.1**: Как ученик, получивший verdict INCORRECT, я хочу сразу увидеть понятное объяснение ошибки с аналогией из жизни — чтобы понять физический смысл ошибки, а не просто «трение не учтено». AI должен предложить конкретный следующий шаг.
- **US-4.2**: Как ученик, исправивший ошибку после объяснения AI, я хочу увидеть резюме: «Ты ошибся потому что X. Запомни: в задачах на наклонную плоскость всегда проверяй, учтено ли трение» — чтобы не повторить ошибку в следующей задаче.
- **US-4.3**: Как ученик, который хочет закрепить понимание, я хочу получить предложение «Попробуй похожую задачу» с одним кликом — чтобы проверить, что действительно понял.

## 6. Requirements

### 6.1. Must-Have (P0) — System Prompt Improvements

Все P0-требования реализуются через изменение system prompts в `guided_ai.ts`. Без изменений UI-компонентов. Минимальный risk, максимальный impact.

#### R1. Structured Condition Parsing в Bootstrap и Hints

**Описание:** System prompt для bootstrap и hint-ответов обновляется: AI обязан при первом сообщении (bootstrap) или по запросу ученика разбирать условие задачи в формате **Дано / Найти / Ситуация**.

**Изменения в prompt:**
- Bootstrap prompt: «Если задача имеет длинное условие (>2 предложений), структурируй его: выдели Дано (числовые данные + единицы), Найти (что нужно определить), Ситуация (кратко что происходит). Используй LaTeX для формул.»
- Hint prompt (когда ученик просит помощь с условием): «Разбери условие в формате Дано/Найти/Ситуация. Если есть изображение задачи — прокомментируй ключевые элементы.»

**Acceptance Criteria:**
- [ ] Bootstrap для задач с task_text > 100 символов включает структуру Дано/Найти/Ситуация
- [ ] Формулы в Дано рендерятся через LaTeX (проверить в MathText)
- [ ] Для коротких задач (<100 символов) bootstrap остаётся кратким («Попробуй решить!»)
- [ ] Если у задачи есть изображение — prompt включает инструкцию прокомментировать его

#### R2. Progressive Hint Ladder

**Описание:** System prompt для `generateHint` обновляется: AI обязан учитывать количество уже данных подсказок по текущей задаче и увеличивать конкретность.

**Уровни:**

| Кол-во предыдущих hints | Стиль AI | Пример |
|--------------------------|----------|--------|
| 0 (первая подсказка) | Общий наводящий вопрос | «Подумай, какая величина здесь сохраняется?» |
| 1 | Более конкретный вопрос | «Обрати внимание: система замкнутая? Есть ли внешние силы?» |
| 2+ | Прямое указание направления | «Попробуй применить закон сохранения энергии. Какие виды энергии участвуют?» |

**Acceptance Criteria:**
- [ ] Prompt для hint включает: «Это подсказка №{N} для этой задачи. При N=1 — задай общий вопрос. При N=2 — будь конкретнее. При N≥3 — укажи конкретный закон/подход, но НЕ решай задачу.»
- [ ] Backend передаёт в prompt кол-во предыдущих hint-сообщений по задаче (count из thread_messages where message_kind='hint')
- [ ] AI по-прежнему НЕ даёт готовый ответ даже на 3+ подсказке
- [ ] Score impact остаётся -0.1 за каждую подсказку (без изменений)

#### R3. AI Actively Suggests Next Action

**Описание:** System prompt для ВСЕХ режимов (check answer, hint, discussion) обновляется: AI обязан в конце каждого ответа явно предложить 1–2 конкретных действия, которые ученик может сделать дальше.

**Маппинг по типу ответа:**

| Тип ответа AI | Что AI предлагает в конце |
|---------------|--------------------------|
| INCORRECT verdict | «Попробуй пересчитать с учётом [конкретика]. Или напиши мне — помогу разобраться.» |
| ON_TRACK verdict | «Продолжай решать. Если застрянешь на следующем шаге — напиши промежуточный результат, проверю.» |
| CORRECT verdict | «Отлично! Переходи к следующей задаче.» (+ резюме если была ошибка до этого, см. R5) |
| Hint | «Попробуй с этим подходом. Если получишь промежуточный результат — напиши, проверю.» |
| Discussion | Зависит от контекста — AI предлагает релевантный следующий шаг |
| Bootstrap | «Попробуй решить самостоятельно. Если не понял условие — напиши, разберём вместе.» |

**Acceptance Criteria:**
- [ ] Каждый system prompt (evaluateStudentAnswer, generateHint, streamChat) содержит инструкцию: «В конце ответа ОБЯЗАТЕЛЬНО предложи ученику 1–2 конкретных следующих шага. Формулируй как действие: "Попробуй...", "Напиши мне...", "Проверь..."»
- [ ] AI не предлагает абстрактные действия («подумай ещё») — только конкретные
- [ ] Предложение привязано к содержанию ответа (не шаблонное)
- [ ] Проверить на 5–10 тестовых задачах: в каждом ответе AI есть actionable suggestion

#### R4. Error Explanation with Analogy

**Описание:** System prompt для `evaluateStudentAnswer` (verdict=INCORRECT) обновляется: AI обязан объяснять ошибку понятно, с аналогией из повседневной жизни, а не формальным языком учебника.

**Изменения в prompt:**
- «Если ученик допустил концептуальную ошибку (не вычислительную), объясни ПОЧЕМУ это неправильно: (1) назови физический принцип, (2) дай аналогию из повседневной жизни, (3) предложи как исправить. Пиши на языке 16-летнего, не цитируй учебник.»
- «Для вычислительных ошибок: укажи конкретный шаг с ошибкой и что должно быть вместо этого.»

**Acceptance Criteria:**
- [ ] При concept error: ответ AI содержит аналогию (не формальное определение)
- [ ] При calculation error: ответ AI указывает конкретный шаг (не «проверь вычисления»)
- [ ] Тон: дружелюбный, без осуждения. «Это частая ошибка! Давай разберёмся...»
- [ ] LaTeX-формулы в объяснении рендерятся корректно
- [ ] Проверить на 5 задачах с concept errors: аналогия релевантна

#### R5. Error Summary After Correction

**Описание:** System prompt для `evaluateStudentAnswer` (verdict=CORRECT) обновляется: если предыдущая попытка по этой задаче была INCORRECT, AI добавляет короткое резюме-правило.

**Изменения в prompt:**
- «Если ученик сначала ответил неверно, а теперь верно — добавь в конце 1–2 предложения: "📌 Запомни: в задачах на [тема], [конкретное правило]". Привяжи к типу допущенной ошибки.»

**Acceptance Criteria:**
- [ ] Резюме генерируется только если history содержит INCORRECT verdict для этой задачи
- [ ] Резюме — 1–2 предложения, конкретное правило (не «будь внимательнее»)
- [ ] Если ученик решил с первой попытки — резюме НЕ генерируется (только «Отлично!»)
- [ ] Backend передаёт в prompt историю попыток (кол-во INCORRECT + тип ошибки)

#### R6. Step Check через Discussion Mode

**Описание:** System prompt для discussion (streamChat) обновляется: AI умеет распознавать промежуточный шаг ученика и давать быструю проверку без выдачи финального ответа.

**Изменения в prompt:**
- «Если ученик отправляет промежуточный результат (формулу, числовое значение, подстановку), проверь его: (1) Верно — скажи "✅ Верно, продолжай!". (2) Неверно — скажи "⚠️ Проверь [что именно]" + короткая подсказка. НЕ показывай правильный промежуточный результат, НЕ решай задачу дальше.»

**Acceptance Criteria:**
- [ ] AI корректно отличает промежуточный шаг от финального ответа (промежуточный = формула, подстановка, «получил v = 15 м/с»; финальный = конечный числовой ответ)
- [ ] При верном шаге: короткий ответ «Верно, продолжай»
- [ ] При неверном шаге: указание что проверить + подсказка (не ответ)
- [ ] Score НЕ снижается при проверке шага через discussion (это free-text, не hint request)
- [ ] AI не путает step check с finаl answer check (ученик пишет в Discussion, не в Answer)

### 6.2. Nice-to-Have (P1) — UI Improvements (после пилота)

#### R7. Quick Action Chips в guided chat

**Описание:** Под полем Discussion отображаются 2–4 контекстных кнопки-чипа, привязанных к текущему состоянию задачи. Реализация требует изменений в `GuidedChatInput.tsx` + backend.

**Состояния и чипы:**

| Состояние задачи | Чипы |
|------------------|------|
| Задача только открыта, 0 сообщений | «Разобрать условие», «Какой подход?» |
| Ученик получил INCORRECT / ON_TRACK | «Где ошибка?», «Подсказка к шагу», «Объясни проще» |
| Ученик получил hint | «Ещё подсказка», «Проверить мой шаг» |

**Acceptance Criteria:**
- [ ] Чипы рендерятся как горизонтально-скроллящиеся кнопки над Discussion field
- [ ] Нажатие на чип отправляет предзаполненный message (не structured intent — проще в реализации)
- [ ] На mobile: touch-action: manipulation, min-height 40px

#### R8. Follow-up Action Buttons на AI-сообщениях

**Описание:** После AI-ответа под сообщением появляются 1–2 кнопки для продолжения. Требует изменений в `GuidedChatMessage.tsx`.

**Acceptance Criteria:**
- [ ] Кнопки рендерятся как ghost-style Button
- [ ] Кнопки исчезают после следующего сообщения ученика

#### R9. Адаптивный Bootstrap по сложности задачи

**Описание:** Backend определяет сложность по длине task_text + номеру задания ЕГЭ и адаптирует bootstrap.

**Acceptance Criteria:**
- [ ] Простые задачи: короткий bootstrap
- [ ] Сложные задачи: развёрнутый с приглашением разобрать условие

#### R10. Предложение «Похожая задача»

**Описание:** После ошибки и исправления AI предлагает похожую задачу из KB для закрепления.

**Acceptance Criteria:**
- [ ] Подбор из kb_tasks с тем же topic_id
- [ ] Задача открывается в мини-thread (не ломает основной)

### 6.3. Future Considerations (P2)

#### R9. Визуальные схемы от AI

AI генерирует SVG-схему (диаграмма сил, электрическая цепь) для задачи. Требует investment в SVG-generation pipeline. Отложено.

#### R10. Voice-to-text для ответов

Ученик диктует промежуточный шаг голосом. Полезно для мобильных, но не критично для пилота.

#### R11. Персональный профиль ошибок

Накопительный анализ: «Ты часто забываешь учитывать трение. За последнюю неделю 4 из 7 ошибок — на трение». Требует accumulated data.

#### R12. Collaborative hints (репетитор подключается к thread)

Репетитор видит, что ученик застрял, и подключается к live-thread с кастомной подсказкой. Частично реализовано (tutor может писать в thread), но нет real-time notification.

## 8. Предложения по улучшению UX для каждого Sub-Job

**Подход v1 (P0):** Все улучшения реализуются через **system prompt** — без изменений UI. Это быстро, безопасно и тестируемо. UI-доработки (chips, buttons) — P1, после обратной связи с пилота.

### 8.1. Sub-Job S1-1: Понять условие задачи

**Текущее решение:** Условие отображается как MathText + zoomable image. AI bootstrap-сообщение — generic intro.

**Предлагаемое решение (prompt-only):**

AI bootstrap автоматически разбирает условие для сложных задач. Prompt-инструкция:

> «Если условие задачи длиннее 2 предложений или содержит несколько физических объектов — начни с краткого разбора в формате:
> **Дано:** (числа + единицы)
> **Найти:** (что определить)
> **Ситуация:** (что происходит, 1–2 предложения)
> Затем предложи: "Попробуй решить. Если не понял условие — напиши, разберём подробнее."»

**Пример AI-ответа:**

```
Давай разберём задачу!

**Дано:** m = 2 кг, v₀ = 0, α = 30°, μ = 0.2
**Найти:** ускорение a
**Ситуация:** тело скользит вниз по наклонной
плоскости с трением, начиная с покоя.

Попробуй решить. Если застрянешь на первом
шаге — напиши, какой подход хочешь применить.
```

**Почему лучше:** Формат «Дано-Найти» знаком каждому школьнику. AI сам предлагает следующее действие. Ноль UI-изменений — только prompt.

### 8.2. Sub-Job S1-2: Выбрать подход к решению

**Текущее решение:** Ученик пишет свободный текст в Discussion field. AI отвечает наводящим вопросом (из generic prompt).

**Предлагаемое решение (prompt-only):**

Progressive hint ladder через prompt. AI помнит кол-во подсказок и увеличивает конкретность:

| Кол-во hints | Стиль AI | Пример |
|--------------|----------|--------|
| 1-я подсказка | Общий наводящий вопрос | «Подумай, какая величина здесь сохраняется?» |
| 2-я подсказка | Конкретнее | «Обрати внимание: замкнутая ли система? Есть ли внешние силы?» |
| 3+ подсказка | Прямое направление | «Попробуй через закон сохранения энергии. Какие виды энергии?» |

Prompt-инструкция:

> «Это подсказка №{N}. При N=1: задай общий наводящий вопрос. При N=2: будь конкретнее, назови область физики. При N≥3: укажи конкретный закон/подход, но НЕ решай задачу и НЕ подставляй числа. В конце каждой подсказки напиши: "Попробуй с этим подходом. Если получишь промежуточный результат — напиши, проверю."»

**Почему лучше:** Нынешний AI может сразу выдать слишком конкретную подсказку (нет laddering). Ladder даёт постепенное раскрытие — core сократовского метода. Реализуется через один параметр (hint count) в prompt.

### 8.3. Sub-Job S1-3: Проверить промежуточный шаг

**Текущее решение:** Ученик может проверить только финальный ответ через Answer field. Для промежуточной проверки — только free-text Discussion.

**Предлагаемое решение (prompt-only):**

Discussion AI научится распознавать промежуточные шаги и давать быструю проверку. Ученик пишет в Discussion: «получил v = 15 м/с, правильно?» — AI проверяет шаг.

Prompt-инструкция:

> «Если ученик отправляет промежуточный результат (формулу, подстановку, числовое значение + "правильно?", "верно?", "проверь"), дай БЫСТРУЮ проверку:
> - Верно: "✅ Верно, продолжай!"
> - Неверно: "⚠️ Проверь [что именно]. Подсказка: [1 предложение]"
> НЕ показывай правильный результат. НЕ решай дальше. Ответ — 1–3 предложения max.»

**Пример:**

```
Ученик: «получил F_тр = μmg·cos30° = 0.2·2·10·0.87 = 3.46 Н, верно?»

AI: «✅ Верно! Сила трения посчитана правильно. Продолжай —
теперь запиши второй закон Ньютона вдоль наклонной плоскости.»
```

**Почему лучше:** Закрывает главную боль S1-3: «не хочу решать 20 минут и узнать, что ошибся на шаге 2». Ученик использует существующее поле Discussion — ноль UI-изменений. AI даёт ответ за секунды.

### 8.4. Sub-Job S1-4: Понять ошибку и не повторить

**Текущее решение:** AI даёт verdict INCORRECT + текстовый feedback с классификацией ошибки (calculation/concept). Один абзац текста, формальный стиль.

**Предлагаемое решение (prompt-only):**

Двухуровневое объяснение ошибки через prompt:

**Уровень 1 (сразу при INCORRECT):** Краткий фидбек + аналогия + next action.

```
❌ Ответ неверный. Ты использовал v = v₀ + at, но
здесь ускорение НЕ постоянно (тело по окружности).

Аналогия: машина на кольцевом перекрёстке — спидометр
показывает 40, но она «ускоряется», потому что
меняется направление. Нужно a = v²/R.

Попробуй пересчитать. Или напиши промежуточный
результат — проверю.
```

**Уровень 2 (после CORRECT, если до этого был INCORRECT):** Автоматическое резюме.

```
🎉 Правильно!

📌 Запомни: в задачах на движение по окружности
формула v = v₀ + at НЕ работает. Используй a = v²/R.
Ты ошибся → разобрался → решил. Так и надо!
```

Prompt-инструкция для INCORRECT:

> «При концептуальной ошибке: (1) назови конкретную ошибку, (2) объясни ПОЧЕМУ это неправильно через аналогию из повседневной жизни, (3) предложи как исправить. Пиши на языке 16-летнего.»

Prompt-инструкция для CORRECT после INCORRECT:

> «Если ученик ранее отвечал неверно, а сейчас верно — добавь "📌 Запомни: [правило, привязанное к типу ошибки]". 1–2 предложения. Только если были ошибки!»

**Почему лучше:** Teachable moment — ученик разобрался прямо сейчас, а не завтра на уроке. Аналогия делает физику понятной. Резюме фиксирует паттерн ошибки. Всё через prompt — deploy за минуты.
