## 1. Главное правило

Сократ — **рабочее место экзаменного репетитора по физике**, а не чат-продукт.

UI должен:

- помогать быстро выполнять работу (wedge: собрать ДЗ за 5–10 мин);
- делать AI-результаты управляемыми артефактами;
- корректно отображать физический контент (формулы, графики, схемы);
- поддерживать export/sharing для доставки ДЗ ученикам.

## 2. Базовые UI-правила для всех экранов

### 2.1. Один экран = одна главная работа

На каждом route-компоненте:

- Ровно **один** `<Button variant="primary">`.
- Остальные кнопки — `variant="secondary"` или `variant="ghost"`.
- Если при ревью на экране >1 primary CTA — это баг.

### 2.2. Чат не является главным UI-паттерном

Свободный текстовый ввод:

- **Допустим** как secondary элемент внутри workspace.
- **Запрещён** как единственный или доминирующий способ начать работу.
- **Запрещён** как отдельный раздел навигации ("Чат", "AI").

Визуальная иерархия ввода:

```
1. Job cards (самое заметное, верхняя часть экрана)
2. Structured input form (поля темы, экзамена, кол-ва задач)
3. Example prompts (кликабельные, под формой)
4. Free text input (нижняя часть, меньший визуальный вес)
```

### 2.3. Каждый AI-результат ведёт к действию

Обязательный action layer для AI-generated контента:

```
Primary actions (видны всегда):
  └─ `В ДЗ`          → добавляет задачу в текущий черновик ДЗ
  └─ `В мою базу`    → сохраняет в личную базу
  └─ `Редактировать`  → inline-редактирование текста задачи

Secondary actions (через кнопку ⋯):
  └─ `Сгенерировать заново`
  └─ `Создать похожую`
  └─ `Скрыть`
```

### 2.4. Статус должен быть видимым

Каждый артефакт (задача, решение, материал) отображает badge:

| Статус | Стиль badge | CSS class |
|--------|------------|-----------|
| `AI-черновик` | outline, muted text | `.badge-draft` |
| `Сохранено в базу` | solid, accent bg | `.badge-saved` |
| `Добавлено в ДЗ` | solid, primary bg | `.badge-homework` |
| `Из базы ФИПИ` | solid, neutral bg | `.badge-fipi` |
| `Моя задача` | solid, accent bg | `.badge-own` |

Обновление статуса — **inline**, без перезагрузки. После действия — checkmark animation ≤ 300ms.

### 2.5. Частые сценарии не прячем

Доступность wedge-сценария:

- `Домашки` → `Создать ДЗ` = **2 тапа** до рабочей формы.
- `Помощник` → `Собрать ДЗ по теме` = **2 тапа** до рабочей формы.
- Если путь до начала сборки ДЗ > 2 тапов — нужен рефакторинг навигации.

## 3. Навигация

### 3.1. Top nav (desktop) / Bottom tab bar (mobile)

```
Главная | Расписание | Ученики | Домашки | Помощник
```

Overflow-меню (`Ещё` или hamburger):

```
База знаний | Материалы | Оплаты | Настройки
```

### 3.2. Правила навигации

| Правило | Обоснование |
|---------|-------------|
| `Домашки` и `Помощник` **всегда** в top-level | Это wedge-точки входа |
| Нельзя добавлять top-level вкладку без обоснования через Job | Защита от feature creep |
| Названия отражают **работу**, не архитектуру | `Домашки`, не `Модули`; `Помощник`, не `AI` |
| На mobile: bottom tab bar с 5 пунктами | iOS/Android convention, большой палец |

### 3.3. Контекстная навигация

Внутри разделов — **tabs** (не отдельные page routes):

- Домашки: `Черновики` | `Отправленные` | `Архив`
- Ученики: `Все` | `Группы`
- Помощник: нет табов — job cards как точки входа

Максимум **3 таба** внутри раздела. Если нужно больше — пересмотреть информационную архитектуру.

## 4. Паттерны ключевых экранов

### 4.1. Экран "Помощник" — jobs-first workspace

#### Layout (desktop)

```
┌─────────────────────────────────────────────────┐
│  Header: "Помощник"                             │
│  Subheader: "Быстрая подготовка практики и ДЗ"  │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌───────────┐ ┌───────────┐ ┌───────────┐     │
│  │ Собрать   │ │ Создать   │ │ Решить /  │     │
│  │ ДЗ по     │ │ похожие   │ │ объяснить │     │
│  │ теме      │ │ задачи    │ │           │     │
│  └───────────┘ └───────────┘ └───────────┘     │
│                                                 │
│  Примеры запросов:                              │
│  ┌─────────────────────────────────────────┐    │
│  │ "Кинематика, ЕГЭ, 5 задач средней..."  │    │
│  │ "3 задачи на закон Ома для полной..."   │    │
│  │ "Решить задачу: тело массой 2 кг..."    │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
│  Последние результаты:                          │
│  [Task Card] [Task Card] [Task Card]            │
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │ 💬 Свободный запрос...                  │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
└─────────────────────────────────────────────────┘
```

#### Порядок элементов (сверху вниз)

1. Header + subheader (ценностное обещание, одна строка).
2. **3 action cards** — ровно три, не больше.
3. **3–5 example prompts** — кликабельные, привязаны к физике ЕГЭ.
4. **Последние результаты** — если есть, максимум 3 карточки.
5. **Free text input** — внизу, `placeholder="Свободный запрос по физике..."`.

#### Layout (mobile)

Тот же порядок, но:

- Action cards стакаются вертикально (по 1 в строке или 1.5 карточки видно с горизонтальным скроллом).
- Example prompts — горизонтальный scroll chips.
- Free text input — sticky bottom.

#### Запрещено

- Blank chat как основной вид.
- Более 3 action cards.
- 8+ режимов на первом экране.
- Лента предыдущих чат-сообщений как дефолтное состояние.

### 4.2. Job workspace — input/output layout

При выборе любой Job открывается workspace с двумя зонами.

#### Desktop: side-by-side

```
┌──────────────────────┬──────────────────────────┐
│   INPUT PANEL (40%)  │   OUTPUT AREA (60%)      │
│                      │                          │
│  Тема: [________]    │  [Result Card 1]         │
│  Экзамен: ЕГЭ ○ ОГЭ │  [Result Card 2]         │
│  Кол-во: [5 ▾]      │  [Result Card 3]         │
│  Сложность: ○○○      │  [Result Card 4]         │
│                      │  [Result Card 5]         │
│  [Создать задачи]    │                          │
│                      │  ─────────────────────── │
│  Расширенные ↓       │  [Добавить все в ДЗ]     │
│  ┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈  │  [Сохранить все в базу]  │
│  Тип задания: [__]   │                          │
│  Источник: [__]      │  💬 Уточнить результат...│
│                      │                          │
└──────────────────────┴──────────────────────────┘
```

#### Mobile: stacked

```
┌─────────────────────────────┐
│  INPUT (collapsed, sticky)  │
│  Кинематика · ЕГЭ · 5 задач│
│  [Изменить] [Создать ещё]   │
├─────────────────────────────┤
│  OUTPUT (scrollable)        │
│  [Result Card 1]            │
│  [Result Card 2]            │
│  [Result Card 3]            │
│  ...                        │
├─────────────────────────────┤
│  [Добавить все в ДЗ]  fixed │
└─────────────────────────────┘
```

#### Обязательные элементы Input Panel

| Элемент | Тип | Обязателен | Пример |
|---------|-----|-----------|--------|
| Тема | text input с автокомплитом | Да | "Кинематика" |
| Экзамен | radio: ЕГЭ / ОГЭ | Да | ЕГЭ |
| Количество задач | stepper или select (3–10) | Да | 5 |
| Сложность | toggle: лёгкие / средние / сложные / микс | Нет (L2) | Микс |
| Тип задания | select из номеров ЕГЭ | Нет (L2) | Задание №7 |
| Источник | chips: ФИПИ / AI / Все | Нет (L2) |  |

L0/L1 — показывать только тему, экзамен, количество.
L2/L3 — раскрывать по `Расширенные параметры`.

#### Обязательные элементы Output Area

- Result cards с action layer.
- Batch actions: `Добавить все в ДЗ`, `Сохранить все в базу`.
- Текстовое поле для уточнения: `"Сделай задачи сложнее"`, `"Замени 3-ю"`.

#### Проверка

```
□ Input и output видны одновременно (desktop) или input sticky (mobile).
□ Primary CTA в input — один: `Создать задачи`.
□ Primary CTA в output — один: `Добавить все в ДЗ`.
□ Есть batch actions для множественных результатов.
□ Контекст input не теряется при скролле output.
```

### 4.3. Drawer / modal для встраивания в workflow

Когда База знаний или Помощник вызываются **изнутри Домашек**.

#### Desktop: side drawer (справа)

```
┌────────────────────────┬───────────────────────┐
│  ДЗ DRAFT (основной)   │  DRAWER (400–500px)   │
│                        │                       │
│  ДЗ: Кинематика, 10А   │  База знаний          │
│  1. [Task Card]        │  Тема: Кинематика     │
│  2. [Task Card]        │  [Task Card] [+В ДЗ]  │
│  3. ________________   │  [Task Card] [+В ДЗ]  │
│     Добавить задачу ↓  │  [Task Card] [+В ДЗ]  │
│                        │                       │
│  [Отправить ДЗ]        │  [Закрыть]            │
└────────────────────────┴───────────────────────┘
```

#### Mobile: full-screen sheet (bottom-up)

```
┌─────────────────────────────┐
│  ← Назад к ДЗ              │
│  База знаний: Кинематика    │
│                             │
│  [Task Card] [+В ДЗ]       │
│  [Task Card] [+В ДЗ]       │
│  [Task Card] [+В ДЗ]       │
│                             │
└─────────────────────────────┘
```

#### Каноничный flow

```
Создание ДЗ → [+Добавить из базы] → drawer → выбрал задачу → [В ДЗ] →
badge обновился → вернулся в draft ДЗ → задача уже в списке
```

#### Запрещено

- Page redirect из ДЗ в Базу знаний (теряется draft).
- Открытие нового окна/вкладки.
- Drawer шире 500px (не должен перекрывать draft).

**Структура two-tab drawer:**

```
├─────────────────────────────────┤
│                                 │
│  Tab: Каталог                   │
│  ┌─ Topic List ──────────────┐  │
│  │ ← Назад (если drill-down) │  │
│  │ Кинематика         42 →   │  │

│  │ ...                       │  │
│  └───────────────────────────┘  │
│                                 │
│  Tab: Моя база                  │
│  ┌─ Folder List ─────────────┐  │
│  │ Физика 10кл         12 →  │  │
│  │ ЕГЭ подготовка       8 → │  │
│  └───────────────────────────┘  │
│                                 │
│  After drill-down:              │
│  ┌─ PickerTaskCard ──────────┐  │
│  │ ☐ [Badge] Preview text... │  │
│  │                  [В ДЗ]   │  │
│  └───────────────────────────┘  │
│  ┌─ PickerTaskCard ──────────┐  │
│  │ ☑ [Badge] Preview text... │  │
│  │                     [✓]   │  │
│  └───────────────────────────┘  │
│                                 │
│  [Добавить выбранные (3)]       │
└─────────────────────────────────┘
```

**Паттерны:**

| Паттерн | Описание |
|---------|----------|
| Two-tab picker | Каталог Сократа (read-only) / Моя база (личные папки) внутри одного drawer |
| Drill-down inside sheet | Topic list → task list с «← Назад» навигацией; аналогично для папок |
| `PickerTaskCard` (compact) | Collapsed-only, no expand, checkbox + single «В ДЗ» CTA. Отличается от полного `TaskCard` из 5.2 |
| Batch bar | Появляется при 3+ доступных (не добавленных) задачах. Checkbox select + «Добавить выбранные (N)» |
| `topicHint` context-passing | Визард передаёт `topicHint` (тему из ДЗ) → picker auto-select topic в каталоге |
| Dedupe via `addedKbTaskIds` | Уже добавленные задачи показывают ✓ вместо «В ДЗ» (disabled) |

**Ограничения текущей реализации:**
- Desktop: `side="right"`. Mobile P2: нужен `side="bottom"` full-screen (как описано в 4.3 Mobile)
- Нет поиска / фильтра внутри picker (P2)

## 5. Паттерны карточек

### 5.1. Action Card — точка входа в Job

Используется на landing "Помощника". Ровно **3 карточки**.

#### Структура

```
┌───────────────────────────┐
│  [Icon]                   │
│                           │
│  Собрать ДЗ по теме       │  ← заголовок (глагол + объект)
│  Сгенерировать набор      │  ← подпись (1 строка, job-language)
│  задач по теме урока      │
│                           │
└───────────────────────────┘
  whole-card clickable
  hover: subtle elevation + border color change
  pressed: scale(0.98)
```

#### Три каноничные карточки

| # | Заголовок | Подпись | Job |
|---|-----------|---------|-----|
| 1 | Собрать ДЗ по теме | Сгенерировать набор задач по теме урока | Wedge job |
| 2 | Создать похожие задачи | По образцу одной задачи | Supporting job 1 |
| 3 | Решить / объяснить | Получить ответ, шаги и объяснение | Supporting job 2 |

#### Запрещено

- Более 3 action cards на landing.
- 2+ CTA внутри одной action card.
- Текст длиннее 2 строк подписи.
- Технические названия в заголовке (`AI-генератор`, `Solver`).

### 5.2. Task Card — главный паттерн для задач

Используется в: Базе знаний, ДЗ draft, AI-результатах, поиске.

#### Collapsed state (основной)

```
┌────────────────────────────────────────────────┐
│ [badge: AI-черновик]  Кинематика · ЕГЭ №7      │
│                                                │
│ Тело брошено вертикально вверх с начальной     │
│ скоростью v₀ = 20 м/с. Определите максималь... │
│                                                │
│ [В ДЗ]  [В базу]  [Редактировать]  [⋯]        │
└────────────────────────────────────────────────┘
```

#### Обязательные элементы collapsed state

| Элемент | Описание | Обязателен |
|---------|----------|-----------|
| Source badge | `AI-черновик` / `Из базы ФИПИ` / `Моя задача` | Да |
| Тема + подтема | Chips или текст | Да |
| Номер задания ЕГЭ/ОГЭ | Если применимо, например `ЕГЭ №7` | Да (если есть) |
| Preview текста | 2–4 строки условия с LaTeX-рендерингом | Да |
| Thumbnail вложения | Если задача содержит график/схему | Да (если есть) |
| Primary actions | `В ДЗ`, `В базу`, `Редактировать` | Да |
| Secondary actions | Через `⋯` | Да |

#### Expanded state

Раскрывается по клику на preview текста или кнопку "Развернуть":

```
┌────────────────────────────────────────────────┐
│ [badge: AI-черновик]  Кинематика · ЕГЭ №7      │
│                                                │
│ Тело брошено вертикально вверх с начальной     │
│ скоростью v₀ = 20 м/с. Определите максималь-  │
│ ную высоту подъёма тела. Сопротивлением        │
│ воздуха пренебречь. g = 10 м/с².               │
│                                                │
│ ┌────────────────────────────────────────┐      │
│ │ [График/схема, если есть]              │      │
│ └────────────────────────────────────────┘      │
│                                                │
│ Ответ: 20 м  (скрыт по умолчанию, toggle)     │
│                                                │
│ [В ДЗ]  [В базу]  [Редактировать]  [⋯]        │
│                                                │
│ ⋯ → Сгенерировать заново                      │
│ ⋯ → Создать похожую                           │
│ ⋯ → Решить / объяснить                        │
│ ⋯ → Скрыть                                    │
│ ⋯ → Удалить                                   │
└────────────────────────────────────────────────┘
```

#### Визуальное различие по источнику

| Источник | Left border color | Badge style |
|----------|------------------|-------------|
| AI-черновик | `var(--color-warning)` dashed | outline, muted |
| Из базы ФИПИ | `var(--color-neutral)` solid | solid, neutral |
| Моя задача | `var(--color-accent)` solid | solid, accent |
| Сохранено в базу | `var(--color-accent)` solid | solid, accent |
| Добавлено в ДЗ | `var(--color-primary)` solid | solid, primary |

Правило: AI-контент визуально отличается от human-authored контента. Dashed border + muted badge = AI. Solid border = проверенный/свой.

#### Рендеринг формул в task card

- Inline формулы (`$v_0 = 20$ м/с`) рендерятся через KaTeX прямо в тексте.
- Display формулы (`$$h = \frac{v_0^2}{2g}$$`) рендерятся как отдельный блок.
- На мобильных: если формула шире экрана — горизонтальный scroll блока, не всей карточки.
- В collapsed state: формулы рендерятся, усечённый текст не обрезает формулу посередине.

#### Проверка Task Card

```
□ Badge источника виден в collapsed state.
□ Тема и номер задания ЕГЭ отображаются.
□ LaTeX рендерится (не сырой код).
□ Primary actions видны без раскрытия.
□ Кнопка `В ДЗ` видна без открытия details (требование wedge).
□ Expanded state открывается без page redirect.
□ Ответ скрыт по умолчанию (toggle).
□ Визуальное различие AI vs. не-AI.
```

## 6. Batch-операции

### 6.1. Зачем

Репетитор с 10+ учениками работает с множеством задач одновременно. Добавлять по одной задаче — слишком медленно для wedge-обещания "5–10 минут".

### 6.2. Batch actions после AI-генерации

```
┌────────────────────────────────────────────────┐
│  ☑ Выбрать все (5)                             │
│                                                │
│  ☑ [Result Card 1]                             │
│  ☑ [Result Card 2]                             │
│  ☐ [Result Card 3]  ← снял выбор              │
│  ☑ [Result Card 4]                             │
│  ☑ [Result Card 5]                             │
│                                                │
│  Выбрано: 4 из 5                               │
│  [Добавить выбранные в ДЗ] [Сохранить в базу] │
└────────────────────────────────────────────────┘
```

### 6.3. Batch actions в Базе знаний

- `Выбрать все на странице` → `Добавить в ДЗ`
- `Добавить N случайных по теме` → stepper (3–10) → `В ДЗ`

### 6.4. Проверка

```
□ После AI-генерации набора (3+ задач) есть batch action.
□ Checkbox на каждой result card.
□ `Выбрать все` / `Снять все`.
□ Счётчик выбранных.
□ Batch CTA активна только при >0 выбранных.
```

## 7. Экспорт и шаринг

### 7.1. Точка экспорта

Кнопка экспорта доступна на экране ДЗ Draft и на экране отправленного ДЗ.

### 7.2. Поддерживаемые форматы

| Формат | Приоритет | Описание |
|--------|-----------|----------|
| Копировать как текст | P0 | Форматированный текст с нумерацией, для вставки в мессенджер |
| Скачать PDF | P0 | С LaTeX-формулами, графиками, номерами задач. Для печати |
| Ссылка на веб-версию | P1 | Публичная страница с задачами (read-only) |
| Отправить через Telegram | P1 | Если подключён Telegram Bot |

### 7.3. UI экспорта

```
[Экспорт ▾]
  ├─ 📋 Копировать текст
  ├─ 📄 Скачать PDF
  ├─ 🔗 Скопировать ссылку
  └─ ✈️ Отправить в Telegram  (если подключён)
```

Dropdown, а не отдельная страница. Действие выполняется **мгновенно** (копирование) или с коротким loading (PDF-генерация).

### 7.4. Требования к PDF

- Заголовок: тема, дата, кол-во задач.
- Нумерация задач.
- LaTeX-формулы рендерятся как изображения или vector.
- Графики/схемы включены.
- Формат: A4, шрифт 12pt, отступы для записи ответов.
- Опционально: с ответами / без ответов (toggle перед экспортом).

### 7.5. Требования к копированию текста

- Формулы: Unicode-символы (v₀, м/с², ΔT) вместо LaTeX-кода.
- Нумерация сохраняется.
- Графики/схемы: пропускаются или заменяются на "[см. рисунок]".

### 7.6. Проверка

```
□ Кнопка экспорта доступна на ДЗ Draft.
□ PDF содержит рендеренные формулы.
□ Копирование даёт читаемый текст в Telegram.
□ Ссылка открывает read-only страницу.
□ Экспорт не требует >2 кликов.
```

## 8. Паттерны состояний

### 8.1. Empty state

Каждый empty state содержит:

| Элемент | Обязателен |
|---------|-----------|
| Заголовок (что здесь будет) | Да |
| Одно главное действие (кнопка) | Да |
| 1–2 примера или hint | Нет (но рекомендуется) |

Примеры:

**Домашки — пусто:**
```
У вас пока нет домашних заданий.
Соберите первое ДЗ по теме урока за пару минут.
[Создать ДЗ]
```

**База знаний — пусто:**
```
В вашей базе пока нет сохранённых задач.
Задачи появляются, когда вы сохраняете их из Помощника или базы ФИПИ.
[Открыть Помощник]
```

**Запрещено:** `Здесь пока ничего нет` без next step.

### 8.2. Loading state

Для AI-генерации — **контекстный loading** с текстом:

| Job | Loading text |
|-----|-------------|
| Собрать ДЗ | `Собираем задачи по теме...` |
| Похожие задачи | `Создаём похожие задачи...` |
| Решить / объяснить | `Решаем задачу...` |
| Общий fallback | `Готовим результат...` |

Визуально: skeleton cards (контур будущих task cards) + текст статуса.

**Запрещено:**

- Только spinner без контекста.
- Имитация "thinking" с бегущими точками как в чате.
- Loading дольше 30 секунд без индикации прогресса.

### 8.3. Error state

Структура:

```
┌────────────────────────────────────────────────┐
│ ⚠ Не удалось сгенерировать задачи.            │
│                                                │
│ Попробуйте уменьшить количество задач или     │
│ уточнить тему.                                 │
│                                                │
│ [Попробовать снова]  [Изменить параметры]      │
└────────────────────────────────────────────────┘
```

Правила:

- Текст ошибки **человеческий**, не технический.
- Всегда есть **recovery action** (кнопка).
- Для физики: если AI не знает тему — предложить выбрать из списка тем.

## 9. Паттерны AI-слоя

### 9.1. AI as draft, not oracle

- Все AI-результаты имеют badge `AI-черновик`.
- Каждый результат содержит `Редактировать` как primary action.
- Тон UI: "Черновик решения" (не "Решение"), "Предложенные задачи" (не "Готовые задачи").

### 9.2. Example prompts

- Не больше 5 на экране.
- Привязаны к физике ЕГЭ/ОГЭ.
- Кликабельны: подставляют шаблон в input field.
- Обновляются контекстно: если пользователь уже выбрал тему — промпты привязаны к этой теме.

Примеры:

```
"5 задач по кинематике, ЕГЭ, средняя сложность"
"3 задачи на закон Ома для полной цепи"
"Задачи на 2-й закон Ньютона с наклонной плоскостью"
"Задачи на фотоэффект, базовый уровень"
"Тепловой баланс, ОГЭ, 4 задачи"
```

### 9.3. Chips — быстрые модификаторы

Появляются рядом с AI-результатом для быстрой итерации:

```
[+ проще]  [+ сложнее]  [+ с ответами]  [+ с решением]  [другой тип]
```

Правила:

- Не более 5 chips на одной строке.
- Клик на chip вызывает **re-generation** с модифицированными параметрами.
- Chip не является формой — один клик = одно действие.

### 9.4. Refinement input

После генерации — текстовое поле для уточнения:

```
💬 Уточнить: "Замени 3-ю задачу на более сложную с графиком"
```

- Placeholder привязан к контексту: `"Например: замени задачу, сделай сложнее, добавь график"`.
- Результат уточнения заменяет/дополняет output, а не создаёт чат-ленту.
- Если уточнение затрагивает одну задачу — обновляется только эта карточка.

## 10. Desktop и Mobile

### 10.1. Стратегия: desktop-first creation, mobile-companion

| Действие | Оптимизировано для |
|----------|-------------------|
| Сборка ДЗ (wedge) | Desktop |
| AI-генерация задач | Desktop |
| Drag-and-drop порядка задач | Desktop |
| Быстрый просмотр ДЗ перед уроком | Mobile |
| Отправка ДЗ ученику | Mobile |
| Просмотр статуса выполнения | Mobile |
| Поиск по базе знаний | Оба |

### 10.2. Desktop-специфичное

- Side-by-side layout для job workspace.
- Side drawer для встраивания базы в ДЗ.
- Drag-and-drop для порядка задач в ДЗ.
- Hover states на карточках.
- Keyboard shortcuts (Ctrl+Enter для генерации, Ctrl+S для сохранения).

### 10.3. Mobile-специфичное

- Stack layout для всех экранов.
- Bottom sheet / full-screen modal вместо drawer.
- Fixed bottom CTA в критичных потоках (ДЗ draft, AI output).
- Swipe actions на task cards: swipe right = В ДЗ, swipe left = Скрыть.
- Sticky collapsed input panel при скролле output.
- Long-press для secondary actions вместо hover.

### 10.4. Формулы на мобильных

- Inline формулы рендерятся в потоке текста, перенос по словам.
- Display формулы: горизонтальный scroll блока формулы, не всей карточки.
- Минимальный font-size для формул: 14px (читаемость на телефоне).

### 10.5. Запрещено

- "Сжимать desktop" — mobile должен быть самостоятельной композицией.
- Таблицы шире 4 колонок на mobile (используй cards вместо table).
- Модалки, закрывающие fixed bottom CTA.

## 11. UI-антипаттерны (обязательные к проверке)

Каждый антипаттерн — повод для reject при code review.

| # | Антипаттерн | Что искать в коде | Почему плохо |
|---|-------------|-------------------|-------------|
| 1 | Чат как основной UI | Chat bubble компоненты в main layout | Убивает workspace positioning |
| 2 | >1 primary CTA на экране | Несколько `variant="primary"` на route | Cognitive overload |
| 3 | Frequent actions в `⋯` | `В ДЗ` или `Редактировать` внутри dropdown | Скрывает wedge-действие |
| 4 | AI output без action layer | Рендер AI-текста без кнопок под ним | Результат не переходит в workflow |
| 5 | Навигация по архитектуре | Названия разделов: `Модули`, `AI`, `Tools` | Не отражает работу пользователя |
| 6 | База не связана с ДЗ | Нет кнопки `В ДЗ` на карточке в Базе | Разрыв workflow |
| 7 | Solver-first landing | Первый экран = "вставьте задачу для решения" | Не wedge |
| 8 | Magic без контроля | Auto-actions без подтверждения | Потеря доверия |
| 9 | Перегруженные карточки | >6 видимых элементов в collapsed state | Сложно сканировать |
| 10 | Формулы как raw LaTeX | `$...$` без KaTeX-рендеринга | Непрофессиональный вид |
| 11 | Нет batch actions | Генерация 5+ задач без `Добавить все` | Медленный wedge flow |
| 12 | Нет экспорта | ДЗ собрано, но нельзя отправить | Обрыв workflow |

## 12. Каноничные UI flows для wedge

### Flow 1. Собрать ДЗ по теме после урока (P0 — wedge)

```
1. Репетитор открывает `Домашки` [1 тап]
2. Нажимает `Создать ДЗ` [2 тапа]
3. Выбирает тему, экзамен, кол-во задач
4. Нажимает `Создать задачи`
5. Видит result cards с checkbox и batch actions
6. Нажимает `Добавить все в ДЗ` (или снимает checkbox с ненужных)
7. Видит ДЗ Draft с задачами в нужном порядке
8. Опционально: drag-and-drop для порядка, добавляет инструкцию
9. Нажимает `Экспорт` → `Копировать текст` → вставляет в Telegram
   ИЛИ `Отправить ДЗ` если подключён бот

Время: ≤ 5 минут
Тапов от входа до отправки: ≤ 8
```

### Flow 2. Нарастить практику похожими задачами (P0)

```
1. Открывает `Помощник` [1 тап]
2. Жмёт `Создать похожие задачи` [2 тапа]
3. Вставляет условие ИЛИ выбирает задачу из базы
4. Получает result cards
5. Жмёт `В ДЗ` на выбранных ИЛИ `Сохранить все в базу`
```

### Flow 3. Решить / объяснить задачу (P1)

```
1. Открывает `Помощник` [1 тап]
2. Жмёт `Решить / объяснить` [2 тапа]
3. Вставляет задачу
4. Получает:
   - Ответ (числовой)
   - Шаги решения (с формулами в LaTeX)
   - Объяснение для ученика (упрощённый язык)
5. Actions: `Создать похожую`, `Сохранить в базу`, `Редактировать`
```

### Flow 4. Добавить задачу из базы в ДЗ (P0)

```
1. Находится в ДЗ Draft
2. Жмёт `+ Добавить из базы`
3. Открывается drawer с Базой знаний (уже отфильтрованной по теме ДЗ)
4. Выбирает задачи → `В ДЗ`
5. Drawer закрывается, задачи появляются в draft
```

## 13. Component naming rules

### Naming convention для UI-элементов

| Тип | Формат | Пример |
|-----|--------|--------|
| Разделы навигации | Существительное (работа) | `Домашки`, `Помощник` |
| Action cards | Глагол + объект | `Собрать ДЗ по теме` |
| Кнопки действий | Глагол / В + место | `Создать`, `В ДЗ`, `В базу` |
| Badges | Статус / источник | `AI-черновик`, `Из базы ФИПИ` |
| Placeholders | Пример + context | `"5 задач по кинематике, ЕГЭ"` |

### Запрещённый словарь

Следующие слова **запрещены** в любых UI labels:

```
AI, Chat, LLM, Assistant, Playground, Generator,
Smart, Auto, Magic, Генерация, Авто-помощь,
Умный режим, Ассистент v2, Нейро-
```

## 14. Definition of Done для UI-фичи

UI-фича считается готовой, когда:

```
□ Привязана к Job из backlog (комментарий // Job: ...).
□ Один primary CTA на экране.
□ AI-result имеет action layer (В ДЗ, В базу, Редактировать).
□ AI-result имеет badge статуса.
□ Формулы рендерятся через KaTeX (не сырой LaTeX).
□ Empty, loading, error states реализованы.
□ Mobile и desktop layouts продуманы и реализованы.
□ Naming по словарю (без запрещённых слов).
□ Batch actions для наборов из 3+ элементов.
□ Экспорт доступен для wedge-сценариев.
□ Не нарушает ни один антипаттерн из раздела 11.
```
