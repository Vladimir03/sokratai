# Промпт: Кросс-анализ B2C×B2B сегментов — поиск точек максимальной силы

> Объединяет результаты B2C-сегментации (родители+школьники) и B2B-сегментации (репетиторы).
> Логика: «follow the money + follow the decision chain» по Замесину.
> Готов к запуску в ChatGPT или Claude.

---

## Промпт для копирования

```
Ты — продуктовый стратег, работающий по методологии Advanced Jobs To Be Done (AJTBD). Тебе даны два результата сегментации для одного продукта — AI-платформы СократAI (sokratai.lovable.app):

1. B2C-сегментация: 5 сегментов РОДИТЕЛЕЙ+ШКОЛЬНИКОВ (demand side, кто платит и использует)
2. B2B-сегментация: 5 сегментов РЕПЕТИТОРОВ (supply side, кто рекомендует и тоже платит)

Продукт работает по B2B2C-модели: репетитор рекомендует → родитель платит → школьник использует. Поэтому точки максимальной силы — там, где работы репетитора СОВПАДАЮТ с работами родителя+школьника.

**B2C-2: «Профильная математика: марафонская подготовка»** (Балл 26/30)

## Задание

### Шаг 1. Построй матрицу пересечений B2C × B2B (5×5 = 25 пар)

Для каждой пары оцени:
- **Совпадение работ (1-5)**: насколько Core Jobs репетитора совпадают с Core Jobs родителя+школьника? (5 = работы усиливают друг друга напрямую, 1 = слабая связь)
- **Сила канала (1-5)**: насколько вероятно, что ЭТОТ тип репетитора порекомендует продукт ЭТОМУ типу родителя? (5 = рекомендация естественна, 1 = нет контакта)
- **Экономическая совместимость (1-5)**: совпадают ли ценовые ожидания? Готов ли родитель платить за продукт, который использует этот репетитор? (5 = идеальное совпадение, 1 = разрыв)
- **Score = произведение трёх оценок** (макс 125)

Выведи матрицу в виде таблицы.

### Шаг 2. Определи ТОП-5 пересечений по Score

Для каждого пересечения из ТОП-5 опиши:

**Пересечение: [B2B-N] × [B2C-M]**

**Название связки:** [короткое ёмкое название]

**Как работает цепочка решения:**
- Триггер репетитора: [что заставляет его искать инструмент]
- Триггер родителя: [что заставляет его платить]
- Точка совпадения работ: [какая конкретная работа выполняется для ОБОИХ одновременно]

**Совпадающие Core Jobs:**
- Работа репетитора → Как она выглядит для родителя → Как она выглядит для школьника

**Конфликты работ (если есть):**
- [Где работы могут противоречить: например, репетитор хочет автоматизацию, а родитель хочет «живое» внимание]

**Экономика связки:**
- Сколько готов платить репетитор (подписка B2B)
- Сколько готов платить родитель (подписка B2C или включено в стоимость репетитора)
- B2B2C-мультипликатор: 1 репетитор → N учеников → N родителей

**Wedge-потенциал:**
- Насколько эта связка подходит как ПЕРВАЯ точка входа на рынок
- Что нужно в MVP для обслуживания именно этой связки

**Score:** [число] = Совпадение работ [X] × Сила канала [X] × Экономика [X]

### Шаг 3. Стратегическая рекомендация

На основе ТОП-5 пересечений:

1. **Wedge-связка (первый пилот):** Какое одно пересечение B2B×B2C атаковать первым? Почему?

2. **Последовательность расширения:** В каком порядке добавлять следующие связки? (с обоснованием — какие работы уже закрыты, какие новые добавляются)

3. **Ключевые риски:** Где цепочка «репетитор рекомендует → родитель платит → школьник использует» может сломаться? Для каждого риска — как его проверить быстро и дёшево.

4. **Инсайт о ценообразовании:** Кто должен платить в каждой связке — репетитор, родитель, или оба? Как это влияет на бизнес-модель?
```
