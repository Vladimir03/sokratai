Егор Блинов предложил формат проверки «развернутое решение»: ученик должен показать ход решения, а не только ответ. Без решения бот не принимает задачу. Цитата: «Интересно было бы попробовать формат проверки: "развернутое решение", чтобы ученики показывали, как они пришли к этому ответу. А без решения бот бы не принимал задачу.»

**Импликация для продукта:** Репетитор хочет контролировать глубину проверки — не только правильность ответа, но и процесс рассуждения ученика. Это усиливает R2 (понимать прогресс) и открывает потенциальную фичу: настраиваемый режим проверки (answer-only vs full-solution) в конструкторе ДЗ.
