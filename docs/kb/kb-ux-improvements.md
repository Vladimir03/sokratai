## Часть 1. Диагностика текущих проблем

### 1.1 Изображения задач не видны без клика

**Проблема:** В `TaskCard.tsx` thumbnail загружается (80×160px), но только для `attachment_url`. На скриншотах видно, что карточка с фото задачи показывает `[Задача на фото]` как текст-заглушку — и только при раскрытии карточки появляется полноразмерная картинка. Репетитор не может бегло просмотреть задачи с графиками, не кликая по каждой.

**Корневая причина:** Thumbnail рендерится (строки 196–216), но:
- Размер thumbnail слишком мал (`h-20 max-w-[160px]`) — для задач с графиками это нечитабельно
- Задачи, где текст = `[Задача на фото]` (вся суть в изображении), выглядят пустыми в свёрнутом виде
- Нет "hero image" режима для задач, где изображение — основной контент

**Рекомендация:**

| Сценарий | Текущее поведение | Целевое поведение |
|----------|-------------------|-------------------|
| Задача с текстом + фото | Мелкий thumbnail 80px | Inline-preview 160–200px высоты |
| Задача `[Задача на фото]` | Текст-заглушка + мелкий thumb | **Hero-image** режим: фото на всю ширину карточки (max-h-64), текст-заглушка скрыт |
| Несколько фото | `+N` счётчик | Горизонтальная лента с превью (scroll snap) |

**Конкретные изменения в `TaskCard.tsx`:**
1. Увеличить thumbnail до `max-h-40 max-w-full` (160px высоты, на всю ширину)
2. Добавить условие: если `task.text` содержит только `[Задача на фото]` или пуст — показывать изображение в "hero" режиме (`max-h-64 w-full object-contain`)
3. Для множественных фото — `flex gap-2 overflow-x-auto snap-x` горизонтальный скролл
4. Eager loading для первого изображения (уже сделан), lazy для остальных

```
// Pseudo-code для hero-image detection
const isImageOnlyTask = !task.text ||
  task.text.trim() === '[Задача на фото]' ||
  task.text.trim().length < 20;

// Если задача = фото, показать hero вместо thumbnail
{!isExpanded && isImageOnlyTask && thumbUrl && (
  <img src={thumbUrl} className="mt-2 max-h-64 w-full rounded-xl object-contain" />
)}
{!isExpanded && !isImageOnlyTask && thumbUrl && (
  <img src={thumbUrl} className="mt-2 max-h-40 rounded-xl object-contain" />
)}
```

### 1.2 LaTeX не рендерится — `stripLatex()` убивает формулы

**Проблема:** `stripLatex.ts` превращает `$v_1$` в `v1`, `$(0;\;2)$` в `(0;2)`. Репетитор видит "сырой" текст без индексов, дробей, степеней. В чате (`ChatMessage.tsx`) те же формулы рендерятся красиво через `remarkMath + rehypeKatex`.

**Корневая причина:** Сознательное решение (preview = plain text), но оно противоречит UX-принципу #16: "Физика — это не plain text. LaTeX-формулы, единицы измерения, графики, номера заданий КИМ — обязательны".

**Рекомендация: KaTeX-рендеринг в карточках задач**

| Аспект | Решение |
|--------|---------|
| Компонент | Новый `<MathText text={task.text} />` — лёгкая обёртка |
| Рендеринг | `react-markdown` + `remarkMath` + `rehypeKatex` (как в ChatMessage) |
| Lazy loading | `React.lazy(() => import('./MathText'))` — KaTeX CSS (~26KB) грузится только при наличии `$` в тексте |
| Preprocessing | Та же `preprocessLatex()` что и в ChatMessage — `\[..\]` → `$$`, `\(..\)` → `$` |
| Fallback | Пока грузится — показывать `stripLatex(text)` как plain text |
| Свёрнутое состояние | `line-clamp-2` работает с KaTeX inline — проверено |

**Важно для перфоманса:**
- `katex.min.css` — lazy import, один раз на страницу
- `MathText` — `React.memo` с `text` как единственная зависимость
- Не импортировать KaTeX в shared UI-компоненты (правило performance.md)

```
// MathText.tsx — новый компонент
import ReactMarkdown from 'react-markdown';
import remarkMath from 'remark-math';
import rehypeKatex from 'rehype-katex';
import { preprocessLatex } from '@/utils/latexPreprocess';

export const MathText = memo(({ text, className }) => {
  useEffect(() => { import('katex/dist/katex.min.css'); }, []);

  return (
    <ReactMarkdown
      remarkPlugins={[remarkMath]}
      rehypePlugins={[rehypeKatex]}
      className={className}
    >
      {preprocessLatex(text)}
    </ReactMarkdown>
  );
});
```

**Где применить:**
- `TaskCard.tsx` — task.text, task.answer, task.solution
- `KBPickerSheet.tsx` — preview текста задачи (120 символов → рендерить с KaTeX)
- `HWDrawer.tsx` — snapshot preview

### 1.3 Текст задач нельзя выделить и скопировать

**Проблема:** Вся карточка обёрнута в `role="button"` с `onClick={onToggle}` (строка 157). При попытке выделить текст — карточка раскрывается/сворачивается. Копирование текста невозможно.

**Рекомендация: разделение кликабельных зон**

**Вариант A (рекомендуемый) — Copy-button:**
- Убрать `role="button"` с контейнера текста
- Оставить клик по заголовку (badges, chevron) для toggle
- Добавить кнопку "Копировать" (иконка Copy) в контекстное меню и/или по hover
- При копировании — копировать raw text с LaTeX (для вставки в другие инструменты)

**Вариант B — Long-press / double-click для выделения:**
- Одиночный клик = toggle
- Двойной клик или long-press = включить режим выделения текста
- Менее интуитивен, но сохраняет текущий UX

**Конкретные изменения:**

```
// Разделить карточку на 2 зоны:
// 1. Header (badges + chevron) — кликабелен для toggle
// 2. Body (текст + изображения) — текст выделяемый

<article className="...">
  {/* Zone 1: Toggle header */}
  <div role="button" onClick={onToggle} className="flex items-center gap-2 px-4 pt-4">
    <SourceBadge /> <span>КИМ №</span> <ChevronDown />
  </div>

  {/* Zone 2: Selectable content */}
  <div className="px-4 pb-4 select-text">
    <MathText text={task.text} className="line-clamp-2" />
    {/* images, answer, solution */}
  </div>

  {/* Zone 3: Actions (stopPropagation уже есть) */}
  <div className="flex gap-2" onClick={e => e.stopPropagation()}>
    <CopyButton text={task.text} />
    <button>В ДЗ</button>
  </div>
</article>
```

**Copy-button поведение:**
- Клик → копирует raw текст задачи (с LaTeX-разметкой) в буфер обмена
- Toast: "Задача скопирована"
- Для задач с фото — копировать текст + "(см. изображение)"

### Категория B: Medium-term (1–2 недели)

#### B1. Поиск по задачам внутри KB
**Job:** A1 (быстро найти задачу по ключевому слову), E2 (найти ранее сохранённую)
**Что делают конкуренты:** Школково — поиск по тексту задачи. РЕШУ ЕГЭ — фильтры по КИМ. DIDAK — поиск по банку.
**Рекомендация:** Search bar вверху страницы KB. Full-text search по `task.text` + `task.answer`. Фильтры: КИМ-номер, подтема, наличие изображения, источник (Каталог/Моя).
**Effort:** 1 неделя | **Impact:** высокий (ключевой retention driver)

#### B2. Фильтры по КИМ-номеру и сложности
**Job:** A2 (проверить формат), A3 (сбалансировать сложность)
**Что делают конкуренты:** Школково и РЕШУ ЕГЭ — фильтрация по номеру задания КИМ.
**Рекомендация:** Chips-фильтры: "КИМ 1–7" (ч.1 с кратким ответом), "КИМ 24–30" (ч.2, развёрнутый ответ). Каждый chip = toggle. Сохраняется в URL для шаринга.
**Effort:** 3–5 дней | **Impact:** средний-высокий

#### B3. Drag-and-drop задач в ДЗ (desktop)
**Job:** B1 (понять что было на уроке → собрать ДЗ)
**Рекомендация:** На desktop — split view: KB слева, черновик ДЗ справа. Drag задачу из KB → drop в черновик. Быстрее чем "В ДЗ" кнопка.
**UX-принцип #9:** "Workflow first, library second" — KB интегрирована в поток создания ДЗ.
**Effort:** 1.5 недели | **Impact:** высокий (wedge speed: 5–10 мин на ДЗ)

#### B4. Быстрый просмотр ответа без раскрытия карточки
**Job:** A2 (верифицировать задачу), D1 (проверить ответ)
**Рекомендация:** Hover или tap на иконку "глаз" → tooltip/popover с ответом. Не нужно раскрывать всю карточку.
**Effort:** 2 дня | **Impact:** средний (ускорение просмотра)

### Категория C: Strategic features (2–4 недели)

#### C1. AI-генерация похожих задач из KB
**Job:** C1–C4 (быстро расширить практику)
**Opportunity score:** 21/25
**Рекомендация:** Кнопка "Похожая AI" (уже есть в меню) → открывает Помощника с контекстом задачи → генерирует 3–5 похожих задач с другими числами → "Добавить в Мою базу" / "В ДЗ".
**UX-принцип #5:** AI output → action layer. Каждая сгенерированная задача имеет кнопки "В базу" / "В ДЗ" / "Редактировать".
**Effort:** 2 недели | **Impact:** очень высокий (core AI value prop)

#### C2. Шаблоны ДЗ из KB
**Job:** B2 (собрать ДЗ), E2 (переиспользование)
**Рекомендация:** Сохранить набор задач как "Шаблон ДЗ" (например, "Кинематика — ДЗ на графики v(t)"). При создании нового ДЗ — выбрать шаблон → все задачи подставляются.
**Effort:** 1.5 недели | **Impact:** высокий (retention, повторное использование)

#### C3. Статистика использования задач
**Job:** E1 (контроль качества при масштабе), A3 (сбалансировать)
**Что делают конкуренты:** DIDAK — аналитика по тестам, время выполнения, сложные задания.
**Рекомендация:** Badge на карточке: "Использована 3 раза", "Avg score: 72%". Помогает выбирать задачи осознанно.
**Effort:** 2 недели | **Impact:** средний (долгосрочный retention)

#### C4. Bulk import задач (OCR из PDF/фото)
**Job:** A4 (подготовить материалы), E2 (наполнить базу)
**Рекомендация:** Загрузить PDF → AI извлекает задачи → каждая становится отдельной карточкой в "Моя база". Массовое наполнение KB.
**Effort:** 3–4 недели | **Impact:** высокий (onboarding, наполнение базы)
