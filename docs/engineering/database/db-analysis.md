# 📊 Анализ работы Mini App и структуры данных

## ✅ Ожидаемая структура данных в БД

### Таблица `solutions`

```sql
CREATE TABLE solutions (
  id UUID PRIMARY KEY,
  telegram_chat_id BIGINT,
  telegram_user_id BIGINT,
  user_id UUID,
  problem_text TEXT NOT NULL,
  solution_data JSONB NOT NULL,  -- ⚠️ КЛЮЧЕВОЕ ПОЛЕ
  created_at TIMESTAMPTZ,
  updated_at TIMESTAMPTZ
);
```

### Структура JSONB `solution_data`

```json
{
  "problem": "Текст задачи",
  "solution_steps": [
    {
      "number": 1,
      "title": "Название шага",
      "content": "Описание шага",
      "formula": "LaTeX формула (БЕЗ $ символов)",
      "method": "Описание метода (опционально)"
    }
  ],
  "final_answer": "LaTeX финального ответа",
  "raw_response": "Полный ответ от AI (для отладки)"
}
```

## 🔍 Как данные создаются

### 1. Gemini генерирует ответ (РЕЖИМ 3)

Промпт требует:
```
**Решение:**

**Шаг 1:** [Подробное объяснение + формулы]
$$[промежуточные вычисления]$$

**Шаг 2:** [Подробное объяснение + формулы]
$$[промежуточные вычисления]$$

**Ответ:** [Финальный ответ]
```

### 2. Функция `parseSolutionSteps` (4 уровня fallback)

#### Уровень 1: Структурированные заголовки
Ищет паттерны:
- `### Заголовок`
- `## Заголовок`
- `# Заголовок`
- `**Шаг 1:** Название`
- `1. Название`
- `**Название:**`

Regex: `/(?:#{1,3}\s+(.+)|(?:\*\*)?(?:Шаг\s+)?(\d+)[.):\s]+\s*(.+?)(?:\*\*)?|(?:\*\*)(.+?)(?:\*\*):)/gm`

#### Уровень 2: Простые нумерованные списки
Ищет:
- `1) текст`
- `2. текст`

Regex: `/(?:^|\n)(\d+)[.)]\s+(.+?)(?=\n\d+[.)]|\n\n|$)/gs`

#### Уровень 3: Разбивка по параграфам
Разделение по двойным переводам строк: `\n\s*\n`

#### Уровень 4: Весь текст как один шаг
Fallback на случай, если ничего не сработало.

### 3. Извлечение формул `extractLatexFormulas`

Ищет:
- Display mode: `$$...$$`
- Inline mode: `$...$`

**⚠️ ВАЖНО:**
- Формулы извлекаются БЕЗ `$` символов
- Формулы сохраняются в поле `formula` каждого шага
- Финальный ответ ищется по паттернам: `**Ответ:**`, `**Итог:**`, etc.

### 4. Сохранение в БД

```typescript
const solutionData = {
  problem: problemText,
  solution_steps: parseSolutionSteps(aiResponse),
  final_answer: extractFinalAnswer(aiResponse),
  raw_response: aiResponse
};

await supabase.from('solutions').insert({
  telegram_chat_id,
  telegram_user_id,
  user_id,
  problem_text,
  solution_data: solutionData  // JSONB
});
```

## ⚠️ Потенциальные проблемы

### Проблема 1: Gemini не следует формату

**Симптом:** Gemini возвращает неструктурированный текст

**Пример:**
```
Давай решим эту задачу. Сначала найдем коэффициенты a=1, b=-5, c=6.
Потом вычислим дискриминант D = 25 - 24 = 1. И так далее...
```

**Что произойдет:**
- Fallback Level 3 или 4 активируется
- Создастся 1-2 больших шага вместо пошаговых
- Формулы могут не извлечься, если они не в `$...$` или `$$...$$`

**Решение:**
- Улучшить промпт для Gemini
- Добавить валидацию ответа AI
- Использовать structured output (если Gemini поддерживает)

### Проблема 2: LaTeX формулы в неправильном формате

**Симптом:** Gemini использует формат Markdown или plain text

**Пример:**
```
x^2 + 5x + 6 = 0  (без $ символов)
или
\[x^2 + 5x + 6 = 0\]  (LaTeX display mode, не Markdown)
```

**Что произойдет:**
- `extractLatexFormulas` не найдет формулы
- Поле `formula` будет `null`
- Mini App покажет только текст без красивых формул

**Решение:**
- Убедиться, что промпт явно требует `$...$` или `$$...$$`
- Добавить парсинг альтернативных форматов
- Постобработка: конвертация plain text в LaTeX

### Проблема 3: Вложенные списки

**Симптом:** Gemini создает подпункты

**Пример:**
```
**Шаг 1:** Найдем коэффициенты
  1.1 a = 1
  1.2 b = -5
  1.3 c = 6

**Шаг 2:** Вычислим дискриминант
```

**Что произойдет:**
- Парсер найдет только топ-level шаги ("Шаг 1", "Шаг 2")
- Подпункты войдут в `content` первого шага
- Структура будет корректной, но менее детальной

**Решение:**
- Промпт должен запрещать подпункты
- Текущая реализация: **работает корректно** (подпункты = часть content)

### Проблема 4: Отсутствие финального ответа

**Симптом:** AI не указывает "Ответ:" в конце

**Что произойдет:**
- `extractFinalAnswer` вернет `null`
- Поле `final_answer` будет пустым
- Mini App не покажет финальный ответ на последнем шаге

**Решение:**
- Улучшить regex паттерны для поиска ответа
- Добавить fallback: взять последнюю формулу как ответ

**Шаг 1: Определяем коэффициенты**
В квадратном уравнении $ax^2 + bx + c = 0$ определим коэффициенты:

$$a = 1, \quad b = -5, \quad c = 6$$

Используем формулу дискриминанта для определения количества корней:

$$x_{1,2} = \frac{-b \pm \sqrt{D}}{2a} = \frac{5 \pm 1}{2}$$

Получаем: $x_1 = 3$, $x_2 = 2$

**Ответ:** $x_1 = 3, \quad x_2 = 2$
