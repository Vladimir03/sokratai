# 🐛 Исправление бага в парсинге решений для Telegram Mini App

## Проблема

Функция `parseSolutionSteps` в `telegram-bot/index.ts` использовала **non-greedy** regex паттерн `.+?`, который захватывал только **первую букву** заголовков шагов вместо полного текста.

### Пример проблемы:

**Входной текст:**
```
**Шаг 1: Определяем коэффициенты**
```

**Результат ДО исправления:**
```json
{
  "number": 1,
  "title": "О"  // ❌ Только первая буква!
}
```

**Результат ПОСЛЕ исправления:**
```json
{
  "number": 1,
  "title": "Определяем коэффициенты"  // ✅ Полный текст!
}
```

## Исправление

### Изменения в коде:

**Файл:** `supabase/functions/telegram-bot/index.ts`
**Строка:** 1052

**БЫЛО (баг):**
```javascript
const sectionRegex = /(?:^|\n)(?:#{1,3}\s+(.+)|(?:\*\*)?(?:Шаг\s+)?(\d+)[.):\s]+\s*(.+?)(?:\*\*)?|(?:\*\*)(.+?)(?:\*\*):)/gm;
//                                                                                    ^^^  ^^^
//                                                                               Non-greedy match
```

**СТАЛО (исправлено):**
```javascript
const sectionRegex = /(?:^|\n)(?:#{1,3}\s+(.+)|(?:\*\*)?(?:Шаг\s+)?(\d+)[.):\s]+\s*([^*\n]+)(?:\*\*)?|(?:\*\*)([^*]+)(?:\*\*):)/gm;
//                                                                                    ^^^^^^^  ^^^^^
//                                                                               Greedy match до * или \n
```

### Что изменилось:

1. **Группа 3:** `.+?` → `[^*\n]+`
   - Захватывает все символы **кроме** `*` и `\n` (жадно)
   - Правильно парсит заголовки типа "**Шаг 1: Название шага**"

2. **Группа 4:** `.+?` → `[^*]+`
   - Захватывает все символы **кроме** `*` (жадно)
   - Правильно парсит заголовки типа "**Заголовок:**"
