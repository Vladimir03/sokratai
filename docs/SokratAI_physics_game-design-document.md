# SokratAI Physics — Game Design Document
## Тренажёр формул механики для ЕГЭ и ОГЭ

---

## 1. Концепция

**Одна фраза:** Duolingo для физических формул — короткие сессии с адаптивной сложностью, которые учат не зубрить, а *понимать* формулы через три слоя знания.

**Целевая аудитория:** школьники 9-11 класс, готовящиеся к ОГЭ/ЕГЭ по физике.

**Ключевая метрика успеха:** ученик, прошедший полный цикл по разделу, решает задачи ЕГЭ/ОГЭ без шпаргалки формул.

---

## 2. Структура игровой сессии

### 2.1. Длина сессии: 3-5 минут (один «раунд»)

Один раунд = **10 заданий**. Почему именно 10:
- Достаточно для ощутимого прогресса
- Не утомляет (когнитивная нагрузка управляема)
- Вписывается в перерыв между уроками / поездку в транспорте
- Duolingo-модель доказала эффективность именно такой длины

### 2.2. Анатомия одного раунда

```
┌─────────────────────────────────────────┐
│  Раунд "Кинематика — Уровень 2"        │
│  ══════════════════════════════          │
│  Прогресс: ●●●●●○○○○○  (5/10)          │
│                                         │
│  [Задание текущего типа]                │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │  Варианты ответа / Поле ввода  │    │
│  └─────────────────────────────────┘    │
│                                         │
│  ❤️ ❤️ ❤️ (3 жизни)                      │
│  🔥 Серия: 7 подряд                     │
└─────────────────────────────────────────┘
```

**Механика жизней:**
- 3 жизни на раунд
- Неправильный ответ = -1 жизнь + мгновенное объяснение
- 0 жизней = раунд завершён (но прогресс сохраняется)
- Позволяет ошибаться, но не бесконечно

**После каждого ответа:**
- Правильно → зелёная вспышка + короткая похвала + XP
- Неправильно → красная вспышка + объяснение «почему неверно» + правильный ответ
- Объяснение ВСЕГДА показывается (даже при правильном — но короче)

### 2.3. Экран после раунда

```
┌─────────────────────────────────────────┐
│  Раунд завершён!                        │
│                                         │
│  Правильно: 8/10  (80%)                │
│  XP получено: +45                       │
│  🔥 Серия дней: 3                        │
│                                         │
│  Проблемные формулы:                    │

│  ⚠️ a_цс = v²/R     (слой 3: перепутал)│
│                                         │
│  [Повторить ошибки]  [Следующий раунд]  │
└─────────────────────────────────────────┘
```

## 3. Прогрессия и уровни

### 3.1. Структура прогрессии (на примере Механики)

```
МЕХАНИКА
├── Кинематика
│   ├── Уровень 1: Равномерное движение (3 формулы)
│   │   ├── Слой 3: Структура → Слой 2: Переменные → Слой 1: Ситуации
│   │   └── ✅ Разблокировано
│   ├── Уровень 2: Равноускоренное движение (4 формулы)
│   │   ├── Слой 3 → Слой 2 → Слой 1
│   │   └── ✅ Разблокировано после Уровня 1
│   ├── Уровень 3: Свободное падение (2 формулы)
│   │   └── 🔒 Разблокируется после Уровня 2
│   ├── Уровень 4: Движение по окружности (4 формулы)
│   │   └── 🔒
│   └── БОСС: Смешанные задачи кинематики
│       └── 🔒 Разблокируется после всех уровней
├── Динамика
│   ├── ...
│   └── БОСС: Смешанные задачи динамики
├── Законы сохранения
│   ├── ...
│   └── БОСС
├── Статика и гидростатика
│   ├── ...
│   └── БОСС
├── Колебания и волны
│   ├── ...
│   └── БОСС
└── ФИНАЛЬНЫЙ БОСС: Вся механика вперемешку
```
**Комментарий от Владимира**: К п. 3.1. думаю, бессмысленно гонять по 3 формулы, блоки должны быть крупнее.  Например, вся кинематика сразу. Потом вся динамика. Потом законы сохранения, статика волны. Потом все вперемешку. И возможность для разблокировки”вручную”. Со стороны преподавателя или даже самого ученика. Если человек пришел в середине года, тотему уже надо сразу все повторять, а не по кусочкам. 

### 3.2. Порядок слоёв ВНУТРИ уровня

**Важное решение: начинаем со слоя 3, а не со слоя 1.**

Почему «снизу вверх» (3 → 2 → 1), а не «сверху вниз»:

1. Слой 3 (структура) — самый конкретный. Ученику показывают формулу, он определяет верно/неверно. Когнитивная нагрузка минимальна: формула УЖЕ перед глазами.

2. Слой 2 (переменные) — средний. Формулу уже не показывают, но дают подсказки (список букв). Ученик уже «видел» формулу много раз на слое 3.

3. Слой 1 (выбор формулы) — самый сложный. Нет ни формулы, ни букв — только описание ситуации. Но к этому моменту ученик уже знает формулу наизусть.

```
Один уровень (например, "Равноускоренное движение"):

Фаза A: СЛОЙ 3 — "Узнай формулу"
  Раунд 1: Правда/ложь по 4 формулам (v=v₀+at, s=v₀t+at²/2, s=(v+v₀)/2·t, v²=v₀²+2as)
  Раунд 2: Заполни пробел + Пропорциональное мышление
  Раунд 3: Проверка размерностью
  → Критерий прохождения: 80% правильных

Фаза B: СЛОЙ 2 — "Собери формулу"
  Раунд 4: Собери из кубиков + Лишняя буква
  Раунд 5: Физическая интуиция
  → Критерий прохождения: 80% правильных

Фаза C: СЛОЙ 1 — "Найди формулу"
  Раунд 6: Ситуация → Формула
  Раунд 7: Какой величины не хватает + Дерево решений
  → Критерий прохождения: 80% правильных

Фаза D: МИКС всех слоёв
  Раунд 8: Смешанные задания по всем слоям
  → Критерий прохождения: 80%

→ Уровень пройден! Разблокирован следующий.
```

### 3.3. Босс-раунды

Босс = раунд из 15 заданий, где:
- Формулы из ВСЕХ уровней раздела перемешаны
- Типы заданий случайные (все слои)
- Повышенная сложность: меньше вариантов, больше ввода вручную
- НЕТ жизней — считается % правильных
- Для получения «звезды» нужно ≥ 90%

## 4. Детализация 12 типов заданий

### СЛОЙ 3: Структура формулы

#### 4.1. Правда или ложь (`task_trueOrFalse`)

```
┌──────────────────────────────────────┐
│  Формула верна?                      │
│                                      │
│         v = t / s                    │
│                                      │
│    [✅ ВЕРНО]     [❌ НЕВЕРНО]        │
│                                      │
│  ⏱️ 8 секунд                         │
└──────────────────────────────────────┘
```

**UX-логика:**
- Таймер 8 секунд (создаёт лёгкое давление, тренирует автоматизм)
- 50/50 показываем верные и неверные формулы
- Неверные формулы = реальные ошибки учеников (из `commonMistakes`):
  - Перевёрнутая дробь: `v = t/s` вместо `v = s/t`
  - Пропущен множитель: `E_к = mv` вместо `mv²/2`
  - Лишнее/недостающее: `h = gt²` вместо `gt²/2`
- После ответа — объяснение через размерность или пропорцию

**Генерация неверных формул (алгоритм):**
```
mutateFormula(formula):
  выбрать случайную мутацию:
  1. swap_fraction: числитель ↔ знаменатель
  2. drop_coefficient: убрать 2, π, или другой коэфф.
  3. wrong_power: v² → v или v → v²
  4. swap_variable: заменить одну переменную на похожую (m↔M, v↔u, s↔S)
  5. add_extra: добавить лишнюю переменную
```

#### 4.2. Заполни пробел (`task_fillTheGap`)

```
┌──────────────────────────────────────┐
│  Заполни пробел:                     │
│                                      │
│       E_к = m · ___ / 2              │
│                                      │
│  [v]  [v²]  [a]  [g·h]              │
│                                      │
└──────────────────────────────────────┘
```

**UX-логика:**
- Показываем формулу с одним пропуском
- 4 варианта ответа (1 верный + 3 дистрактора)
- Дистракторы = переменные из СВЯЗАННЫХ формул (из `relatedFormulas`)
- Пропуск может быть: в числителе, знаменателе, коэффициенте, показателе степени
- На высоких уровнях: свободный ввод вместо вариантов

#### 4.3. Проверка размерностью (`task_dimensionCheck`)

```
┌──────────────────────────────────────┐
│  Проверь формулу размерностью:       │
│                                      │
│       a = v · t                      │
│                                      │
│  Ускорение = [м/с²]                  │
│  Правая часть: [м/с] · [с] = [м]    │
│                                      │
│  ❌ [м] ≠ [м/с²] — формула НЕВЕРНА! │
│                                      │
│  Что правильно?                      │
│  [a = v/t]  [a = v·t²]  [a = v²/t]  │
│                                      │
└──────────────────────────────────────┘
```

**UX-логика:**
- Двухшаговое задание: сначала определить верно/неверно, потом исправить
- Показываем размерности «пошагово» (обучающий момент)
- На начальном уровне: подсказываем единицы всех переменных
- На продвинутом: ученик сам определяет единицы

#### 4.4. Напиши формулу (`task_writeFromMemory`)

```
┌──────────────────────────────────────┐
│  Напиши формулу:                     │
│                                      │
│  "Кинетическая энергия"              │
│                                      │
│  ┌────────────────────────────────┐  │
│  │  E_к = |                       │  │
│  └────────────────────────────────┘  │
│                                      │
│  💡 Подсказка (−50% XP)              │
│                                      │
└──────────────────────────────────────┘
```

**UX-логика:**
- Самый сложный тип слоя 3 — чистый recall
- Поле ввода с распознаванием формул (принимаем разные форматы: `mv^2/2`, `mv²/2`, `(mv²)/2`)
- Кнопка подсказки: показывает переменные, но вычитает XP
- Появляется только после прохождения предыдущих типов по этой формуле

### СЛОЙ 2: Переменные

#### 4.5. Собери формулу (`task_buildFormula`)

```
┌──────────────────────────────────────┐
│  Собери: "Закон всемирного тяготения"│
│                                      │
│  Доступные элементы:                 │
│  [G] [m₁] [m₂] [r] [r²] [μ] [v]    │
│                                      │
│  Формула: F = ___________            │
│                                      │
│  Зона числителя: [перетащи сюда]     │
│  ─────────────────────────────────   │
│  Зона знаменателя: [перетащи сюда]   │
│                                      │
└──────────────────────────────────────┘
```

**UX-логика:**
- Drag-and-drop интерфейс (на мобильном — tap для выбора)
- Даём 5-7 «кубиков»: правильные переменные + 2-3 лишних из смежных формул
- Отдельные зоны для числителя и знаменателя (тренирует слой 2 И слой 3 одновременно)
- Визуальная обратная связь: при сборке проверяем размерность в реальном времени

#### 4.6. Лишняя буква (`task_whatBelongs`)

```
┌──────────────────────────────────────┐
│  Найди ошибку:                       │
│                                      │
│       E_к = m·g·h / 2               │
│                                      │
│  Что лишнее / что не так?            │
│  [g лишняя]  [h лишняя]             │
│  [нет /2]    [нужно v² вместо g·h]   │
│                                      │
└──────────────────────────────────────┘
```

**UX-логика:**
- Показываем «почти правильную» формулу с подменой
- Подмены: переменная из похожей формулы (mgh вместо mv² — перепутали потенциальную с кинетической)
- Тренирует различение похожих формул
- Пары-ловушки из базы: E_к=mv²/2 vs E_п=mgh, p=mv vs F=ma, P=F/S vs P=A/t

#### 4.7. Физическая интуиция (`task_physicalIntuition`)

```
┌──────────────────────────────────────┐
│  Кинетическая энергия                │
│                                      │
│  Если массу увеличить в 2 раза,      │
│  а скорость уменьшить в 2 раза,      │
│  как изменится E_к?                  │
│                                      │
│  [Увеличится в 2 раза]               │
│  [Не изменится]                      │
│  [Уменьшится в 2 раза]  ← верно     │
│  [Уменьшится в 4 раза]              │
│                                      │
│  💡 E_к = mv²/2                      │
│     m×2, v÷2 → 2m·(v/2)²/2         │
│     = 2m·v²/8 = mv²/4 = E_к/2      │
└──────────────────────────────────────┘
```

**UX-логика:**
- Комбинированное изменение: меняем 2 переменные одновременно
- Проверяет, помнит ли ученик СТЕПЕНИ в формуле (v² — ключевой момент)
- Формулу НЕ показываем в условии (но показываем в объяснении)
- Уровни сложности:
  - Простой: одна переменная меняется
  - Средний: две переменные, одна линейная
  - Сложный: две переменные, одна квадратичная

### СЛОЙ 1: Существование формулы

#### 4.8. Ситуация → Формула (`task_situationToFormula`)

```
┌──────────────────────────────────────┐
│  Автомобиль тормозит.                │
│  Известно: начальная скорость        │
│  и тормозной путь.                   │
│  Время НЕ дано.                      │
│                                      │
│  Какую формулу использовать?         │
│                                      │
│  [v = v₀ + at]                       │
│  [s = v₀t + at²/2]                  │
│  [v² = v₀² + 2as]     ← верно       │
│  [F = ma]                            │
│                                      │
└──────────────────────────────────────┘
```

**Комментарий от Владимира**: 4.8 в предложенном виде будет непонятно ученику и не всегда корректно. Мысли по модификации: взять реальные задачи тип 2, 3, 4, 22 {для механики} из банка ФИПИ/решу ЕГЭ, и задавать вопрос типа "какая/какие формулы необходимы и достаточны для решения". Это будет прямо сложный тип заданий, но может принести пользу.

**UX-логика:**
- Описание задачи БЕЗ чисел (фокус на навигации, не на вычислениях)
- 4 варианта: верная формула + 3 дистрактора из того же раздела
- Подсказка-триггер выделена: «Время НЕ дано» → формула без t
- Данные для генерации берутся из поля `whenToUse`
- Дистракторы — формулы из `relatedFormulas`

#### 4.9. Какой величины не хватает? (`task_whatsMissing`)

```
┌──────────────────────────────────────┐
│  Дано:                               │
│    • масса тела (m)                  │
│    • высота (h)                      │
│                                      │
│  Какую величину можно найти?         │
│                                      │
│  [Кинетическую энергию E_к]          │
│  [Потенциальную энергию E_п] ← верно │
│  [Импульс p]                         │
│  [Ускорение a]                       │
│                                      │
│  💡 E_п = mgh (нужно ещё g, но оно  │
│     известно: g ≈ 10 м/с²)          │
└──────────────────────────────────────┘
```

**UX-логика:**
- Обратная задача: не «какая формула», а «что можно вычислить»
- Учитываем, что g — «бесплатная» константа (ученик должен это помнить)
- Тренирует «навигацию в пространстве формул»

#### 4.10. Дерево решений (`task_formulaMap`)

```
┌──────────────────────────────────────┐
│  Помоги выбрать формулу:             │
│                                      │
│  Движение равномерное или            │
│  с ускорением?                       │
│                                      │
│  [Равномерное]    [С ускорением]      │
│       │                  │           │
│       ▼                  ▼           │
│   s = v·t       Дано ли время?       │
│                 [Да]      [Нет]      │
│                  │          │        │
│                  ▼          ▼        │
│             s=v₀t+at²/2  v²=v₀²+2as │
│                                      │
└──────────────────────────────────────┘
```

**UX-логика:**
- Интерактивное дерево: ученик отвечает на вопросы-развилки
- Каждый путь ведёт к конкретной формуле
- Визуализируется как flowchart
- Со временем ученик интернализирует дерево и перестаёт в нём нуждаться
- Деревья строятся из полей `whenToUse` и `relatedFormulas`

### СЛОЙ 4 (бонусный): Применение

#### 4.11. Реши задачу (`task_fullProblem`)

```
┌──────────────────────────────────────┐
│  Задача:                             │
│  Тело массой 2 кг скользит по       │
│  горизонтальной поверхности со       │
│  скоростью 10 м/с и останавливается.│
│  Коэффициент трения μ = 0.5.        │
│  Найти тормозной путь.              │
│                                      │
│  Шаг 1: Какую формулу использовать?  │
│  [v² = v₀² + 2as]  ← ученик выбирает│
│                                      │
│  Шаг 2: Чему равно ускорение?        │
│  a = μg = [____]  м/с²              │
│                                      │
│  Шаг 3: Подставь и найди s:          │
│  s = [____] м                        │
│                                      │
└──────────────────────────────────────┘
```

**UX-логика:**
- Пошаговое решение (не «введи финальный ответ», а проведение через логику)
- Шаг 1 = слой 1 (выбор формулы)
- Шаг 2 = промежуточное вычисление
- Шаг 3 = финальный ответ
- Если ошибка на шаге 1 — не пускает дальше, объясняет выбор формулы
- На высоком уровне: все шаги сразу (ввод финального ответа)

## 5. Мотивационные механики

### 5.1. XP и уровень ученика

| Действие | XP |
|---|---|
| Правильный ответ | +5 |
| Правильный с первой попытки (без подсказки) | +8 |
| Серия 5 правильных подряд | +15 бонус |
| Серия 10 правильных подряд | +30 бонус |
| Раунд без ошибок | +25 бонус |
| Новый уровень пройден | +50 |
| Босс пройден | +100 |

**Уровни ученика** (не путать с уровнями контента):
- Новичок: 0-200 XP
- Ученик: 200-500 XP
- Знаток: 500-1200 XP
- Мастер: 1200-2500 XP
- Физик: 2500+ XP

### 5.2. Серия дней (streak)

- Ежедневная активность (хотя бы 1 раунд) увеличивает серию
- Серия 7 дней → бонус +50 XP
- Серия 30 дней → бонус +200 XP
- Пропуск дня → серия сбрасывается (но можно «заморозить» 1 раз в неделю)
- Визуально: огонёк 🔥 с числом дней

### 5.3. Карта прогресса

Визуальная карта раздела (как в Duolingo / Mario):
- Каждый уровень = точка на карте
- Пройденные = зелёные, текущий = пульсирующий, заблокированные = серые
- Боссы = крупные точки с иконкой
- Показывает общий прогресс по разделу (% пройден)

### 5.4. «Формула дня»

- Каждый день на главном экране — одна случайная формула с коротким фактом
- «Знал ли ты, что v² = v₀² + 2as выводится из двух других формул кинематики?»
- Создаёт точку контакта даже в дни, когда ученик не играет

### 6.2. Алгоритм выбора задания в раунде

```
selectNextQuestion():

  1. ПРИОРИТЕТ 1: Формулы с просроченным review (srNextReview < сегодня)
     → Выбрать тип задания по weakLayer этой формулы

  2. ПРИОРИТЕТ 2: Формулы с низким mastery (< 0.5) в текущем уровне
     → Задание по самому слабому слою

  3. ПРИОРИТЕТ 3: Новые формулы (ещё не встречались)
     → Начать со слоя 3 (самый лёгкий вход)

  4. ПРИОРИТЕТ 4: Укрепление (mastery 0.5-0.8)
     → Смешанные задания

  5. ПРИОРИТЕТ 5: Поддержание (mastery > 0.8)
     → Редкие проверки, повышенная сложность

  В каждом раунде из 10 заданий:
  - 3-4 задания из приоритетов 1-2 (слабые места)
  - 2-3 задания из приоритета 3 (новый материал)
  - 2-3 задания из приоритета 4 (укрепление)
  - 1 задание из приоритета 5 (поддержание)
```

### 6.3. Обновление mastery после ответа

```
updateMastery(formula, layer, isCorrect, responseTime):

  // Базовое обновление
  if isCorrect:
    layer.mastery += 0.1 * (1 - layer.mastery)  // приближаемся к 1.0
    if responseTime < 5 секунд:
      layer.mastery += 0.05  // бонус за скорость = уверенность
  else:
    layer.mastery -= 0.15  // штраф за ошибку больше, чем награда

  // Spaced repetition (упрощённый SM-2)
  if isCorrect:
    srInterval *= srEaseFactor
    srEaseFactor += 0.1
  else:
    srInterval = 1  // сброс интервала
    srEaseFactor = max(1.3, srEaseFactor - 0.2)

  srNextReview = сегодня + srInterval
```

### 6.4. Генерация дистракторов

```
generateDistractors(formula, taskType, count=3):

  pool = []

  // 1. Связанные формулы (самые опасные путаницы)
  pool.push(...formula.relatedFormulas)

  // 2. Формулы того же раздела
  pool.push(...sameSection.filter(f => f.id !== formula.id))

  // 3. Формулы, с которыми ученик ЛИЧНО путает (из confusedWith)
  pool.prioritize(student.formulaState[formula.id].confusedWith)

  // Выбрать count дистракторов, приоритизируя confusedWith
  return pool.slice(0, count)
```

## 7. Обратная связь (самая важная часть)

### 7.1. Принцип: объяснение ВСЕГДА, не только при ошибке

После КАЖДОГО ответа показываем микро-объяснение:

**При правильном ответе (короткое, 1 строка):**
> ✅ Верно! v = s/t — скорость это расстояние делить на время.

**При неправильном ответе (развёрнутое, 2-4 строки):**
> ❌ Не совсем. Ты выбрал s = v₀t + at²/2, но в задаче НЕТ времени.
>
> Когда нет времени — нужна формула v² = v₀² + 2as.
> Запомни: нет t в условии → ищи формулу без t.

### 7.2. Типы объяснений по слоям

| Слой | Тип объяснения |
|---|---|
| 3 (структура) | Размерностная проверка: «[м/с²] ≠ [м·с], значит формула неверна» |
| 3 (структура) | Пропорция: «Больше площадь → меньше давление → S в знаменателе» |
| 2 (переменные) | Смысловая связь: «E_к — энергия ДВИЖЕНИЯ, а h — это высота. В кинетической нет h.» |
| 1 (навигация) | Триггер: «Нет времени в задаче? → v² = v₀² + 2as, единственная формула кинематики без t» |

### 7.3. «Карточка формулы» (по нажатию)

В любой момент ученик может нажать на формулу и увидеть полную карточку:

```
┌──────────────────────────────────────┐
│  v² = v₀² + 2as                     │
│  "Формула без времени"               │
│                                      │
│  📖 Смысл: Связывает скорости,       │
│  ускорение и перемещение без t.      │
│                                      │
│  📐 Размерность:                     │
│  [м²/с²] = [м²/с²] + [м/с²]·[м]   │

│  s=v₀t+at²/2 (исключить t)          │

│  ⚠️ Используй когда:                 │
│  • В задаче НЕТ времени             │
│  • Тормозной путь                    │
│  • Свободное падение с высоты h      │

```
