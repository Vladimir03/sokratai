# UX Audit — Сократ AI

**Дата:** 2026-04-02
**Стадия продукта:** Paid pilot (2 репетитора, 7 учеников)
**Аудитор:** Claude (design:design-critique framework)
**Контекст:** B2B2C платформа для репетиторов физики ЕГЭ/ОГЭ. Три роли: репетитор, ученик (16-18 лет), родитель.

---

## 1. Лендинг (sokratai.ru)

### Overall Impression
Лендинг читается как «студенческий проект на Tailwind» — градиентный hero, emoji в заголовках, зелёный секционный блок без визуальной связи с фиолетовым hero. Ощущение дизайн-слопа подтверждается: нет единого visual language, цвета случайны, типографика generic.

### First Impression (2 секунды)
- **Что видит глаз первым:** Огромный заголовок «AI-помощник по школьным предметам» на фиолетово-синем градиенте
- **Эмоция:** «Ещё один AI-чат для школьников» — не дифференцируется от ChatGPT / YandexGPT
- **Проблема:** Лендинг адресован ученику, но **primary buyer — репетитор**. Нет ни одного экрана «для репетиторов» в hero

### Usability

| Находка | Severity | Рекомендация |
|---------|----------|--------------|
| «Открыть в Telegram» — primary CTA, но Telegram заблокирован в РФ | 🔴 Critical | Поменять приоритет: «Открыть в браузере» = primary, Telegram = secondary |
| Навигация: 8 пунктов + Войти = перегружена | 🟡 Moderate | Сократить до 4-5: Как работает, Для репетиторов, Цены, Войти |
| Нет секции «Для репетиторов» на лендинге | 🔴 Critical | Добавить блок с value prop для B2B: «Собирайте ДЗ за 5 минут» |
| «Войти» — dropdown без видимого состояния | 🟡 Moderate | Прямая кнопка «Войти» без dropdown, или две кнопки: Ученик / Репетитор |
| Лендинг на «ты» → приложение на «вы» | 🟢 Minor | Выбрать один tone of voice |

### Visual Hierarchy
- **Цвета хаотичны:** hero = фиолетовый градиент (#4A3B8C → #6B5B9E), секция «Как работает» = яркий зелёный (#22C55E), cards = белый, текст = серый/чёрный. Нет связующей палитры
- **Emoji вместо иконок:** 🎯 ✨ 💡 🧩 — создаёт ощущение «наспех собранного», не premium-продукта для репетиторов с hourly rate 3000-4000₽
- **Типографика:** Inter или system font, нет hierarchy между heading/body/caption
- **Whitespace:** Секция «Понимай предметы сам» — огромные пустые пространства без контента, выглядит незаконченной

### Consistency

| Элемент | Проблема | Рекомендация |
|---------|----------|--------------|
| Цвет CTA | «Открыть в Telegram» = синий, «Открыть в браузере» = серый. Оба primary? | Один primary цвет, один secondary |
| Градиент hero | Фиолетовый → зелёный → белый → зелёный. 4 разных фона | Единая палитра: 1 primary, 1 accent, нейтральные |
| Блоки «Способ 1/2» | Карточки на зелёном фоне — карточки внутри секции — типичный design slop | Убрать вложенные карточки, использовать плоский layout |
| Отзывы | Italic курсив на зелёном фоне — плохой контраст | Белый фон, нормальный размер, аватар |

### Accessibility
- **Контраст:** Серый текст подзаголовков на фиолетовом hero — FAIL (WCAG AA)
- **Touch targets:** CTA кнопки OK (48px+)
- **Текст:** Базовый body text читаемый, но italicized quotes на зелёном фоне — плохо

## 2. Login Page

### Overall Impression
Минималистично, но безлико. Полупрозрачная карточка на том же фиолетовом градиенте. Нет логотипа, нет ощущения бренда. Выглядит как default auth template.

### Usability

| Находка | Severity | Рекомендация |
|---------|----------|--------------|
| «Войти через Telegram (нужен VPN)» — антипаттерн | 🔴 Critical | Убрать Telegram login с основного экрана или сделать tertiary |
| «Вы репетитор? Вход для репетиторов» — зачем отдельный? | 🟡 Moderate | Единый вход, роль определяется после auth (уже так в коде) |
| Нет логотипа на login page | 🟡 Moderate | Добавить логотип + tagline — это первый touchpoint |
| Автозаполнение работает — хорошо | 🟢 OK | — |

### Visual Hierarchy
- **Нет бренда:** ни логотипа, ни названия «Сократ» на экране входа
- **Два CTA конкурируют:** «Войти по email» (тёмная) и «Войти через Telegram» (синяя) — оба prominent

## 3. Tutor Dashboard

### Overall Impression
Самый зрелый экран. Метрические карточки (Ученики / К оплате / Получено) понятны и полезны. Но dashboard не отвечает на главный job: «Что мне делать прямо сейчас?»

### Usability

| Находка | Severity | Рекомендация |
|---------|----------|--------------|
| «Требуют внимания» — список неактивных учеников без actionable context | 🟡 Moderate | Добавить: «Не сдал ДЗ», «Застрял в guided chat», «Нет активности 3 дня» — конкретные actions |
| Нет блока «Непроверенные ДЗ» на dashboard | 🔴 Critical | Dashboard должен показывать pending reviews — core job R1 |
| «Добавить оплату» — рядом с «Добавить ученика», равный вес | 🟢 Minor | «Добавить ученика» = primary (зелёная), остальное = text links |
| Навигация: 7 пунктов + Ещё = перегружена | 🟡 Moderate | Объединить: Домашки + База знаний → «Задачи». Расписание + Ученики → «Ученики» |

### Visual Hierarchy
- **Хорошо:** карточки с числами (7 учеников, 2 500₽, 30 500₽) — immediate value
- **Плохо:** «Требуют внимания» = одинаковые строки «Неактивен — Нет активности» — нет иерархии, нет urgency
- **Навигация:** Emoji в tabs (📋 📅 👥 📝 🤖) — выглядит игрушечно для профессионального инструмента

### Consistency

| Элемент | Проблема | Рекомендация |
|---------|----------|--------------|
| Header | «Сократ AI для репетиторов» — текст+emoji логотип, не графический | Нужен proper logo |
| Цвет primary | Зелёная кнопка «Добавить ученика» vs тёмные карточки — нет единого primary | Зафиксировать: primary = dark navy (#1E293B), accent = green |
| Typography | Заголовок «Добро пожаловать, Владимир!» с emoji 👋 — informal для prof tool | Убрать emoji, оставить приветствие |

## 4. Tutor Homework List

### Overall Impression
Grid карточек ДЗ — чистый, функциональный. Но все карточки выглядят одинаково, нет visual differentiation по статусу/прогрессу.

### Usability

| Находка | Severity | Рекомендация |
|---------|----------|--------------|
| Все карточки визуально одинаковы (зелёный badge «Активное») | 🟡 Moderate | Разные цвета/визуал для: не начато, в процессе, проверено, просрочено |
| «ДЗ 02.04» — нет имени ученика на карточке | 🟡 Moderate | Показать имя ученика под названием ДЗ |
| «Ш 95%» — непонятная аббревиатура | 🟡 Moderate | Написать «Балл: 95%» или прогресс-бар |
| «⚡ Физика» — emoji для предметов | 🟢 Minor | Заменить на цветовой dot или иконку |

### Visual Hierarchy
- **Grid 3 колонки — хорошо** для обзора
- **Нет visual weight** на ДЗ, требующих внимания (0/1 сдано, без балла)
- **Badge «Активное»** — одинаковый зелёный для всех, не несёт информации

## Сквозные проблемы (Cross-screen)

### 1. Отсутствие единого бренда
- **Tutor nav:** «Сократ AI для репетиторов» (emoji логотип, белый фон, серая навигация)
- **Student nav:** «Сократ AI» (фиолетовый accent, emoji, другой стиль)
- **Landing:** Третий стиль — фиолетовый градиент
- **Login:** Вообще без бренда
- **Итог:** 4 разных визуальных identity в одном продукте

### 2. Emoji-driven design
Почти каждый заголовок и элемент содержит emoji: 👋 ⚡ 📋 📅 👥 📝 🤖 🎯 ✨ 💡 🧩. Это создаёт ощущение «наспех собранного» вместо «профессионального инструмента». Репетитор с hourly rate 3000₽ ожидает Teachworks-уровня, а получает Discord-вайб.

### 3. Inconsistent color system
- Landing: фиолетовый + зелёный + белый
- Tutor: белый + navy buttons + green accent + orange money
- Student: фиолетовый accent + green progress + navy buttons
- KB: бежевый header + green socrat brand
- **Нет единой палитры.** Каждый экран как отдельное приложение.

### 4. Typography = generic
System font или Inter на всех экранах. Нет hierarchy (heading weights, sizes). Нет шрифта с характером — буквально default Tailwind.

### 5. Cards inside cards (design slop marker)
Лендинг: карточки «Способ 1/2» внутри зелёной секции. Homework list: badge внутри карточки. Множество уровней вложенности.

## What Works Well

1. **Tutor dashboard метрики** — «7 учеников, 2500₽ к оплате, 30500₽ получено» — сразу понятна ценность
2. **Knowledge Base** — лучшая структура: каталог + своя база, темы + подтемы, поиск
3. **AI Assistant: три карточки-действия** — structured tasks вместо generic чата, правильный UX pattern
4. **Student chat quick actions** — «Покажи решение / Дай подсказку / Объясни / Построй график» — снижают барьер
5. **TaskStepper** (1-10 circles) в guided homework — визуальный прогресс, свободный порядок
6. **Два поля ввода** (Ответ vs Обсуждение) — решает реальную проблему путаницы

## Priority Recommendations (Top 10)

### P0 — Сделать за 1-2 дня (quick wins с максимальным эффектом)

1. **Зафиксировать color tokens** — выбрать 1 primary (dark navy #1E293B), 1 accent (socrat green #1B6B4A), 1 warning (amber), 1 success (emerald), нейтральные (slate). Применить ВЕЗДЕ. Убрать random фиолетовый, ярко-зелёный, бежевый.

2. **Убрать все emoji из navigation и headings** — заменить на Lucide icons или вообще ничего. Emoji допустимы только в контенте (отзывы, сообщения), не в UI chrome.

3. **Сменить шрифт** — c Inter/system на Golos Text (российский, бесплатный, modern, хорошие кириллические начертания) или Onest. Один font family, 3-4 weight (400, 500, 600, 700).

4. **Убрать debug info из production** — «4/10 ⚡ Отличное (150ms)» в student chat, «Раздел в разработке» placeholder — убрать.

5. **Landing: перевернуть CTA** — «Открыть в браузере» = primary green button, «Telegram» = secondary/text. Telegram заблокирован у целевой аудитории.

### P1 — Сделать на следующей неделе

6. **Student homework cards: добавить прогресс** — progress bar (3/10), deadline badge, streak counter. Это core мотивация для 16-18 летнего.

7. **Tutor dashboard: добавить «Непроверенные ДЗ»** — блок с pending reviews. Это Job R1 — главная работа репетитора.

8. **Единый nav стиль** — одинаковый header component для tutor и student. Разные items, но один visual frame.

9. **Landing: добавить «Для репетиторов» секцию** — hero для B2B buyer: «Собирайте ДЗ за 5 мин, AI проверяет, родители видят прогресс».

10. **Guided workspace: убрать пустое пространство** — при пустом чате показать onboarding hint вместо void.
