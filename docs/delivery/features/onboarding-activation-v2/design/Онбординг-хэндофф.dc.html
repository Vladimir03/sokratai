Онбординг-активация · хэндофф репетитор → ученик
Сократ AI · hi-fi по дизайн-системе · добавил по имени → собрал ДЗ → подключил → ученик вошёл по ссылке → регистрация → первое ДЗ → возврат
Контекст и решения
Модель входа ученика
  1. Первый вход — личная ссылка/QR минтит сессию. Регистрации на входе нет.
  2. Повседневно — персистентная сессия: открыл sokratai.ru — уже внутри.
  3. Регистрация до домашки — после «ты подключён», перед заданием: email (подставлен от репетитора) + пароль. Причина — не потерять прогресс.
  4. Новое устройство — вход: email + пароль или «войти по коду» (OTP на почту).
  5. Нет email и пароля — вход только по новой ссылке от репетитора.
Инварианты
  • Добавление по имени — контакт опционален
  • Гейт «подключить» — одноразовый, до первого ДЗ
  • Беспарольный вход по ссылке + персистентная сессия
  • Регистрация — до первой домашки (email + пароль); вход по ссылке остаётся беспарольным
  • Прогресс ДЗ автосохраняется по токену — не теряется, даже если не закончил
  • Только email, без телефона
Решения этой версии
Ссылка — первый способ Регистрация до ДЗ — обязательно Email без верификации (пока) Модалки в кабинете

Одна инструкция для Claude Code
Реализуй онбординг-флоу «хэндофф репетитор→ученик» в репозитории Vladimir03/sokratai (React + Vite + Tailwind + shadcn). Импортируй colors_and_type.css и шрифт Golos Text; все цвета — через токены --sokrat-*. Репетитор: data-sokrat-mode="tutor" (десктоп, плотно, 36–40px). Ученик: data-sokrat-mode="student" (mobile, 44/52px, инпуты ≥16px). Экраны репетитора (модалки в кабинете): (1) AddStudentDialog — обязательно только имя, «Дополнительно» свёрнуто, добавление списком имён, плашка «без контакта — это нормально», один primary; (2) ConnectDialog — ссылка первым (поле + «Скопировать»), QR вторым, email/Telegram третьим (бэкенд сам шлёт приглашение + первое ДЗ), гейт одноразовый; (3) StudentProfile с чипом «не подключился» + nudge «добавьте email/Telegram». Экраны ученика: (4) /join → сессия по ссылке без пароля, «ты подключён» + превью задания + «Продолжить»; (5) Register — ОБЯЗАТЕЛЬНЫЙ экран ДО домашки: email (подставлен от репетитора, если есть) + пароль, причина «чтобы не потерять прогресс», без «Позже», без верификации почты; (6) Problem — сразу в задаче, KaTeX, HintLadder, SAnswerInput, прогресс автосохраняется на сервере по токену; (7) /login — email + пароль или OTP на email, «забыли пароль». Инварианты: вход по ссылке беспарольный + персистентная сессия; регистрация ДО первой домашки (email + пароль); прогресс ДЗ АВТОСОХРАНЯЕТСЯ по токену на сервере (не теряется, даже если ученик не закончил); только email; один primary-CTA; sentence case; Lucide; без эмодзи; на онбординг-экранах ученика без градиентов и теней. Верстай строго по дереву компонентов и токенам выше. Позиционирование: Сократ — AI-ПОМОЩНИК РЕПЕТИТОРА (не замена) — проверяет ДЗ и ведёт ученика наводящими вопросами, решения за репетитором. НЕ ПОКРЫТО этим дизайном, спроектировать при реализации в той же системе: (а) конструктор ДЗ между «собрать ДЗ» и «подключить» — выбор задач из базы + свои, назначение ученику/группе, отправка; (б) состояние «ученик подключился» у репетитора — ростер/карточка «не подключился»→«подключён», снять nudge, показать первую активность; (в) экран задачи на реальном контенте — KaTeX-условие, HintLadder, проверка ответа, AI-вердикт; плюс OTP-ввод кода («войти по коду»), подтверждения отправки на email/в Telegram, флоу «добавить списком имён», состояния загрузки/ошибок/повтора, deep-link /join/:token + персистентная сессия + уведомления о новом ДЗ.
Не покрыто дизайном — учесть при разработке
Прототип ведёт по «счастливому пути». Эти куски намеренно за кадром — их нужно спроектировать при разработке в той же дизайн-системе, прежде чем флоу заработает целиком.
3 главных пробела
1
Конструктор ДЗ — за кадром между шагом «Собрать ДЗ» и «Подключить». Экран: выбор задач из базы + свои задачи, назначение ученику/группе, отправка. В прототипе шаг есть в чеклисте, но экран не нарисован.
2
«Ученик подключился» у репетитора — ростер и карточка переходят «не подключился» → «подключён»: снимается чип и nudge, появляется первая активность ученика.
3
Реальное первое ДЗ — экран задачи на настоящем контенте: KaTeX-условие, HintLadder (Сократовы подсказки), проверка ответа и AI-вердикт; что из этого видит репетитор.
Вторично — для полноты
—OTP «войти по коду»: экран ввода кода (показан только вход по паролю)
—Подтверждения отправки на email / в Telegram (состояние «отправлено»)
—«Добавить списком имён» — сам флоу массового добавления
—Состояния: загрузка, ошибки, пустые, повтор отправки ссылки
—Бэкенд-механика: deep-link /join/:token, персистентная сессия, уведомления о новом ДЗ
