Mid-fi экраны для дизайна (Claude Design) и разработки · обновлено 2026-06-26 · регистрация обязательна ДО задачи, только email, автосейв прогресса по токену

Модель входа ученика (итог)

  1. Первый вход — одноразовая личная ссылка/QR минтит сессию (беспарольно).
  2. «Вы подключены» + превью задания.
  3. Регистрация — обязательна ДО задачи. email (подставлен репетитором, если реальный) + пароль. Без «Позже», без верификации. Причина — «чтобы не потерять прогресс».
  4. Задача. Прогресс автосохраняется по токену — не теряется, даже если ученик бросил на полпути.
  5. Повседневно — персистентная сессия. Новое устройство — email + пароль или «войти по коду» (OTP на email).
Только email (без телефона). Регистрация = доустановка пароля поверх активной сессии; ссылку не хард-блокируем. Экран «сохрани доступ» после ДЗ убран (рега уже на входе).

4. Поток

Завёл по имени → собрал ДЗ → подключил (QR/ссылка) → ученик: вход по ссылке → «подключён» + превью → регистрация (email+пароль) → задача → готово
Cross-side «ага» репетитора — когда он видит в «последних действиях», что ученик сам прошёл ДЗ. Гейт «подключить» — одноразовый, до подключения; дальше отправка ДЗ молча.

5. Открытые решения

  1. Тайминг регистрации — решено: обязательна ДО задачи, без «Позже».
  2. Верификация email — решено: пока без; включить позже при росте.
  3. QR vs ссылка — что первым по умолчанию (онлайн-сегмент чаще шлёт ссылку). Открыто.
