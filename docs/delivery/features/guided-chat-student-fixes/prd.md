## 1. Job Context

| Участник | Core Job | Sub-job |
|---|---|---|
| Ученик (B2C) | S1 — Самостоятельно подготовиться к экзамену | S1-1 — решать задачи с AI-разбором, S1-2 — получать feedback без стресса |
| Репетитор (B2B) | R1 — Автоматическая проверка ДЗ | R1-3 — доверять AI-фидбеку, R1-4 — давать ПОС ученику |

**Wedge alignment:** Guided chat — ключевой продуктовый инструмент пилота. Баги в нём снижают доверие к AI-проверке и напрямую угрожают конверсии пилота в платный (15 апреля).

**Pilot impact:** Оба сигнала пришли в период активного пилота. Bug 1 — фактический блокер workflow (ученик не может досдать задачи). Bug 2 — недоверие к системе («говорит ЕГЭ, а я ОГЭ»).

## 2. Problem

### Bug 1 — Ранняя завершённость треда (КРИТИЧЕСКИЙ)

**Текущее поведение:**
Система позволяет ученику решать задачи в **любом порядке** (введено ранее). Однако функция `performTaskAdvance` в backend определяет «следующую задачу» как `sortedOrders[currentIdx + 1]` — следующую по порядку номера. Если ученик решает последнюю задачу **по номеру** (например, #5 из 5, пропустив #2, #3), функция получает `nextOrder = null` и переводит тред в статус `completed`, хотя #2 и #3 ещё активны.

**Последствие:**
- Backend: все `check`/`hint` запросы к треду возвращают `400 ALREADY_COMPLETED`.
- Frontend: `threadStatus === 'completed'` → `GuidedChatInput` задизейблен на всех задачах, включая нерешённые.
- Ученик видит итоговый экран с «Вы решили X из N задач» (не все), но ввод заблокирован.

**Боль:** Ученик не может сдать оставшиеся задачи. Репетитор получает неполные результаты. Доверие к системе падает.

### Bug 2 — Хардкод «ЕГЭ» в UI (MINOR)

**Текущее поведение:**
Для задач с `check_format = 'detailed_solution'` в `GuidedHomeworkWorkspace.tsx` показывается amber-баннер: *«Задача с развёрнутым решением — покажи ход решения, как на ЕГЭ. Без хода решения получишь 0 баллов»*. Текст также передаётся в AI-промпт: *«Мотивируй это подготовкой к ЕГЭ»*. Слово «ЕГЭ» хардкодировано.

**Боль:** Ученики ОГЭ и других экзаменов получают текст с ЕГЭ. Смущает и снижает доверие к платформе. Елена готовит учеников к ОГЭ.

## 3. Solution

### Bug 1 — Fix `performTaskAdvance`

Изменить логику нахождения «следующей задачи»: вместо `sortedOrders[currentIdx + 1]` искать **первую незавершённую** (status = `'active'`) task_state после завершения текущей. Тред помечается `completed` только когда **все** task_states переведены в `completed`.

### Bug 2 — Динамическая метка экзамена

Добавить колонку `exam_type VARCHAR DEFAULT 'ege' CHECK IN ('ege', 'oge')` в `homework_tutor_assignments`. Репетитор выбирает тип в конструкторе ДЗ (L1, «Расширенные параметры»). На стороне ученика заменить хардкод на `examTypeLabel` — «ЕГЭ» или «ОГЭ».

## 4. Scope

**IN (v1):**
- Исправить `performTaskAdvance` — завершать тред только когда все задачи решены (Bug 1, P0)
- Добавить `exam_type` в таблицу и конструктор ДЗ (Bug 2, P1)
- Динамически подставить «ЕГЭ»/«ОГЭ» в баннер и AI-промпт на student-side (Bug 2, P1)

**OUT:**
- Добавление новых типов экзаменов (ВПР, ЕГЭ по другим предметам) — достаточно `ege`/`oge` для пилота
- Персонализация мотивационного текста (тон, стиль) — отдельная итерация
- Ретроактивная миграция существующих ДЗ к `exam_type` (хватит дефолта `'ege'`)
- Изменения в логике scoring при смене `exam_type`

**LATER:**
- Расширение типов: ВПР, олимпиады, вузовские контрольные
- Автоопределение типа экзамена по `subject` или профилю репетитора

## 5. User Stories

### Ученик
> Когда я решаю задачи ДЗ в произвольном порядке, я хочу, чтобы система принимала мои ответы на любую незавершённую задачу, даже если я уже решил последнюю по номеру.

> Когда мне нужно показать развёрнутое решение, я хочу видеть подсказку в терминах **моего** экзамена (ОГЭ или ЕГЭ), чтобы инструкция не вызывала недоумения.

### Репетитор
> Когда я создаю ДЗ для ученика, готовящегося к ОГЭ, я хочу указать тип экзамена, чтобы AI-интерфейс обращался к ученику корректно.
