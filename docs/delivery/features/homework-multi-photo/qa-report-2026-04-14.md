## Why local auto-QA не покрывает реальный E2E

1. **Auth.** Preview cloud не содержит test-tutor/test-student creds; login flow ведёт на production Supabase Auth.
2. **Seed данные.** Legacy-задача, 4-фото KB-задача и modern assignment — не засеены локально (seed лежит в `supabase/seed/`, подразумевается production-like env).
3. **Storage upload.** Тестовые upload'ы пишут реальные объекты в bucket — leak квоты на dev-окружении.
4. **Backend endpoints.** Edge functions хостятся в Supabase cloud; local Vite preview проксит к production URL через `VITE_SUPABASE_URL`.

Корректный путь для AC-1..9 E2E — запуск на Lovable preview-deploy со своим seed и test account (см. Lovable prompt в предыдущем шаге коммита).
