3
Папка «сократ» → Auto-Publish TRIGGER
При перемещении задачи в «сократ» (или подпапку) срабатывают триггеры:
trg_kb_after_update_moderation / trg_kb_after_insert_moderation → kb_publish_task(source_id)
Триггер создаёт каноническую публичную копию (owner_id = NULL) в каталоге
Двунаправленные ссылки: published_task_id на источнике ↔ source_task_id на копии
fingerprint = MD5(normalized text + answer) для дедупликации
Аудит: запись publish в kb_moderation_log
3a
Auto-Resync TRIGGER
Правка источника в «сократ» →
trg_kb_after_update_moderation → kb_resync_task()
Обновляет публичную копию: text, answer,
solution, attachment_url, fingerprint
Аудит: resync
3b
Fingerprint Dedup BLOCK
trg_kb_before_update_block_dup (BEFORE UPDATE)
pg_advisory_xact_lock для атомарности
Первый опубликовавший fingerprint побеждает
Дубли → hidden_duplicate
Правка, создающая дубль → RAISE EXCEPTION
Каноническая копия (owner_id=NULL, moderation_status='active')
4
Каталог Сократа PUBLIC
fetch_catalog_tasks_v2(topic_id) — SECURITY DEFINER
Возвращает: kb_tasks WHERE owner_id IS NULL AND moderation_status = 'active'
Все репетиторы видят задачи, группированные по topic → subtopic
Read-only витрина: копирование в «Мою базу» или добавление в ДЗ
RLS: не-модераторы видят только moderation_status = 'active'
Модерационные действия (RPCs)
5
Модерация RPC
kb_mod_unpublish(task_id) — снять публикацию (копия → unpublished)
kb_mod_reassign(task_id, new_source_id) — перепривязать источник
Проверка: has_role(uid, 'moderator'), новый источник в «сократ» дереве
Аудит: все действия → kb_moderation_log
После unpublish скрытые дубли НЕ восстанавливаются автоматически

Аудит-лог LOG
kb_moderation_log — все действия: publish, resync, unpublish, reassign, hide_duplicate
Поля: action, task_id, source_task_id, moderator_id, details (JSONB), created_at
RLS: только модераторы видят лог (SELECT WHERE has_role())

promote_folder_to_catalog(folder_id, topic_id)

Таблицы, триггеры и функции

kb_tasks Задачи. Новые поля: published_task_id, source_task_id, fingerprint, moderation_status, hidden_reason, published_by, published_at
kb_folders Папки модераторов. «Черновики» (скрыты) + «сократ» (auto-publish) + подпапки
kb_moderation_log Аудит-лог. action, task_id, source_task_id, moderator_id, details (JSONB)
user_roles Роли пользователей. has_role(uid, 'moderator') — SECURITY DEFINER
kb_publish_task() SECURITY DEFINER. Создаёт каноническую копию. REVOKE FROM PUBLIC — только триггеры
kb_resync_task() SECURITY DEFINER. Синхронизирует копию с источником. REVOKE FROM PUBLIC — только триггеры
fetch_catalog_tasks_v2() RPC SECURITY DEFINER. WHERE owner_id IS NULL AND moderation_status = 'active'
kb_normalize_fingerprint() MD5(lower(trim(text)) || '::' || lower(trim(answer))). Для дедупликации

Триггеры (3 шт.)

trg_kb_before_update_block_dup BEFORE UPDATE. Блокирует правку, создающую дублирующий fingerprint. pg_advisory_xact_lock
trg_kb_after_update_moderation AFTER UPDATE. Auto-publish при перемещении в «сократ» + auto-resync при правке → kb_publish_task() / kb_resync_task()
trg_kb_after_insert_moderation AFTER INSERT. Auto-publish если задача вставлена сразу в «сократ» дерево с topic_id → kb_publish_task()
UNIQUE indexes idx_kb_tasks_unique_source (source_task_id), idx_kb_tasks_unique_published (published_task_id) — one-to-one
