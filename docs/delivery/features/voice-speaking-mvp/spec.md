## 0. Job Context (обязательная секция)

### Какую работу закрывает эта фича?

| Участник | Core Job | Sub-job |
|---|---|---|
| Репетитор (B2B) | Проверять работы учеников по критериям экзамена быстро и без рутины | Проверить **устный** ответ ученика (монолог) по критериям DELF/ЕГЭ/ОГЭ → получить разбор на 1 странице по-русски с баллом → переслушать аудио и поправить балл |
| Школьник/студент (B2C) | Готовиться к устной части экзамена с обратной связью «здесь и сейчас» | Записать устный ответ → сразу получить понятный разбор, не дожидаясь репетитора неделю |

### Wedge-связка

- **B2B-сегмент:** репетиторы иностранных языков (Эмилия — power-user, евангелист, готова платить отдельно).
- **Killer-работа:** проверка устной продукции — её **не закрывает ни одна другая платформа** (cloud.text без AI-проверки, progress.me без фидбека, Telegram — хаос). Gemini не понимает FR-голос.

### Pilot impact

Эмилия прямо сказала, что готова платить за это **отдельно**. Фича превращает её из довольного пользователя в платящего евангелиста и открывает весь сегмент языковых репетиторов (DELF/ЕГЭ/ОГЭ говорение). Первый платный тест — на ней.

## 1. Summary

Фича состоит из двух связанных частей:

1. **Пооценочный разбор по критериям (ядро, «вау» Эмилии).** ИИ раскладывает балл на **критерии экзамена** (DELF B1/B2 production, ЕГЭ/ОГЭ говорение и письмо) — таблица «критерий → балл → короткий комментарий», суммируется в итог. Сейчас проверка ДЗ выдаёт один балл + прозу; добавляем структурированный разбор по дескрипторам, как `elements_check` в пробниках. Применяется к **письменным И устным** заданиям языков.
2. **Голосовой ввод (энейблер для устной части).** Ученик записывает монолог голосом → система транскрибирует речь (Groq Whisper, `language='fr'`) → транскрипт идёт в тот же покритериальный грейдинг. Аудио сохраняется: репетитор переслушивает, оценивает произношение на слух, правит итоговый балл.

Всё это переиспользует существующую инфраструктуру homework-проверки + subject-rubric layer (§19). Новые слои: (а) структурированный покритериальный вывод грейдинга, (б) захват/хранение аудио + транскрипция.

## 2. Problem

### Текущее поведение

Устную часть проверить негде. Gemini не понимает французскую речь. Telegram — «супернеудобная площадка» (30 веток, ученик забывает подписать задание). cloud.text сохраняет аудио, но без AI-проверки. progress.me фидбек не даёт.

### Боль

Репетитор языка не может масштабировать проверку говорения: каждый устный ответ — ручное прослушивание + устный разбор на уроке. Ученик бросает устные задания без проверки, потому что «привык к ИИ — связь здесь и сейчас».

### Текущие «нанятые» решения

cloud.text (аудио без AI) + ручная проверка на уроке + Telegram-голосовухи. Ни одно не даёт «транскрипт + разбор по критериям на 1 странице».

## 3. Solution

### Описание

Новый `task_kind = 'speaking'`. Поток:

1. Репетитор создаёт задание с устной формулировкой (sujet монолога) — как обычную задачу, но помечает её как устную.
2. Ученик открывает задачу → записывает монолог через существующий `useVoiceRecorder` (умеет iOS Safari m4a / webm) → отправляет.
3. Backend: аудио → Groq Whisper (`whisper-large-v3-turbo`, `language` из предмета) → транскрипт.
4. Транскрипт идёт в `runStudentAnswerGrading` как ответ ученика → subject-rubric layer (DELF/ЕГЭ/ОГЭ oral) → разбор на 1 странице по-русски + черновой балл.
5. Сохраняем: аудио (`voice_ref`) + транскрипт. Репетитор видит плеер + транскрипт + разбор, переслушивает, правит балл (override, §6).

### Ключевые решения

- **Покритериальный разбор — ядро.** ИИ возвращает структурированный вывод: массив `criteria_breakdown[{ label, score, max, comment }]`, сумма = `ai_score`. Критерии берутся из subject-rubric layer (§19), который уже знает дескрипторы DELF/ЕГЭ/ОГЭ. Mirror паттерна `elements_check {I,II,III,IV}` из пробников (CLAUDE.md §12), но с именованными критериями и баллами. Только для языков (нульуемое поле); физика/математика не затрагиваются.
- **Покритериальный разбор применяется к письму И голосу.** Письменная проверка получает его сразу (её основной поток, живой сегодня — можно валидировать без голоса); голос едет поверх готового грейдинга.
- **Override — на итоговый балл (как сейчас, §6).** Эмилия правит общий балл. Покритериальная правка (override по каждому дескриптору) — out of scope MVP.
- **Формат задания = монолог.** Общий знаменатель DELF B1 (expression d'un point de vue), DELF B2 (exposé), ЕГЭ и ОГЭ (монологическое высказывание). Сознательно обходит диалог (нужен собеседник) и чтение вслух (это произношение).
- **ИИ грейдит содержание, не произношение.** Транскрипт даёт ~80% критериев устной части (выполнение задачи, лексика, грамматика, связность). Phonétique/произношение оценивает Эмилия на слух из сохранённого аудио — это и есть вторая половина ценности «переслушать».
- **Транскрипция через Groq Whisper.** Движок уже в проекте (Telegram-бот, веб-чат), знает FR/EN/ES нативно. Решает «Gemini не понимает FR-голос»: голос → Whisper → текст → Gemini грейдит транскрипт.
- **Переиспользуем письменный pipeline.** `runStudentAnswerGrading`, subject-rubric layer (§19), разбор на 1 странице, tutor override (§6), RU-bypass signed URLs. Новое — только аудио-захват, хранение, шаг STT.
- **Разбор строго на одной странице** (жёсткое требование Эмилии: «на нескольких — фигня, он не всё читает»).
- **Код language-agnostic, запуск только FR.** Feature-флаг на Эмилию (mirror `feature_mock_exams_enabled`).
- **Балл по-русски** (её прямая просьба «прочитала за один раз»).

### Scope

**In scope (MVP).** Приоритеты внутри scope IN (pipeline шаг 4): P0 = без этого фича не решает задачу; P1 = fast-follow через 1-2 дня после P0.

- **[P0] Покритериальный разбор** (`criteria_breakdown`) для языковых заданий — письмо И голос. Таблица «критерий → балл → комментарий» на 1 странице по-русски.
- **[P0]** `task_kind = 'speaking'` — запись монолога учеником (+ tutor-side пометка задачи как устной, Spec §3 шаг 1).
- **[P0]** Транскрипция FR через Whisper.
- **[P0] Прослушать запись ДО отправки** (ученик). Запись 5-7 мин вслепую — дилбрейкер («записалось? не обрезало?»). Playback + возможность перезаписать перед commit. *(UX-добавка 2026-05-29, signal-driven.)*
- **[P0] Транскрипт виден ученику** как часть разбора, помечен «Распознанная речь» (не сырой user-бабл). Мост доверия: «вот что AI услышал → вот как оценено». Закрывает open question #3. *(UX-добавка 2026-05-29.)*
- **[P0]** Сохранение аудио + транскрипт; на стороне репетитора — **нативный `<audio controls>`** (scrubbing + скорость 1.5× бесплатно, лучше для прослушивания 5-7 мин) + транскрипт под ним. *(Уточнение 2026-05-29: было «плеер play/pause» — нативный контрол строго лучше и меньше кода.)*
- **[P0]** Tutor override **итогового** балла (существующий механизм §6).
- **[P0]** Feature-флаг, включён только для Эмилии.
- **[P1] Двухфазный прогресс после отправки** (ученик): «Распознаю речь… → Проверяю по критериям…» вместо фриза спиннера (STT + Gemini = заметная задержка). *(UX-добавка 2026-05-29.)*
- **[P1] Предупреждение перед хард-капом длительности** (ученик): на ~6:00 при 7-мин лимите «осталась минута», не обрубать на полуслове. *(UX-добавка 2026-05-29.)*

**Out of scope (сознательно):**
- Покритериальный override (правка балла по каждому дескриптору отдельно) — пока override только итог.
- Произношение / сравнение с носителем (фонемный анализ) — отдельный движок, отдельная фича.
- Тайм-коды «клик на момент записи».
- Голос-комментарий репетитора на моменте (custdev P1, вторая итерация — в MVP репетитор комментирует текстом).
- Диалог / débat / условный диалог-расспрос (нужен AI-собеседник).
- Чтение вслух (это произношение).
- Метеринг минут / отдельный биллинг (после валидации).
- EN/ES в UI (код готов, флаг выключен).

## 4. User Stories

### Репетитор
> Когда ученик готовится к устной части DELF/ЕГЭ, я хочу дать ему устное задание и получить транскрипт + разбор по критериям на одной странице + аудио для прослушивания, чтобы проверять говорение так же быстро, как письмо, и не тратить урок на ручной разбор.

### Школьник/студент
> Когда я тренирую монолог к экзамену, я хочу записать ответ голосом и сразу получить понятный разбор по-русски с баллом, чтобы не ждать репетитора неделю.

## 5. Technical Design

### Затрагиваемые файлы (ориентировочно)

- `supabase/migrations/` — `task_kind` CHECK + `'speaking'`; storage bucket для аудио; `tutors.feature_voice_speaking_enabled`.
- `supabase/functions/homework-api/guided_ai.ts` — расширить output-схему `evaluateStudentAnswer` полем `criteria_breakdown` (массив `{label, score, max, comment}`) для языков; prompt просит разложить балл по дескрипторам, сумма = `ai_score`.
- `supabase/functions/homework-api/index.ts` — `handleStudentSubmission`: ветка `task_kind='speaking'` → транскрипция перед `runStudentAnswerGrading`. 4 write-path `task_kind` (§0 двойной derive-инвариант) — `'speaking'` ставится явно, не через `deriveTaskKind`. Сохранение `criteria_breakdown`.
- `supabase/functions/_shared/` — helper транскрипции через Groq Whisper (вынести общий с telegram-bot/chat паттерн; `language` из предмета).
- `supabase/functions/_shared/subject-rubrics/languages-ege.ts` — критерии с **именами и балльной шкалой** (DELF B1/B2 production écrite+orale, ЕГЭ/ОГЭ письмо+монолог), чтобы AI мог разложить балл; фонетика помечена «оценивает репетитор».
- `src/components/homework/` — рендер `criteria_breakdown` таблицей на 1 странице (письмо + голос, student post-grade + tutor view).
- `src/components/student/homework-problem/` — UI записи монолога в SubmitSheet (переиспользовать `useVoiceRecorder`).
- `src/components/homework/` / `GuidedThreadViewer.tsx` — плеер аудио + транскрипт на стороне репетитора.
- `src/types/homework.ts` — `task_kind` union + `'speaking'`.

### Data Model

- **`criteria_breakdown` хранение** — новое nullable JSONB поле под покритериальный разбор. Варианты: колонка `homework_tutor_task_states.ai_criteria_json JSONB NULL` (mirror `ai_score`/`ai_score_comment`, доступно и для письма, и для голоса) — рекомендуется. Решить на реализации. Формат: `[{label, score, max, comment}]`. Anti-leak: это feedback, виден ученику post-submit; tutor-only поля не добавляем.
- `homework_tutor_tasks.task_kind` — расширить CHECK на `'speaking'` (миграция; mirror `20260509120000`).
- `homework_tutor_thread_messages.submission_payload` — уже зарезервировано поле `voice_ref?: string | null` (§16). Аудио-ref сюда; транскрипт — в `text` / синтезированный answerText.
- Storage: новый bucket `homework-voice` (private) **или** переиспользовать `homework-submissions` с подпапкой `{userId}/{assignmentId}/voice/`. Решить на этапе реализации (склонность — переиспользовать существующий submission-флоу).
- `tutors.feature_voice_speaking_enabled boolean default false` — флаг (mirror `feature_mock_exams_enabled`).

### API

- `POST /student/problem/:hwId/:taskId/submission` (существующий `handleStudentSubmission`) — принимает `voice_ref` для speaking. Backend: signed URL аудио → `rewriteToDirect()` → Whisper → транскрипт → `runStudentAnswerGrading(feedbackKind='check_result')`.
- AI-квота: вызвать `checkAiQuota(userId, db, { context: 'homework', incrementUsage: true })` до Whisper+Gemini (§17). Whisper + Gemini оба стоят денег — гейт до вызовов.

### Subject → language map

`french→fr`, `english→en`, `spanish→es`, `russian→ru`. MVP: только `fr` (через флаг).

### Миграции

1. `homework_tutor_task_states.ai_criteria_json JSONB NULL` (покритериальный разбор).
2. `task_kind` CHECK + `'speaking'`.
3. `tutors.feature_voice_speaking_enabled`.
4. (если новый bucket) storage bucket `homework-voice` private + RLS.

## 6. UX / UI

### Поток

- **Ученик:** карточка задачи (формулировка монолога) → запись (`useVoiceRecorder`, видимый таймер) → **прослушать запись** (P0) → при желании перезаписать → отправка. Один primary CTA. После отправки — **двухфазный прогресс** «Распознаю речь… → Проверяю по критериям…» (P1), затем разбор: **транскрипт** (помечен «Распознанная речь», P0) + **таблица критериев** + итоговый балл по-русски.
- **Репетитор:** в треде/разборе — **нативный `<audio controls>`** (scrubbing + скорость, P0) + транскрипт под ним + **таблица критериев** (критерий → балл/макс → комментарий) + итоговый балл + кнопка «Изменить балл» (existing override). Всё на 1 странице.

### UX-принципы

- Разбор на **одной странице** — жёсткий инвариант (и у ученика, и у репетитора).
- **Никакой отправки вслепую** — ученик обязательно слышит свою запись до commit (7 мин нельзя отправлять, не проверив).
- **Транскрипт = мост доверия** — ученик видит, что AI услышал, рядом с тем, как это оценено.
- Фидбек «здесь и сейчас» — ученик получает разбор сразу после отправки; задержку STT+grading показываем явным двухфазным прогрессом, не фризим спиннер.
- Минимум трения: пометить задачу устной / добавить ученика — за секунду (как для письма).

### Cross-browser

- `useVoiceRecorder` уже обрабатывает iOS Safari m4a vs webm/opus. Whisper принимает оба формата. Не вводить новый recorder.

## 7. Validation

### Критерий успеха теста Эмилии

**Главный (её формулировка):** AI оценивает содержание **по критериям**, и Эмилия **принимает покритериальный разбор без ручной правки** в большинстве случаев. Не zero-touch — override остаётся, но она перестаёт переразбирать вручную.

Измеримые прокси за неделю теста:
- Покритериальный разбор приходит на ≥ 80% языковых работ (письмо + голос), сумма по критериям = итоговый балл.
- Override итогового балла **≤ 30%** работ (в остальных Эмилия принимает балл как есть).
- Эмилия больше **не декомпозирует балл по критериям руками** (её прямая боль «я сама вручную это делаю» снята).
- Транскрипт FR читаемый на типичной записи ученика.
- Явное подтверждение «беру / готова платить».

> Сначала валидируем покритериальный разбор на **письменной** части (живой поток сегодня), потом включаем голос — он наследует тот же грейдинг.

### Связь с pilot KPI

Конвертирует power-user-евангелиста в платящего → реферальный мотор (custdev: «кружок языковиков») → вход в сегмент языковых репетиторов.

## 8. Risks & Open Questions

| Риск | Вероятность | Митигация |
|---|---|---|
| **AI неточен в покритериальной оценке** (её бар «без правок») | Высокая | Валидируем сначала на письме (живой поток); метрика override ≤30%; критерии с явной балльной шкалой в `languages-ege.ts`; override всегда доступен. Если точность низкая — это и есть главный результат теста, итерируем prompt |
| Качество FR-транскрипции на акценте ученика | Средняя | Whisper-large знает FR хорошо; аудио всегда сохранено — Эмилия слышит оригинал и правит |
| Oral-рубрика отличается от письменной (нет в subject-rubric layer) | Высокая | Добавить production orale критерии в `languages-ege.ts`; фонетику явно отдать репетитору |
| `task_kind='speaking'` сломает derive-инвариант (§0) | Средняя | `'speaking'` ставится явно во всех write-path; `deriveTaskKind` не должен его перетирать |
| Latency Whisper + Gemini | Низкая | Whisper turbo быстрый; грейдинг как для письма |
| Размер аудио / лимиты | Низкая | Хард-кап ~7 мин + size-cap (`MAX_VOICE_BYTES`); предупреждение на ~6:00; компрессия как в существующем voice-флоу. 7 мин m4a/webm ≈ 3-7 МБ — под лимитом Groq 25 МБ |
