# AI-промпты: AI-загрузка задач (`kb-ai-extract`)

**Версия:** v0.1 · **Дата:** 2026-06-25 · См. `spec.md`, `concept.md`.

Контракт системного промпта и JSON-выхода для извлекающего AI. Используется в edge function `kb-ai-extract` через `callLovableJson` (`homework-api/ai_shared.ts`, модель `google/gemini-3-flash-preview`, мультимодал, `temperature: 0.2`). Функция **только извлекает** черновики; запись — на клиенте через существующий `insertTask`.

---

## 1. Главные инварианты (НЕ нарушать)

1. **Ингест, не генерация.** Извлекаем то, что есть в материале. Ничего не выдумываем.
2. **Ответ не угадываем.** Если ответа в материале нет или AI не уверен → `answer: null` + `answer_confidence: "low"`. Неверный ответ в базе отравляет авто-проверку ДЗ — пустое поле лучше догадки.
3. **Рисунки не перерисовываем.** Для каждой задачи указываем, какой из приложенных файлов к ней относится (`image_index`); система прикрепляет **оригинал**. `image_action` всегда `"attach_original"`. Волна 2 (2026-07-11): дополнительно модель возвращает `image_bbox` — рамку рисунка внутри изображения (Gemini-конвенция `[ymin, xmin, ymax, xmax]` 0..1000); кроп из мультизадачного скрина делает **клиент** (canvas) после подтверждения/правки рамки тутором — оригинальные пиксели, не генерация.
4. **LaTeX `$…$`.** Все формулы и физвеличины — inline-LaTeX в `$…$`, как Сократ хранит задачи сейчас. Чистый LaTeX (`S_x`, не `Sx`).
5. **Только валидный JSON** по схеме §3. Никакого текста вне JSON. Бэкслеши LaTeX экранируются по правилам JSON (один `\` → `\\` в строке).

## 2. Системный промпт (готов к вставке)

```
Ты — ассистент репетитора физики в сервисе «Сократ». Тебе дают сырой материал
(текст, фото страницы сборника, PDF-задачник, таблицу) и ты извлекаешь из него
ОТДЕЛЬНЫЕ задачи в структурированном виде для базы задач репетитора.

ТВОЯ ЗАДАЧА — РАСПОЗНАТЬ И СТРУКТУРИРОВАТЬ, НЕ ПРИДУМЫВАТЬ.

Правила:
1. Раздели материал на отдельные задачи. Не склеивай разные задачи и не дроби
   одну на части. Извлеки ВСЕ задачи ДО ЕДИНОЙ: на странице сборника их может
   быть 10-20 и больше — НИКОГДА не сокращай и не обрывай список. Если на
   странице 15 задач — верни 15 объектов.
2. Текст условия (поле text): перепиши дословно, исправляя только явные опечатки
   распознавания. Все формулы и физические величины оформи inline-LaTeX в долларах:
   $v_0 = 2\ \text{м/с}$, $S_x = 15 - 9t + 3t^2$, $a = g(\sin\alpha - \mu\cos\alpha)$.
   Пиши ЧИСТЫЙ LaTeX: индексы $S_x$ (не «Sx»), степени $t^2$, $\sqrt{}$, $\frac{}{}$,
   $\cdot$, $\alpha$, $\mu$, $\pi$. Единицы — внутри $…$ через \text{} или как обычный
   текст после формулы. Не оставляй битых конструкций вроде «$1$S» или «3t²($S$».
   Таблицы с данными оформляй LaTeX-таблицей прямо в тексте (окружение array для
   KaTeX, например $$\begin{array}{cc} t,\ \text{с} & v,\ \text{м/с} \\ 0 & 0 \\ 2 & 4 \end{array}$$),
   а не прикрепляй таблицу картинкой.
3. Ответ (поле answer): впиши ТОЛЬКО если он явно есть в материале или однозначно
   следует из условия и ты уверен. Если ответа нет или есть сомнение —
   answer = null, answer_confidence = "low". НИКОГДА не выдумывай число.
4. Решение (поле solution): впиши ход решения, ТОЛЬКО если он есть в материале.
   Иначе null.
5. Критерии (поле rubric_text): впиши, ТОЛЬКО если в материале есть схема
   оценивания/критерии. Иначе null.
6. Классификация: определи exam ("ege"/"oge"/null), kim_number (1..30 или null),
   primary_score (если указан), answer_format, и предложи тему/подтему
   (topic_suggestion/subtopic_suggestion) по содержанию. Это подсказки — репетитор
   подтвердит. Если рядом с номером задачи указано «Тип N» (формат сборников
   РЕШУ ЕГЭ: «3. Тип 5 № 8085») — это номер задания КИМ: kim_number = N.
   Порядковый номер задачи в подборке номером КИМ НЕ является.
7. Рисунки: для КАЖДОЙ задачи укажи source_image_index — 0-based номер приложенного
   изображения, с которого она распознана (null, если задача из текстового материала).
   Если в задаче есть существенный рисунок (график, схема, чертёж, электрическая
   цепь, иллюстрация), без которого её не решить — укажи image_index (номер
   изображения, где находится рисунок) И image_bbox — рамку ИМЕННО РИСУНКА (не всей
   задачи, без текста условия) в формате [ymin, xmin, ymax, xmax]: целые 0..1000
   относительно высоты и ширины изображения. Рамка работает и когда на изображении
   НЕСКОЛЬКО задач — система вырежет нужный фрагмент. Если в задаче нет рисунка
   (чисто текстовая задача) — image_index = null, image_bbox = null. Если
   сомневаешься, нужен ли рисунок — оба null и добавь "image" в needs_review_fields.
   НЕ описывай рисунок текстом вместо самого рисунка и НЕ перерисовывай его;
   image_action = "attach_original". НИКОГДА не считай рисунком бланк ответов,
   номер задания, рамку или служебные пометки.
8. Уверенность: для каждого сомнительного поля добавь его имя в needs_review_fields.
9. Верни СТРОГО валидный JSON по заданной схеме. Без пояснений, без markdown-обёрток.

РАБОТА С ТЕКСТОМ СТРАНИЦ PDF (если в материале есть блоки «— ТЕКСТ СТРАНИЦЫ N —»):
это ТОЧНЫЙ текст страницы из цифрового PDF — бери условия задач из него, а не
пере-распознавай с картинки (в словах могут быть лишние пробелы от переносов:
«Ка мень» = «Камень» — восстанавливай слова). Приложенные изображения тех же
страниц используй для рисунков, графиков и image_bbox. Блок «— ПРОДОЛЖЕНИЕ … —»
служит ТОЛЬКО для завершения последней задачи, если её текст обрывается концом
страницы; НЕ извлекай из него новые задачи.

ТАБЛИЦА ОТВЕТОВ: если на странице или изображении есть СПИСОК/ТАБЛИЦА ответов
(пары «номер задачи → ответ» БЕЗ условий) — верни её содержимое в поле
"answers_table" (массив объектов {"num": номер задачи, "answer": "ответ"}).
НИКОГДА не создавай задачи из строк таблицы ответов. Если условие задачи и её
ответ из таблицы видны в ОДНОМ материале — сразу впиши ответ в поле answer этой
задачи. Нет таблицы ответов → answers_table = [].

Контекст от репетитора (если передан): exam_hint, topic_hint — учитывай как приоритет.
```

> **Кросс-чанковый мердж ответов (2026-07-17):** сборники часто держат ответы
> отдельной таблицей в конце — при чанках по 2 страницы модель чанка с таблицей
> не видит условий из других чанков. Поэтому таблица возвращается структурно
> (`answers_table` во всех 3 схемах), а КЛИЕНТ мержит ответы в задачи по номеру
> сборниковой нумерации (`source_num` из маркеров текстового слоя, zip при
> точном count-совпадении чанка). Пустые answer заполняются, непустые НЕ
> перетираются.

> **W4 (2026-07-16):** усиление полноты (правило 1 «ВСЕ задачи до единой») + блок
> «РАБОТА С ТЕКСТОМ СТРАНИЦ PDF» добавлены во все 3 промпта (physics/social/generic,
> в social/generic — «задание» вместо «задача»). Клиент шлёт текст страниц цифрового
> PDF вместе с картинками (чанки по 2 страницы) — см. `InputStage.buildChunkText`.

> **«Тип N» = КИМ (2026-07-22, репорт Милады):** сборники РЕШУ ЕГЭ подписывают
> каждую задачу `N. Тип K № <id>`, где «Тип» — номер задания КИМ, а N — порядковый.
> Правило добавлено во все 3 промпта (для фото-страниц без текстового слоя); для
> цифровых PDF клиент дополнительно проставляет КИМ детерминированно из текстового
> слоя (`taskMarkers.ts::scanTaskMarkers` → `typeNumbersPerPage`; Тип побеждает и
> догадку AI, и variant-numbering-эвристику — `hasTypeMarkers` её гасит).

## 3. JSON-выход (схема)

`callLovableJson` парсит JSON-объект. Контракт (mirror `MockExamPart2Draft`-стиля — расширять аддитивно):

```json
{
  "tasks": [
    {
      "text": "Координата тела меняется по закону $S_x = 15 - 9t + 3t^2$ (СИ). Определите проекцию ускорения $a_x$.",
      "answer": "6 м/с²",
      "answer_confidence": "high",
      "solution": null,
      "answer_format": "number",
      "check_format": "short_answer",
      "kim_number": 1,
      "exam": "ege",
      "primary_score": 1,
      "rubric_text": null,
      "topic_suggestion": "Кинематика",
      "subtopic_suggestion": "Уравнения движения",
      "source_label": "Демидова ЕГЭ-2026",
      "image_index": null,
      "image_bbox": null,
      "source_image_index": null,
      "image_action": "attach_original",
      "needs_review_fields": [],
      "notes": null
    }
  ],
  "stats": { "found": 1, "low_confidence_answers": 0, "unreadable_images": 0 }
}
```

**Маппинг на `kb_tasks` (что персистится при commit, поля `CreateKBTaskInput`):**

| JSON-поле | → `kb_tasks` | Примечание |
|---|---|---|
| `text` | `text` | LaTeX `$…$` |
| `answer` | `answer` | `null` если не уверен |
| `solution` | `solution` | только из материала |
| `answer_format` | `answer_format` | персистится |
| `check_format` | — | **не в `CreateKBTaskInput`** — advisory; формат проверки выводится при импорте в ДЗ (`mapAnswerFormatToCheckFormat`/`inferCheckFormat`). Хранить advisory можно в UI до записи (field-parity Q3) |
| `kim_number` | `kim_number` | 1..30 \| null |
| `exam` | `exam` | `ege`/`oge` |
| `primary_score` | `primary_score` | |
| `rubric_text` | `rubric_text` | tutor-only, в Каталог не уходит |
| `topic_suggestion`/`subtopic_suggestion` | `topic_id`/`subtopic_id` | клиент резолвит название→id (подсказка) |
| `source_label` | `source_label` | default `my` |
| `image_index` | → `attachment_url` | сервер берёт оригинал приложенного файла |
| `image_bbox` | → кроп на клиенте | волна 2 (2026-07-11): `[ymin, xmin, ymax, xmax]` целые 0..1000 (нативная конвенция детекции Gemini). Edge валидирует (clamp, min-площадь ~3% по каждой оси, ymin<ymax, xmin<xmax) и отдаёт клиенту как `{x, y, w, h}` в долях 0..1. Клиент показывает превью кропа, тутор правит рамку; сам кроп — canvas на клиенте при commit |
| `source_image_index` | — | волна 2: с какого изображения распознана задача (контекст для режима `refine`); в БД не персистится |

**Перечисления:** `answer_confidence ∈ {high, medium, low}`; `answer_format ∈ {number, text, detailed, matching, choice, null}`; `check_format ∈ {short_answer, detailed_solution, null}`; `exam ∈ {ege, oge, null}`; `image_action = "attach_original"` (P0 — единственное значение; `"redraw"` зарезервировано под P2 opt-in, в P0 не эмитить).

**Режим `refine` (волна 2, 2026-07-11):** body `{ mode: "refine", folder_id, subject, comment ≤2000, draft, image_refs? ≤2 }` → тот же subject-промпт + схема; user-контент = JSON текущего черновика + «Комментарий репетитора: …» (+ контекст-изображения). Ответ — ровно одна задача в `tasks[]` → нормализация `normalizeTask` → `{ draft }`. Usage-source `kb_extract_refine`.

## 4. LaTeX-контракт (подробно)

- Формулы и физвеличины — **inline `$…$`**. Display-формулы (отдельной строкой) — `$$…$$`, только если так в оригинале.
- Чисто: `S_x`, `v_0`, `t^2`, `\sqrt{2h/g}`, `\frac{mv^2}{2}`, `\cdot`, `\alpha`, `\mu`, `\Delta`, `\sin`, `\cos`.
- Единицы: `$v_0 = 2\ \text{м/с}$` (внутри) либо `$v_0 = 2$ м/с` (после) — приоритет: как в оригинале; по умолчанию — чисто и единообразно по всему пакету.
- JSON-экранирование: один бэкслеш LaTeX → `\\` в JSON-строке. Модель обязана вернуть валидный JSON (валидатор edge отклонит битый → retry-once).
- Рендер на клиенте — `MathText` (KaTeX) по `$…$`, поле редактируется как сырой LaTeX с живым превью.

## 5. Confidence и anti-hallucination

- `answer_confidence`: `high` — ответ явно в материале/однозначно следует и проверен; `medium` — выведен, но возможна неоднозначность; `low` — нет основания → **`answer: null`**.
- Формулы из плохого скана → `medium` + поле в `needs_review_fields`.
- Нечитаемый фрагмент → не выдумывать; `notes` + соответствующее поле в `needs_review_fields`.
- Запрещено: придумывать числовой ответ, дописывать решение «от себя», склеивать/дробить задачи без явных границ.

## 6. Политика рисунков (резюме для промпта)

- **Прикрепляем рисунок ТОЛЬКО когда он нужен (решение владельца 2026-06-25).** `image_index` ставится лишь если изображение содержит **ровно одну** задачу И в ней есть существенный рисунок (график/схема/чертёж/цепь/иллюстрация). Чисто текстовая задача → `image_index = null` (только распознанный текст). Скриншот с **несколькими** задачами → не прикрепляем ни к одной (P0 не умеет вырезать один рисунок). Сомнение → `null` + `"image"` в `needs_review_fields`. Таблицы → LaTeX-таблица в тексте, не картинкой. `image_action: "attach_original"`; перерисовка/генерация запрещены в P0.
- **Backstop на сервере:** если AI всё же привязал одно изображение к >1 задаче — edge обнуляет `attachment_ref` у всех (мультизадачный скрин = только текст).
- **Ручной контроль тутора:** на карточке-черновике можно и добавить картинку (если AI не прикрепил), и убрать (если прикрепил зря) — независимо от решения AI.
- Не подменять рисунок текстовым описанием в `text` (можно кратко упомянуть «см. рисунок», но условие — без выдуманных числовых данных с рисунка, если они нечитаемы).
- **P2 (opt-in, вне этого промпта):** отдельный режим «Перерисовать (бета)» — vector-first (SVG/TikZ-спека) для простых схем, результат — «AI-черновик», оригинал сохраняется как fallback, тутор подтверждает. Растровая генерация для экзаменной физики не используется.

## 8. После извлечения (клиент)

1. Рендер карточек-черновиков (LaTeX-превью, confidence-флаги, дедуп-баннеры).
2. Дедуп: `kb_normalize_fingerprint(text, answer, attachment_url)` сверяется с базой репетитора и Каталогом до записи.
3. Правки: inline (сырой `$…$`) или репликой в чат (повторный мини-вызов AI на конкретную карточку).
4. Подтверждение → `insertTask` (батч P1) в выбранную папку; `attachment_url` = оригинал; `owner_id` ставит сервер; `source_label` default `my`.
5. Телеметрия PII-free (`kb_ai_extract_run`, `kb_ai_tasks_saved`) — counts, без текста/имён.
