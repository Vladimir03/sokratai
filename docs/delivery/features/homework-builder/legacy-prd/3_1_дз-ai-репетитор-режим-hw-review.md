## 3.1 ДЗ → AI‑репетитор (режим HW_REVIEW)
**Цель**: дать ученику интерактивный разбор только ошибочных задач через Сократический диалог.

### In‑scope
- Callback `hw_review:{submission_id}`
- Состояние `HW_REVIEW` в `user_bot_state.context`
- Диалоговый prompt (2 вопроса максимум)
- Кнопки `[➡️ Следующая]` и `[✅ Хватит]`

### UX
1) Ученик нажимает `🧠 Разобрать ошибки`.
2) Бот вытаскивает `items` где `ai_is_correct=false`.
3) Показывает задачу N:
- условие
- «Твой ответ»
- «Что не так» (ai_feedback)
- вопрос: «Как решал?»
4) На каждое сообщение ученика бот отвечает 1–2 уточняющими вопросами, не выдавая решение.
5) По завершению — «Молодец! 💪» и выход в `IDLE`.

### Контекст и хранение
`context` для HW_REVIEW:
```json
{
  "mode": "HW_REVIEW",
  "submission_id": "...",
  "errors": [
    {"task_id":"...","task_text":"...","recognized_text":"...","correct_answer":"...","error_type":"...","ai_feedback":"..."}
  ],
  "cursor": 0
}
```

### Prompt requirements (строго)
- Роль: «Ты Сократ. Ученик ошибся.»
- Правила:
  - НЕ давай готовый ответ.
  - Максимум 2 вопроса.
  - Опирайся на `error_type`.
  - Тон: дружелюбный.

### Приёмка
- После проверки ученик может запустить разбор.
- Бот ведёт диалог по ошибкам последовательно.
- Готовый ответ не появляется.

### Инструкция для Codex/Claude
Добавь обработчик callback `hw_review:*` и обработку сообщений в состоянии `HW_REVIEW`. Используй Gemini text‑вызов с отдельным prompt.
