## 1.1 Миграция БД
**Цель**: добавить таблицы для ДЗ, сдач и состояния бота + индексы + RLS + bucket для изображений.

### In‑scope
- SQL‑миграция: новые таблицы, enum/constraints, индексы.
- RLS policies для tutor/student.
- Storage bucket `homework-images` и policies.

### Out‑of‑scope
- UI Lovable.
- Уведомления, cron.

### Требования к данным (schema)
Создать миграцию: `supabase/migrations/[timestamp]_homework_system.sql`

Таблицы:
1) `homework_assignments`
- `id uuid pk default gen_random_uuid()`
- `tutor_id uuid not null references auth.users(id)`
- `title text not null`
- `subject text not null check (subject in ('math','physics','history','social','english','cs'))`
- `topic text null`
- `description text null`
- `deadline timestamptz null`
- `status text not null default 'draft' check (status in ('draft','active','closed'))`
- `created_at timestamptz not null default now()`

2) `homework_tasks`
- `id uuid pk default gen_random_uuid()`
- `assignment_id uuid not null references homework_assignments(id) on delete cascade`
- `order_num int not null`
- `task_text text not null`
- `task_image_url text null`
- `correct_answer text null`
- `solution_steps text null`
- `max_score int not null default 1`
- уникальность: `unique(assignment_id, order_num)`

3) `homework_submissions`
- `id uuid pk default gen_random_uuid()`
- `assignment_id uuid not null references homework_assignments(id) on delete cascade`
- `student_id uuid not null references auth.users(id)`
- `telegram_chat_id bigint not null`
- `status text not null default 'in_progress' check (status in ('in_progress','submitted','ai_checked','tutor_reviewed'))`
- `submitted_at timestamptz null`
- `total_score int null`
- `total_max_score int null`
- `unique(assignment_id, student_id)`

4) `homework_submission_items`
- `id uuid pk default gen_random_uuid()`
- `submission_id uuid not null references homework_submissions(id) on delete cascade`
- `task_id uuid not null references homework_tasks(id)`
- `student_image_urls text[] null`  (может быть до 4)
- `student_text text null`
- `recognized_text text null`
- `ai_is_correct boolean null`
- `ai_confidence double precision null`
- `ai_feedback text null`
- `ai_error_type text null` check (ai_error_type in ('calculation','concept','formatting','incomplete','factual_error','weak_argument','wrong_answer','partial','correct'))
- `ai_score int null`
- `tutor_override_correct boolean null`
- `tutor_comment text null`
- `created_at timestamptz not null default now()`
- уникальность: `unique(submission_id, task_id)`

5) `homework_student_assignments`
- `id uuid pk default gen_random_uuid()`
- `assignment_id uuid not null references homework_assignments(id) on delete cascade`
- `student_id uuid not null references auth.users(id)`
- `notified boolean not null default false`
- `notified_at timestamptz null`
- уникальность: `unique(assignment_id, student_id)`

6) `user_bot_state`
- `user_id uuid primary key references auth.users(id) on delete cascade`
- `state text not null default 'IDLE'`
- `context jsonb not null default '{}'::jsonb`
- `updated_at timestamptz not null default now()`

### Индексы (минимум)
- `homework_assignments(tutor_id, status, deadline)`
- `homework_tasks(assignment_id, order_num)`
- `homework_student_assignments(student_id, assignment_id)`
- `homework_submissions(assignment_id, student_id, status)`
- `homework_submission_items(submission_id, task_id)`

### RLS — логика доступа (минимальная, но безопасная)
Включить RLS на всех таблицах.

**Tutor** (где `homework_assignments.tutor_id = auth.uid()`)
- assignments: select/insert/update/delete
- tasks: select/insert/update/delete (через join на assignments)
- student_assignments: select/insert/delete
- submissions + items: select/update (для review)

**Student** (где назначено через `homework_student_assignments`)
- assignments: select только active/closed, которые назначены ему
- tasks: select только задачи из назначенных assignments
- submissions: insert/select/update только свои (до submitted)
- items: insert/select/update только свои (до submitted)

### Storage bucket + policies
- Создать bucket: `homework-images` (private)
- Политики (вариант A — через owner):
  - upload: `auth.uid() = owner` (записываем owner=student)
  - read: student owner + tutor (через проверку membership по пути и таблицам)

> Примечание: если текущая архитектура уже хранит `telegram_chat_id`/mapping user↔telegram, использовать существующее.

### Приёмка (Acceptance Criteria)
- Миграция применима на чистой БД Supabase.
- Все таблицы созданы, constraints и индексы на месте.
- RLS включён; tutor видит только своё; student видит только назначенное.
- Bucket `homework-images` существует и доступен согласно политикам.

### Тест‑кейсы
- Tutor создаёт assignment+tasks → student не может читать, пока не назначили.
- Назначили student → student может читать assignment+tasks.
- Student создаёт submission → другой student не видит.
- Tutor видит submissions по своему assignment.
