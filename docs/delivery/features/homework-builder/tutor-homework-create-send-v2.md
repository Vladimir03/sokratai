## 1. Контекст и проблема

### 1.1. Что делает репетитор сегодня

Типичный флоу подготовки ДЗ у репетитора по физике ЕГЭ/ОГЭ:

1. Урок закончился. Репетитор знает тему (например, «Кинематика»).
2. Открывает свои архивы / Решу ЕГЭ / PDF-сборники → ищет 5–7 задач.
3. Копирует задачи в заметки или документ.
4. Форматирует (нумерация, формулы, рисунки).
5. Отправляет ученику через Telegram / WhatsApp.

**Время:** 30–60 минут. При 10+ учениках — 3–5 часов в неделю только на ДЗ.

### 1.2. Что обещает Сократ (wedge)

> Собрать ДЗ и новую практику по теме урока за 5–10 минут вместо 30–60 минут.

### 1.3. Что есть сейчас в продукте

Текущий конструктор ДЗ (`TutorHomeworkCreate.tsx`) — 3-шаговый визард:

| Шаг | Что собирает | Боли |
|-----|-------------|------|
| 1. Метаданные | Название, предмет, тема, дедлайн, режим | Много полей до начала работы с задачами |
| 2. Задачи и материалы | Ручной ввод, KB picker, загрузка изображений | KB picker в drawer — контекст теряется; нет AI-генерации; материалы скрыты в collapsed-секции |
| 3. Назначение и уведомление | Выбор учеников, Telegram-нотификация | Предупреждения о Telegram скрыты; нет preview сообщения; нет альтернативных каналов |

**Ключевые проблемы текущего флоу:**

1. **3 шага до отправки** — Google Classroom делает это за 2 клика.
2. **Задачи на шаге 2, отправка на шаге 3** — репетитор не видит финальный результат целиком.
3. **Нет экспорта** — собранное ДЗ нельзя скопировать текстом или скачать PDF.
4. **Единственный канал доставки** — Telegram-бот. Если ученик не подключён — тупик.
5. **Нет preview** — репетитор не видит, как ДЗ выглядит для ученика, до отправки.
6. **Нет быстрого пути** — даже для простого ДЗ из 3 задач нужно пройти все 3 шага.
7. **Image upload ненадёжен** — fallback-bucket, нет retry, нет прогресса.

## 2. Целевое состояние

### 2.1. Идеальный сценарий (happy path)

```
Репетитор открывает «Домашки» → «Создать ДЗ» →
→ Вводит тему и класс/ученика →
→ Добавляет задачи (из базы / ручной ввод / AI) →
→ Видит preview ДЗ →
→ Нажимает «Отправить» →
→ Ученик получает ДЗ (Telegram / ссылка / текст)

Время: 3–5 минут для 5 задач
Клики: ≤6 от входа до отправки
```

### 2.2. Принципы (из doc 16)

| Принцип | Как применяем |
|---------|--------------|
| Один экран = одна работа (П2) | Один экран для сборки + отправки |
| AI output → action (П5) | Задачи из AI сразу добавляются в ДЗ |
| Progressive disclosure (П7) | Базовый путь: тема + задачи + отправить. Расширенный: дедлайн, инструкция, режим |
| Export = часть workflow (П17) | Экспорт доступен без перехода на другую страницу |
| 2 тапа до формы (П8) | Домашки → Создать ДЗ |

## 3. Новый дизайн: Single-Page конструктор

### 3.1. Почему отказываемся от 3-шагового визарда

| Визард (текущий) | Single-page (новый) |
|------------------|--------------------|
| 3 шага, потеря контекста между ними | Всё на одном экране с секциями |
| Кнопка «Далее» не несёт ценности | Каждое действие — это прогресс |
| Нельзя вернуться к задачам после шага 3 | Всё редактируемо до отправки |
| Нет preview | Preview = сам экран |
| 6+ кликов | 4–6 кликов |

| Уровень | Что видно | Когда |

## 5. Отправка (Submit Flow)

### 5.1. Что происходит при нажатии «Отправить ДЗ»

**Единый атомарный submit** (текущие 4 фазы объединяются):

```
Пользователь нажимает «Отправить ДЗ»
  ↓
UI: кнопка меняется на «Отправляем...» + спиннер
  ↓
1. Валидация клиентская (тема, задачи, получатели)
  ↓
2. API: createAssignment (метаданные + задачи + материалы)
  ↓
3. API: assignStudents (список student_ids)
  ↓
4. API: notifyStudents (Telegram + fallback info)
  ↓
UI: Inline success state с результатами по каждому ученику
```

### 5.2. Telegram notification — улучшения

**Текущее:**
```
📚 Новое домашнее задание: Кинематика
Предмет: Физика
<a href="...">Открыть ДЗ</a>
```

**Новое:**
```
📚 Новое ДЗ: Кинематика
5 задач · Сдать до 20 марта

Открыть и решить: [ссылка]

Решай задачи в чате с AI — он проверит и подскажет!
```

Изменения:
- Количество задач (мотивация: «всего 5, не страшно»)
- Дедлайн (если указан)
- CTA: «Открыть и решить» вместо просто «Открыть ДЗ»
- Подсказка про AI-чат (для guided_chat mode)

### 5.3. Fallback для учеников без Telegram

| Ситуация | Действие |
|----------|----------|
| У ученика нет Telegram | Показать warning при выборе + предложить ссылку |
| Бот заблокирован учеником | Toast: «Пётр заблокировал бота. Отправьте ссылку вручную» |
| Все ученики без Telegram | Экспорт текстом / PDF как primary action вместо «Отправить» |

## 9. Acceptance Criteria

### Single-page конструктор

```
□ Все секции на одной странице (нет step-навигации)
□ L0: только Тема + Кому + Задачи + Отправить
□ Расширенные параметры скрыты по умолчанию
□ Название auto-генерируется
□ 4–6 кликов от входа до отправки (L0 path)
□ Один primary CTA на экране: «Отправить ДЗ»
□ KB picker работает без изменений
□ Image upload работает без изменений
□ Все провенанс-поля KB сохраняются
```

### Поле «Кому»

```
□ Группы и ученики в одном dropdown
□ Telegram-статус виден при выборе
□ Warning для учеников без Telegram
□ Fallback-предложение (ссылка) — не тупик
```

### Экспорт

```
□ Dropdown [Экспорт ▾] виден на create и detail
□ Copy text — формулы Unicode, нумерация, [см. рисунок]
□ PDF — A4, формулы рендерены, кириллица
□ ≤2 клика до результата экспорта
```

### Submit и Success

```
□ Единый submit (create + assign + notify)
□ Inline success state (не redirect)
□ Per-student delivery status
□ Кнопка «Скопировать ссылку» для учеников без Telegram
```

### Технические

```
□ npm run lint — pass
□ npm run build — pass
□ npm run test — pass
□ Нет framer-motion в новых компонентах (кроме page-level)
□ Lazy import для jsPDF + html2canvas
□ React Query keys: ['tutor', 'homework', ...]
□ Structural breakpoints: md: для grid, не sm:
```
