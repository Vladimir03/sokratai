## Ядро (КЛЮЧЕВОЙ инвариант)
**AI судит только узлы блок-схемы (да/нет/частично) — балл считает КОД** (`walkPhysicsFlowchart`), детерминированно проходя блок-схему ФИПИ. Зеркало русского cascade: балл модели НЕ доверяется. Схема ≠ сумма-таблица, а дерево решений.

Пайплайн: **эталон** (AI решает сам если нет от репетитора; eager при создании ДЗ, кэш на задаче, tutor-only, confidence) → **сравнение** ученика с эталоном → **узлы-суждения** (1 мультимодальный вызов, без балла от модели) → **`walkPhysicsFlowchart`** → балл + трасса → **отображение** (трасса по схеме: узлы ✓/частично/✗ + итог). Ошибки: снижающие балл vs заметки записи (обе ученику; «указать на ошибку», не спойлер).

## Блок-схемы ФИПИ — ПОДТВЕРЖДЕНЫ ЕГОРОМ (2026-06-30)
Реализованы в `supabase/functions/_shared/physics-flowcharts.ts`. Узлы-суждения = interfaces там же.

**№21 качественная (макс 3):**
- 3 — верный ответ + полное объяснение + без ошибок
- 2 — верный ответ + [полное объяснение, но есть ошибка] ИЛИ [не хватает 1 закона/явления]
- 1 — [верный ответ, не хватает 2+ законов/явлений] ИЛИ [неверный ответ, но верные рассуждения]
- 0 — неверный ответ + неверные рассуждения

**№22–23 расчётная (макс 2):**
- 2 — все формулы/законы + верный ответ+размерность + верные обозначения/преобразования/вычисления + нет лишних записей
- 1 — все формулы + одно из: [лишние записи] / [ошибка в обозначениях-преобразованиях-вычислениях при верном ответе] / [ответ-размерность неверны, но преобразования верны]
- 0 — нет всех формул ИЛИ (ответ неверен И преобразования неверны)

**№24–25 расчётная (макс 3):**
- 3 — все формулы + верны общ.формула/размерность/ответ + верны обозначения/преобразования/вычисления
- 2 — все формулы + [верны общ.формула/разм/ответ, недочёт в обозначениях-преобр-вычисл] ИЛИ [общ.формула/разм/ответ неверны, но преобразования верны]
- 1 — [все формулы + общ.формула неверна + преобразования неверны] ИЛИ [одна формула с ошибкой/отсутствует, но преобразования верны]
- 0 — иначе (нет 2+ формул, или одна формула+преобразования неверны)

**№26 (макс 4)** = Критерий 1 обоснование (+1/0) + Критерий 2 расчёт (как №24–25). Итог = обоснование + расчёт.

**Адиабата (кейс Всеволода):** № 24-25-тип, ответ 3,7 кДж верен, но `ΔU=3/2·pV` вместо `3/2·Δ(pV)` → узел «обозначения/преобразования/вычисления» = НЕ верно → **2 из 3**. Холистика пропускала; walker ловит (тест «адиабата» в `test-physics-flowcharts.mjs`).

## Обоснование № 26 — чеклист Егора (все применимые пункты → +1; хоть один нет → 0)
Источник: «Как_писать_обоснование_для_26ой_задачи.pdf», версия 24.06.2025, Егор Блинов. Рекомендации ФИПИ-2022 + Демидова-2022. **Условный** — применяются только пункты, релевантные физике задачи. Идёт в node-prompt как эталон полноты обоснования (AI судит `justification_complete`). Ключевые пункты:

1. **2-й з-н Ньютона / ур. моментов / ЗСИ / ЗСЭ** → «рассмотрим задачу в СО, связанной с Землёй; будем считать её **инерциальной**».
2. **Вертикально/под углом только под силой тяжести** → тела = материальные точки, пренебрегаем сопротивлением воздуха, движение = **свободное падение**.
3. **Кинематика равноускоренного (не свободное падение)** → силы постоянны → движение равноускоренное.
4. **Движение тел + 2-й з-н Ньютона / ЗСИ / ЗСЭ** → «применяем 2-й з-н Ньютона, справедливый для материальных точек, т.к. тела движутся **поступательно**».
5. **Условия твёрдого тела:** (а) модель твёрдого тела; (б) не движется поступательно → Σ внешних сил = 0; (в) не вращается → Σ моментов относительно оси через т. A ⊥ плоскости = 0; (г) обосновать, что сила тяжести приложена к середине (напр. стержень однородный); (д) выбрать ось для моментов (указать точку).
6. **ЗСИ:** (а) вдоль оси нет внешних сил (какие силы дают проекцию 0, какие внутренние; пример: шарик на нити + пуля → за время удара нить не отклоняется → проекция натяжения на горизонталь = 0); (б) время взрыва/удара мало → импульсом внешней силы пренебречь → система **замкнута**; изменением импульса малых масс пренебрегаем.
7. **ЗСЭ:** «изменение мех. энергии в ИСО = работе всех **непотенциальных** сил (трение, сопротивление, натяжение нити, реакция опоры)»; указать потенц./непотенц.; «работа непотенц. сил = 0»; если натяжение/реакция — **доказать** ⊥ скорости в каждый момент.
8. **Тела на нити:** «нить невесома, масса блока ничтожна, трения в оси блока нет, трение о воздух отсутствует → модуль натяжения в любой точке один и тот же». (а) нерастяжимость тут НЕ писать (см. п.9); (б) «идеальные» без пояснений — балл не ставят. (пружина: лёгкая → силы упругости на концах равны по модулю).
9. **Связь ускорений:** «нить нерастяжима → модули ускорений [подвижного блока и m отличаются в 2 раза] / [равны]». Нужна только нерастяжимость / постоянство длины пружины.
10. **«Поверхность гладкая»** и модельные допущения → написать, как используете (гладкая → нет трения); пренебрегаем трением о воздух.
11. **Ищут вес (не N)** → вес и N связаны 3-м з-ном Ньютона; силы трения на 2 тела равны и противоположны (3-й з-н).
12. **Отрыв тела от поверхности** → в этой точке **N=0**.
13. **Угол падения = угол отражения** (абс. упругий удар) — пример доказательства: энергия сохраняется → сила трения = 0, N ⊥ поверхности; N≫mg (время удара мало); `m v₂ − m v₁ = N Δt`; N∥Oy → v₂ₓ=v₁ₓ; из ЗСЭ v₂=v₁ → |v₂ᵧ|=|v₁ᵧ| → угол падения = угол отражения.

## Data Model (Phase A, миграция additive)
`homework_tutor_tasks`: `ai_reference_solution TEXT NULL`, `ai_reference_confidence TEXT NULL` (low/medium/high), `ai_reference_status TEXT NULL` (pending/ready/failed), `ai_reference_generated_at TIMESTAMPTZ NULL`. **tutor-only** (anti-leak, как `solution_text`). **Инвалидация при правке `task_text`/`task_image_url`/`solution_text`** (Phase C: `referenceResetFieldsIfChanged` в `handleUpdateAssignment` сравнивает до/после → сброс `ai_reference_*` + regen).

**Узлы-трасса результата (Phase C, РЕШЕНО) → отдельный `homework_tutor_task_states.ai_nodes_json JSONB`** (миграция `20260704160000`, student-visible, column-GRANT). Shape `{ score, max_score, confidence, steps: [{node, verdict, note?}] }`. **НЕ reuse `ai_criteria_json`:** трасса = дерево решений (балл — исход ветки), а не Σ критериев → в языковой sum-table (которая суммирует score/max) она бы «врала» (напр. узел `Лишние записи: yes` = плохо, но зелёная галочка/сумма ввели бы в заблуждение). Отдельный столбец + отдельный компонент `PhysicsFlowchartTrace.tsx` = чистое разделение (mirror того, что физика — отдельный `evaluatePhysicsPart2` путь). Узлы приведены к положительной полярности (yes=выполнено) для единообразного ✓/⚠/✗.

## Инварианты
- **Балл — только код** `walkPhysicsFlowchart` (не модель). Модель даёт узлы-суждения.
- **Эталон tutor-only**; `solution_text` репетитора побеждает AI-эталон (AI не решает при наличии).
- **Latency:** узел-вызов = 1 вызов на кэш-эталоне (без re-solve) → мгновенно ученику. Эталон готов заранее (eager).
- **Numeric-гейт:** Часть 1 (№1-20) не затронута; `physicsFlowchartKind` → null для не-21-26.
- **Tone-split:** строгость в баллах/разборе; hint/chat — сократические.
- **Строгость = как реальный эксперт = баллы Егора** (истина — размеченные работы, Phase D).

## Eval (Phase D)
`scripts/eval-physics-grading.mjs`: размеченные Егором работы (адиабата + N по № 21-26: грубая ошибка / верный ответ+кривая запись / частичное / неверный закон) → метрики: совпадение балла AI vs Егора (exact + ±1) + recall по пометкам «что поправить». Истина = баллы Егора.
