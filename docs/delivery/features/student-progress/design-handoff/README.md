# Handoff: «Прогресс по ученикам + галочка „проверено“» (IA v1.0)

> Фича для **тьюторского** модуля Сократ AI (`/tutor/*`). Четыре связанных экрана,
> закрывающих сценарий репетитора Елены: увидеть, кто отстаёт → открыть ученика →
> проверить работу (подтвердить AI-баллы) → отправить отчёт родителю.

---

## Overview

Репетитор ведёт десятки учеников на разных треках (ЕГЭ / ОГЭ / школа). AI проверяет
сданные работы и выставляет **предварительные** баллы по задачам. Тьютор должен:

1. **Увидеть общую картину** — кто отстаёт от цели (риск удержания) и где скопился
   его бэклог проверки. → экран **Успеваемость**.
2. **Открыть конкретного ученика** — прогресс к экзаменационной цели в его родной
   шкале, список работ, мини-карты задач. → экран **Прогресс ученика (hero)**.
3. **Проверить одну работу** — подтвердить AI-вердикты по задачам (поштучно или
   bulk), при необходимости поправить балл; «на проверке» → «подтверждено». →
   экран **Проверка работы**.
4. **Отправить отчёт родителю** — публичная одностраничная ссылка `/p/:slug` с
   динамикой, картой тем и прогнозом, **только по подтверждённым** работам. →
   экран **Отчёт родителю**.

Ключевая ценность галочки «проверено»: **ученик и родитель видят балл только после
подтверждения тьютором**. AI-оценка — черновик для репетитора, не финальный балл.

## About the Design Files

Файлы в этом бандле — **дизайн-референсы, собранные в HTML** (React 18 через Babel-standalone
в браузере, без сборки). Это прототипы, показывающие **внешний вид и поведение**, а
**не продакшн-код для копирования**.

Задача: **воссоздать эти экраны в целевом коде Сократ AI** — стек репозитория
`github.com/Vladimir03/sokratai`: **Vite + React 18 + TypeScript + shadcn-ui + Tailwind +
Supabase**, иконки **lucide-react**, шрифт **Golos Text**. Используйте существующие
компоненты (`src/components/tutor/*`, `src/components/ui/*`) и токены (`tailwind.config.ts`,
`src/index.css`). Прототипы намеренно переиспользуют паттерны из репозитория —
`HeatmapGrid`, `EditScoreDialog`, `StudentsActivityBlock`, approve-экран пробника — их
на сборке нужно **переиспользовать, а не перерисовывать**.

## Среда и сквозные правила (читать первым)

### Треки и родные шкалы (НЕ смешивать)
На странице одного ученика трек **один**, и баллы показываются в его **родной**
единице. Не нормализуйте и не усредняйте разные шкалы в одно «среднее /5».

| Трек | Текущая метрика | Цель | Источник конвертации |
|---|---|---|---|
| **ЕГЭ** (11 кл) | тестовый балл 0–100 | напр. 80 | таблица ФИПИ первичный→тестовый |
| **ОГЭ** (9 кл) | оценка 2–5 | напр. 5 | первичный→оценка по порогам (предмет-зависимо) |
| **Школа** | оценка 2–5 | напр. 4 | ручной ввод, без таблицы |

- **ЕГЭ-конвертация** — точная таблица-lookup ФИПИ физика-2026 (45 первичных → 100
  тестовых), **без интерполяции**. См. `hero2/data.js` → `EGE_PHYS.map` (индекс =
  первичный балл, значение = тестовый). Пороги: `36` аттестат, `39` вуз.
- **Прогноз** всегда помечается `≈` (экстраполяция темпа, детерминированный линейный
  расчёт), и переводов ЕГЭ→школьная оценка не существует — не показывать.

### Цвет ячейки задачи = % от макс. балла (универсально для всех шкал)
Единый источник — `src/components/tutor/results/heatmapStyles.ts` в репозитории.
Пороги (tailwind 100/900 пары):

| Доля от макс | Класс | bg | fg |
|---|---|---|---|
| `≥ 0.8` | emerald | `#D1FAE5` | `#064E3B` |
| `0.3 – 0.8` | amber | `#FEF3C7` | `#78350F` |
| `< 0.3` | red | `#FEE2E2` | `#7F1D1D` |
| нет данных (`null`) | slate | `#F1F5F9` | `#94A3B8` |
| ждёт подтверждения | + рамка | `inset 0 0 0 1.5px #E8913A` | — |

### Anti-leak (критично для проверки и отчёта)
Подтверждение балла **не раскрывает** ученику/родителю: AI-рубрику, разбор,
подсказки, полное решение. Ученик видит **только итоговый балл + «проверено»**.
Родительский отчёт — **без решений, рубрик, AI-вердиктов**, только итоги и динамика.

### Два разных сигнала (не объединять в одно «внимание»)
- **«Отстаёт» (риск удержания)** — далеко от цели и/или падающая динамика. Красный.
- **«Ждёт моей проверки» (бэклог)** — на проверке / просрочки. Янтарный.
Смешение даёт шум («40 из 62 требуют внимания») и теряет дисциплинированных-но-отстающих
учеников (сценарий, на котором Елена теряла ученика).

### Зелёный = семантика «успех / проверено / ≥80%»
Не используйте зелёный для нейтральных тегов (трек-чипы ЕГЭ/ОГЭ/Школа держите
нейтрально-серыми + иконка lucide для различения).

    ссылку».

## Кликабельный поток (happy-path Елены)

**Файл:** `flow/flow.js` (plain JS, без React). Активируется query-параметром `?flow=1`.

- Связывает 4 desktop-экрана: Успеваемость → Прогресс → Проверка → Отчёт.
- Рисует нижнюю плавающую ленту-навигатор (back + 4-шаговый брэдкрамб с подсветкой
  текущего шага), переходы сохраняют `flow=1` и прокидывают `track` / `work`.
- Точки навигации внутри экранов:
  - USP: клик по строке → `navStep('hero', {track})`.
  - Hero: на работах «на проверке/ручная» появляется кнопка **«Открыть проверку»** →
    `navStep('review', {work})` (маппинг названия работы → workId); поповер отчёта →
    `navStep('report', {track})`.
  - Review: после полного подтверждения в done-баннере — **«Перейти к отчёту родителю»**
    → `navStep('report', {track})`.
- **Без `?flow=1` каждый экран работает автономно** (нет ленты, стандартные CTA). Это
  демонстрационный слой навигации — в реальном приложении замените `flow.js` на роутер
  целевого стека (react-router и т.п.), сохранив те же переходы и передачу параметров.

## State Management

- **Успеваемость:** `sort` (attention|groups|goal|name), `filterMode` (null|risk|backlog),
  `sheetOpen` (mobile); список — мемоизированные `filtered`/`sorted`/`grouped`. Сигналы
  `risk`/`backlog`/`attnScore` вычисляются в `usp/data.js` → `compute()`.
- **Прогресс (hero):** `useHero2(profile)` — `works` (клон, мутируется при подтверждении),
  `target` (редактируемая цель), `toast`. Действия: `confirmWork`, `reopenWork`,
  `bulkConfirm`, `saveScore`. Состояние экрана — проп `state` (ready|loading|empty|error).
- **Проверка:** `workId`, `tasks` (клон), `sel` (all|index), `editing`, `bulkOpen`,
  `toast`. Действия: `confirmTask`, `reopenTask`, `bulkConfirm`, `saveTask`. Производное:
  `pending`/`manual`/`verified` → `overallState`.
- **Отчёт:** `trackKey`, `periodKey`, `comment` (editable), `toast`; mobile добавляет
  `screen` (report|expired) и `webview` (для деградации скачивания). Прогноз —
