## ЧАСТЬ A · Как работать с Claude design (рекомендованный процесс)

### Принцип: 1 экран за раз, от IA к полировке
Не проси «сделай весь интерфейс». Идёшь по экранам в порядке ценности, на каждом — сначала структура (что где), потом состояния, потом визуал. Так макет не разваливается и его можно валидировать на персонах.

**Порядок экранов:**
1. 🥇 **Страница ученика → вкладка «Прогресс»** (hero — закрывает P0-боль Елены).
2. **Обзор «Успеваемость»** (список всех учеников со статусом «проверено»).
3. **Галочка «проверено» в ДЗ** (паритет с approve пробника).
4. **Отчёт родителю** (предпросмотр + ссылка `/p/:slug` + картинка/PDF).

### Шаг за шагом

1. **Запинь контекст в начале чата с Claude design** (иначе получишь generic AI-slop):
   - дизайн-систему (см. Часть B — токены, Golos Text, Lucide, без emoji);
   - этот бриф + PRD;
   - **референс-файлы из репо:** `HomeworkResultsV2Mock.jsx` (твой стиль карточек/heatmap) и экран approve пробника (для паритета галочки).
2. **Формат артефакта:** одностраничный **HTML** (как `landing-trial-mockup.html`) для быстрых кликабельных макетов, либо **React+Tailwind (.jsx)** (как `HomeworkResultsV2Mock.jsx`) если нужна интерактивность (табы, раскрытие, toggle). Один файл, без внешних зависимостей кроме Lucide/Tailwind.
3. **Цикл итерации на каждом экране:** сгенерировать → посмотреть/снять скриншот (в т.ч. 390px iOS) → раскритиковать против 3 персон + дизайн-системы → уточнить. Подключай скиллы `design:design-critique` и `design:accessibility-review`.
4. **Валидируй на персонах прямо в промпте** (см. чек-лист в конце Части C).
5. **Mobile/Safari с самого начала:** проверяй на 390px; `font-size ≥16px` на инпутах; sticky-таблицы по `border-separate` (rule 80). Репетиторы часто на iPad/iPhone.
6. **Не финализируй полировку, пока не утверждена IA** двух развилок (Q1/Q2/Q3 из PRD §9). Макет нужен, чтобы их и решить.

### Готовые промпты (копипаст в Claude design)

**Промпт 0 — системный контекст (один раз в начале):**
> «Делаем экран для SokratAI — рабочее место репетитора физики/языков (ЕГЭ/DELF). Дизайн-система: шрифт Golos Text; палитра slate-800 (#1E293B) для заголовков/primary, socrat-green (#1B6B4A) для CTA/success/прогресса, amber-500 для "требует внимания", red-500 для ошибок, фон slate-50, карточки белые border slate-200 rounded-lg. Иконки только Lucide (без emoji в навигации/кнопках/заголовках). Один primary CTA на экран. Не cards-inside-cards. Mobile-first, проверь на 390px, инпуты ≥16px. Тон: серьёзный помощник, ближе к Brilliant/Khan Academy, не Duolingo. Выводи одностраничный React+Tailwind компонент без внешних зависимостей кроме lucide-react.»

**Промпт 1 — hero (страница ученика, см. Часть C §1):** дать структуру из C§1 + состояния.

> Совет: после каждого экрана проси Claude design выдать **3 варианта layout-а на выбор** для hero, затем дожимай выбранный.

## ЧАСТЬ B · Дизайн-система (жёсткие рамки — против AI-slop)

Из `.claude/rules/90-design-system.md` и `80-cross-browser.md`:

**Палитра (токены, не hex):** `text-slate-900` заголовки · `text-slate-500` body · `bg-accent` (#1B6B4A) CTA/прогресс/success · `bg-amber-500` дедлайны/«требует внимания» · `bg-red-500` ошибки · `bg-slate-50` фон · карточка `bg-white border-slate-200 rounded-lg`.
**Статус-цвета прогресса (уже de-facto в проекте — heatmap):** emerald ≥80% · amber 30–80% · red <30% · slate «—» нет данных.
**Шрифт:** Golos Text (400/500/600/700). Запрещены Inter/Roboto/Arial.
**Иконки:** Lucide React. **Без emoji** в навигации, заголовках, кнопках, бейджах. Галочка «проверено» — Lucide `CheckCircle2`/`BadgeCheck` (НЕ ✅).
**Кнопки:** один primary `bg-accent text-white` на экран; вторичные — white+border. Destructive — red.
**Статус-бейдж:** цветная точка + текст, НЕ цветная pill.
**Safari/iOS (rule 80):** sticky-колонка в таблице → `border-separate border-spacing-0` + `<colgroup>` фикс-ширины (НЕ `border-collapse`, НЕ `w-full`+`min-w` на `<td>`); `touch-action: manipulation` на кликабельном; `100dvh`; инпут ≥16px; `loading="lazy"` на img.

## ЧАСТЬ C · Экраны, состояния и компоненты

### 1. 🥇 Страница ученика → вкладка «Прогресс» (hero)

**URL:** `/tutor/students/:studentId`. **Шапка:** имя + фото ученика (соц.присутствие), группа, кнопка `Отчёт родителю` (вторичная, ведёт на вкладку «Отчёт»).

**Тело — «один взгляд по ученику»:**
- **Сводка сверху:** 3–4 метрики-плашки: «Сдано/всего», «Средний балл / динамика ↑↓», «% проверено», «Требует внимания N» (amber).
- **Список ДЗ + пробников** (карточки/строки, по дате убыв.), на каждой:
  - название · дата/дедлайн · балл (через `computeFinalScore`) · мини-heatmap задач (emerald/amber/red/slate) ·
  - **статус проверки:** `Проверено ✓ (Lucide BadgeCheck, accent)` / `На проверке N задач (amber)` / `Не сдано (slate)`.
  - клик → раскрытие задач этого ДЗ ИЛИ переход в существующий Results v2.
- **Ручные активности** (R5, может быть заглушкой v1): строки, добавленные репетитором вручную (внешний пробник, устный ответ) — визуально отличаются (иконка `Pencil`).

**Состояния:** пустой (ученик без ДЗ → «Назначьте первое ДЗ», Lucide `Inbox`); загрузка (skeleton, `animate={false}`); ошибка («Не удалось загрузить» + retry).
**Главный тест:** Елена должна за **один экран** понять «что у Васи сделано, что я проверила, что висит» — без обхода 15 домашек.

### 2. Обзор «Успеваемость» (подвкладка во вкладке «Ученики»)

- Таблица/список: строка на ученика → фото+имя · группа · сдано/всего · средний балл · **% проверено** · индикатор «есть непроверенное» (amber dot).
- **Фильтры:** «есть непроверенное», по группе (переиспользовать group-by из `StudentsActivityBlock`).
- Сортировка: по «требует внимания» / по группе.
- Клик по строке → страница ученика (§1).
- **Edge:** репетитор без групп → плоский список; 50+ учеников (Эмилия) → виртуализация/пагинация, не лагать.

### 3. Галочка «проверено» в ДЗ (паритет с пробником)

- **Эталон — экран approve mock-exam.** Та же кнопка-аффорданс: `Подтвердить` (primary accent) / `Изменить балл и подтвердить` (открывает существующий `EditScoreDialog`).
- На уровне задачи: после подтверждения — `BadgeCheck` + «Проверено» + лок (можно `Открыть обратно`, как reopen force-completed).
- **Bulk:** «Подтвердить всё, что AI проверил (N)» — один клик для физики (Елена/Вадим), где AI дал CORRECT.
- На уровне ДЗ: прогресс-полоска «Проверено M/N задач».
- **Состояния:** ничего не сдано (кнопка disabled + подсказка); не-физика без AI-вердикта (подтверждение вручную доступно — паритет с force-complete).

### 4. Отчёт родителю (вкладка «Отчёт» на странице ученика)

- **Предпросмотр** авто-собранного отчёта: сделано/баллы/динамика/период; **только подтверждённые** данные; БЕЗ решений/рубрик (anti-leak).
- **Редактируемый сопроводительный текст** (Вадим: «успехи такие-то, проблемы такие-то»).
- **Два выхода:** `Создать ссылку` → `/p/:slug` (read-only, без регистрации, как share-links) · `Скачать картинку/PDF` (одна страница, для пересылки в Telegram/VK) + `Скопировать текст` (fallback).
- **Развилка для макета (Q3):** что показываем родителю — баллы по темам? динамику-график? список ДЗ со статусами? Цель родителя — «вижу, что между уроками что-то происходит» + «обоснование стоимости», но «не вникает как». → лаконичный, «солидный» одностраничник.
- **Состояния:** нет подтверждённых данных («Сначала подтвердите хотя бы одно ДЗ»); ссылка истекла («срок истёк», не 404).

### Карта переиспользования компонентов
`HeatmapGrid`/`heatmapStyles` (цвета клеток) · `EditScoreDialog` (правка+подтверждение) · approve-паттерн mock-exam (визуал галочки) · share-link `/p/:slug` + print-CSS из `/preview` (картинка/PDF) · group-by из `StudentsActivityBlock` · `MathText` (формулы в задачах).
