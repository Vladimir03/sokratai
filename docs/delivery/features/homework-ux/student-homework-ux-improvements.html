Скриншот 1: Список ДЗ

  • Нет индикации прогресса (сколько задач решено)
  • Нет дедлайна на карточке
  • Одинаковые названия «ЕГЭ №1, 3» — не различимы
  • Бейдж «Назначено» не несёт полезной информации (все одинаковые)
  • Нет приоритизации — что решать в первую очередь?
  • Кнопка «Открыть» слишком большая, занимает половину карточки
Проблема

Скриншот 2-3: Guided solving

  • Условие задачи — маленькая картинка, трудно прочитать
  • Нет прогресса «3 из 8 задач решено»
  • Нет индикации сложности задачи или баллов на stepper
  • Кнопки «Предыдущая / Следующая» занимают много места
  • Нет индикации что AI «Введение» — это не обязательное к прочтению
  • Нет мотивации / геймификации (streak, прогресс)
  • Нет визуальной разницы между типами сообщений AI

Что хорошо уже сейчас

  • Свободный порядок задач (как на ЕГЭ) — ученик не заблокирован
  • Enter = безопасный «Шаг», Ctrl+Enter = «Ответ» — защита от случайной потери баллов
  • LaTeX рендеринг через MathText — формулы читаемы
  • Bootstrap (AI intro) для каждой задачи — ученик не остаётся один
  • Изображения задач с zoom — можно рассмотреть
  • Подсказки по кнопке — ученик контролирует уровень помощи

3.6. Загрузка фото, скриншотов и PDF в чат решения

Best practice: Photomath + Gradescope + ChatGPT

Для физики и математики текстовый ввод недостаточен — ученику нужно отправлять рукописные решения, графики, схемы. 80%+ решений в ЕГЭ физике пишутся от руки.

Проблема

  • Сейчас в guided chat можно отправить только текст — нет кнопки вложения
  • Ученик не может показать AI своё рукописное решение
  • Нет поддержки Ctrl+V / Cmd+V для вставки скриншота из буфера
  • Нет drag-and-drop для файлов на десктопе
  • Для физики это критично: формулы, чертежи, графики — всё пишется от руки

Техническое решение: WebApp Button vs URL

ПодходПлюсыМинусыРекомендация
Telegram WebApp Button
web_app: { url }		 Открывает сайт прямо внутри Telegram (как мини-приложение). Ученик не уходит из мессенджера. Фулл-скрин. Не все ученики знают что это. На десктопе открывает в sidebar (узко). Рекомендуемый для mobile
Inline URL Button
url: "https://..."		 Открывает в браузере (полноценный UI). Привычно. Работает везде. Ученик уходит из Telegram. Дополнительный переход. Fallback для десктопа

Рекомендация: использовать web_app button как основной (ученик остаётся в Telegram → сразу решает). URL ведёт на /homework/{assignmentId} — это guided workspace, не промежуточная страница.

Anti-spam правила

  • Максимум 3 уведомления в день на одного ученика
  • Не отправлять уведомления с 22:00 до 08:00 (часовой пояс ученика)
  • Streak-уведомления — максимум 1 раз в день
  • Если ученик уже решил ДЗ — не отправлять напоминание о дедлайне
  • Дедупликация через homework_tutor_reminder_log (уже реализовано)
  • Кнопка «Начать решать» → WebApp button (web_app URL), не обычная ссылка

Принцип: Telegram = трамплин → Сайт

Telegram-бот не должен быть самостоятельной средой решения ДЗ. Его задача — приглашать на сайт, где ученик получает полный guided experience с AI, LaTeX, изображениями, историей чата.
