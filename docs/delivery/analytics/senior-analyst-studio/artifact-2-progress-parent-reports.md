## 0. Заземление: какие данные по ученику реально есть [✓ РЕАЛЬНО]

### 0.1 Успеваемость и прогресс

- **`tutor-progress-api`** (edge): `GET /students/progress-overview` → per-student `pct_to_goal`,
  `reviewed_pct`, `signals` (`review_backlog` / `overdue` / `behind_goal` / `declining`);
  `GET /students/:id/progress` → `works[]`, `summary`, `current_level`, `trend[]`,
  `hw_done`/`hw_total`/`hw_overdue`/`hw_success_pct`.
- **`computeFinalScore`** (`_shared/score-compute.ts`): приоритет
  `tutor_score_override → earned_score → ai_score → status`.
- **Шкалы** (`scoreScales.ts`): ЕГЭ физика 2026 (первичный 45 → тестовый 100, порог 36 / «вуз» 39),
  ОГЭ (2–5). Реализованы и зеркалированы фронт/бэк.
- **`tutor_students`:** `target_score`, `exam_type`, `subject`, `status`, `archived_at`,
  `current_score`, `display_name`, `learning_goal`, `gender`.
- **Пробники** (`mock_exam_attempts`): `total_score` / `total_part1_score` / `total_part2_score`,
  `status` (in_progress / submitted / ai_checking / awaiting_review / approved / manually_entered).
  Подтверждённые = `approved` / `manually_entered`.

### 0.2 Сигналы активности и вовлечённости

| Сигнал | Поле/механизм |
|---|---|
| Первое открытие задачи | `homework_tutor_task_states.student_opened_at` |
| Лента действий | `recent-dialogs` `kind` = opened / wrote / submitted / **stuck** / completed (`stuck` = `wrong_answer_count` или `hint_count` ≥ 3) |
| Время на работу | `total_time_minutes` (wall-clock между первым и последним сообщением треда) |
| Усилия | `attempts`, `hint_count`, `wrong_answer_count` |
| «Требует внимания» (results) | итог < 30% макс **ИЛИ** переизбыток подсказок (`needs_attention`) |

### 0.3 Чего НЕТ [✚ СОБРАТЬ]

- ❌ **Посещаемости занятий.** `tutor_lessons` имеет только `status` (booked / completed / cancelled),
  без флага attended/no-show.
- ❌ Готового risk-score / предиктора оттока.
- ❌ Топик-уровня «что западает» для ДЗ (есть только `subject` / `topic` на задании; по № КИМ —
  только для пробника Часть 1).
- ❌ Метрики «темп vs срок до экзамена» (и, вероятно, самой даты экзамена — см. 4.1).

### 0.4 Отчёты родителям — текущее состояние [✓ существуют, детерминированные, БЕЗ AI]

- **`public-student-report` → `/p/report/:slug`** (bearer-slug, service_role, anti-leak whitelist).
  Конфиг в `student_report_links`: `verdict` (good/ok/attention), `tutor_comment`, тумблеры метрик,
  период (`all` / `last_month`), `show_debt_line`.
- **Содержимое:** вердикт-чип (репетитор ставит **вручную**), комментарий (пишет **руками**), числа
  (сделано ДЗ, % верных, балл пробника, `current_level` vs `target`, `trend`), авто-«что требует
  внимания» (**шаблонные** факты), баланс/история оплат.
- **AI/LLM в отчётах сейчас НЕТ** (подтверждено: ни Gemini, ни Claude в `public-student-report` и
  `_shared/student-progress-build.ts`). Всё детерминированно + ручной ввод.
  **Claude-агент — это и есть наш проект.**
- **Уже обезличено для родителя:** без UUID, без решений/критериев/внутренних заметок; ученик =
  `display_name` + класс + предмет; репетитор = только имя.
- Статус фичи: построена; деплой ⚠ частичный (подтвердить).

## 1. Спека Claude-агента отчётов родителям

**Назначение:** генерировать **черновик** отчёта (вердикт + человеческий комментарий + «что западает»
+ рекомендация) из обезличенных метрик. Репетитор подтверждает/правит перед отправкой —
**human-in-the-loop** (зеркалит текущий «репетитор пишет комментарий руками» и принцип Сократа
«AI = черновик + действие»). Ложится в существующий surface `ParentReportDialog` как pre-fill
`verdict` + `tutor_comment`.

**Модель:** Claude. Для батч-отчётов — `claude-sonnet-4-6`; сложные кейсы — Opus; system-prompt
кешировать между учениками (prompt caching).

**Вход** (обезличенный; из существующего `StudentProgressPayload` + опц. квалити-слой из Артефакта 1)
[✓ поля реальны, кроме помеченных]:

```json
{
  "summary": { "hw_done": 7, "hw_total": 10, "hw_overdue": 2,
               "hw_success_pct": 64, "current_level": 59, "target": 80,
               "trend": [52, 56, 59], "reviewed_pct": 80, "needs_attention": true },
  "works": [ { "kind": "homework", "subject": "physics", "score_kind": "primary",
               "raw": 6, "max": 10, "reviewed": true, "status": "verified",
               "date": "2026-06-10" } ],
  "weak_topics": [ { "topic": "Электродинамика", "success_pct": 40 } ],  // ✚ агрегировать
  "quality_layer": { "override_rate": 0.10, "check_failed_rate": 0.05 } // опц., Артефакт 3
}
```

**Не передаём:** имена, фото, тексты сообщений, решения задач, внутренние заметки, `ai_score_comment`.

**Выход** (структура = существующий отчёт, чтобы лечь в текущий surface):

```json
{ "verdict": "ok",                          // good | ok | attention (тот же enum)
  "comment": "2–4 предложения для родителя, дружелюбно, без жаргона",
  "covered": "что прошли (темы, объём работ)",
  "progress": "где относительно цели + динамика (59 → цель 80, тренд вверх)",
  "weak": "что западает: конкретные темы/типы задач + почему (по числам)",
  "recommendation": "одно конкретное действие" }
```

**Скелет промпта:**

```
SYSTEM:
Ты — ассистент репетитора, готовишь ЧЕРНОВИК отчёта РОДИТЕЛЮ ученика 16–18 лет.
Правила: дружелюбно и честно; без педагогического жаргона; адресат — родитель, не ученик.
Опирайся ТОЛЬКО на переданные числа — НИЧЕГО не выдумывай. Если данных мало, скажи прямо.
НИКОГДА не упоминай решения задач, критерии оценивания, внутренние заметки. Не называй имён.
verdict выбирай по pct_to_goal, динамике trend и числу просрочек:
  good — всё хорошо; ok — есть над чем работать; attention — нужен контроль.
Комментарий — 2–4 предложения.

USER:
<JSON входа>
Верни строго JSON по схеме {verdict, comment, covered, progress, weak, recommendation}.
```

**Прогон на нескольких учениках:** батч обезличенных payload → Claude → черновики → репетитор ревьюит
в `ParentReportDialog`. **Оценка качества черновика:** сверка с тем, что репетитор реально оставляет
после правки; критерии — фактологическая точность (не противоречит числам), тон, отсутствие утечек.

⚠ «Что западает» по темам сейчас **слабо заземлено**: для пробника Часть 1 слабость выводима по
№ КИМ, а для ДЗ топик-теги — только `subject`/`topic` задания. Нужно [✚] агрегировать слабые темы
(покрытие тегов — открытый вопрос 4.5).

## 2. Определение целевой метки «отстал» (3 рабочих варианта)

| Вариант | Определение (на реальных полях) | Плюсы | Минусы |
|---|---|---|---|
| **A. Динамика баллов** | `declining` (≥2 подряд подтверждённых пробника с падением scaled) **или** `hw_success_pct` ниже порога (напр. <60%) N недель подряд | Прямо к результату; `declining` уже считается | Пробники редки → лаг; success зависит от сложности задач |
| **B. Активность/вовлечённость** | рост времени-до-сдачи + нет `submission` M дней + рост `stuck`/`wrong`/`hint` + рост `overdue` | **Ранний** сигнал, частые данные | Шумно; активный ≠ успевающий и наоборот |
| **C. Цель vs темп** | `behind_goal` (`pct_to_goal < 50`, уже есть) **или** траектория `trend` не достигает `target` к дате экзамена | Бизнес-смысл (родитель платит за результат к ЕГЭ) | Нужны `target` (есть) и дата экзамена ⚠ (вероятно нет); школьники без пробников → нет `current_level` |

**Рекомендация:** комбинированный лейбл (any-of) для recall на пилоте + ручная валидация
репетитором. Старт: **B как раннее предупреждение**, **A/C как тяжесть (severity)**.

⚠ Оговорки: пробники подтверждаются вручную (approved/manually_entered) → лаг; `reviewed_pct < 100%`
→ часть баллов «сырые AI», не финал.

## 3. Фичи предсказателя «риск отстать»

| Группа | Фичи | Доступность |
|---|---|---|
| **Прогресс** | `current_level`, `target`, `pct_to_goal`, наклон `trend`, `reviewed_pct`, число пробников, давность последнего | ✓ / ⚠ агрегировать |
| **ДЗ-успеваемость** | `hw_success_pct` (rolling), final/max по ДЗ, доля completed, доля overdue, ср. `attempts`, ср. `hint_count`, `wrong_answer_count` | ✓ |
| **Активность** | дней с последней сдачи, частота сдач/нед, тренд `total_time_minutes`, давность `student_opened_at`, доля «открыл, но не сдал», частота `stuck` | ✓ / ⚠ агрегировать |
| **Занятия** | число booked/completed/cancelled, отмены ученика (⚠ **посещаемости нет** — только статусы) | ⚠ / ✚ |
| **Квалити-слой (от Кати-Дубай)** | `override_rate` (репетитор не доверяет AI на ученике → возможно грязные/сложные работы), `check_failed_rate` / `image_missing_rate` (косвенно «рукопись/фото плохо читаются»), доля `verdict=INCORRECT`, ср. `confidence`, доля `detailed_solution` | ✚ см. Артефакт 3 |
| **Контекст** | класс, `exam_type`, `subject`, срок до экзамена ⚠, длительность отношений (tenure) | ✓ / ⚠ |
