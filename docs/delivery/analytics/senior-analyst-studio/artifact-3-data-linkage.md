Легенда: **✓ РЕАЛЬНО** (есть в коде/схеме) · **⚠ ПОДТВЕРДИТЬ** · **✚ СОБРАТЬ**.

## 0. Заземление: связь сущностей и где живут ПДн [✓ РЕАЛЬНО]

### 0.1 Цепочка и гранулярность

```
auth.users (ученик)
  └─ homework_tutor_student_assignments  (grain: assignment × student)   ← ключевой линк
       └─ homework_tutor_threads          (1:1 с student_assignment)
            ├─ homework_tutor_task_states (grain: ученик × задача)        ← grain проверки
            └─ homework_tutor_thread_messages (per message: сдачи + фидбэк AI)
homework_tutor_tasks.id (UUID = канон; task_order = только отображение)
```

### 0.2 FK-дрейф (важно для соединения проектов) [⚠ известный источник багов]

- `homework_tutor_assignments.tutor_id` и `mock_exam_assignments.tutor_id` → `auth.users.id` **напрямую**.
- `tutor_students.tutor_id` → `tutors.id` (PK); для связки с auth-слоем нужна конверсия
  `tutors.user_id ↔ tutors.id`.
- **Общий атом ученика для обоих проектов = `auth.users.id`** (к нему сходятся обе цепочки:
  квалити-слой через `thread → student_assignment.student_id`; прогресс-слой через
  `tutor_students.student_id → profiles.id → auth.users.id`). Именно его хешируем.

### 0.3 Инвентарь ПДн (что маскировать)

| Категория | Поля | Действие |
|---|---|---|
| **Прямые ПДн** | `auth.users.email`; `tutors.name` / `telegram_*` / `bio` / `avatar_url`; `tutor_students.display_name` / `gender` | НЕ экспортировать |
| **Свободный текст (риск утечки личности)** | `thread_messages.content`, `tutor_students.learning_goal` / `notes`, все `*_comment`, `ai_score_comment`, `mock_exam_anonymous_leads.lead_name`/`lead_contact` | Решение владельца 2026-06-30: `ai_score_comment`/`tutor_score_override_comment` в выгрузке проверок **включаются как есть** (под NDA, нужны для оценки качества). Прочий free-text — НЕ экспортировать |
| **Фото/рукопись** | bucket `homework-submissions` (`storage://` ссылки) | НЕ экспортировать файлы/URL; только признаки (`photo_count`, `has_voice`) |
| **Безопасные ключи (opaque UUID)** | `student_assignment.id`, `task_id`, `thread.id`, `task_state.id`, attempt `id` | можно использовать для трассировки **внутри** проекта; наружу — только хеши |

> ⚠ Отдельный флаг владельцу (вне этих проектов): bucket `chat-images` (общий AI-чат) может быть
> **публичным** — потенциальная экспозиция изображений. Прямого отношения к проектам не имеет, но
> стоит проверить.

⚠ `profiles.full_name` — упоминается в коде как fallback имени; присутствие колонки в схеме
подтвердить. На дизайн не влияет (всё равно ПДн, не экспортируется).

## 1. Квалити-слой (Дубай) → фичи предиктора (Кипр)

**Несоответствие гранулярностей.** Квалити-слой живёт на уровне **(ученик × задача × ДЗ)**
(`task_states` / `hw_ai_check_events`); предиктор риска — на уровне **(ученик × окно времени)**.
Поэтому квалити-слой **агрегируется до per-student-week ДО подачи** в предиктор — передаются
агрегаты, не сырые проверки.

**Ключ соединения:** хешированный `auth.users.id` ученика → `student_anon` (см. раздел 2). Он
общий для обоих слоёв. Для трассировки внутри проекта Дубай — safe-UUID `student_assignment.id`,
`task_id`.

**Что именно передаётся (per `student_anon` × неделя, обезличенно):**

```
checks_count
false_accept_proxy   = override_down_count       // F1: правок вниз
false_reject_proxy   = override_up_count         // F2: правок вверх
override_rate, mean_override_delta
check_failed_rate, image_missing_rate            // прокси «грязная рукопись/фото»
verdict_mix          = {CORRECT%, ON_TRACK%, INCORRECT%}
mean_confidence
leak_retry_rate, hint_fallback_rate
detailed_solution_share, has_photo_share, voice_share
```

**Не передаётся:** тексты сообщений, фото/URL, имена, любые UUID в сыром виде.

**Почему это сильные фичи риска (гипотезы для проверки):** высокий `override_rate` → репетитор не
доверяет AI на этом ученике (сложные/грязные работы); высокий `image_missing_rate`/`check_failed_rate`
→ ученик «решает на бумаге», фото плохо читаются (косвенный сигнал трудностей и трения); рост доли
`INCORRECT` + падение `mean_confidence` → объективное ухудшение.

## 2. Подход к обезличиванию (несовершеннолетние, NDA)

| Приём | Правило |
|---|---|
| **Псевдонимизация** | `student_anon = HMAC/хеш(auth.users.id, salt)`; `tutor_anon = хеш(auth.users.id, salt)` (для `tutor_students` сначала конверсия `tutors.id → tutors.user_id`). Salt хранится под NDA, **не в репозитории**. Та же соль между выгрузками → ключи стабильны и сопоставимы. Сырые UUID наружу не отдаём |
| **Free-text** | Решение владельца: `ai_score_comment` / `tutor_score_override_comment` в выгрузке проверок включены как есть (под NDA — для оценки качества). Прочий свободный текст (`content` сообщений, `learning_goal`, `notes`, имена) — НЕ экспортировать |
| **Фото/рукопись** | НЕ экспортировать файлы/URL → только признаки. Если Кате-Дубай нужны фото для ручной разметки F4–F6 — отдельный **строго ограниченный** доступ к выборке под NDA (лучше: размечает доверенное лицо внутри, наружу идут только метки) |
| **k-анонимность** | Пилот мал (десятки учеников) → числа + класс + предмет + редкая дата могут реидентифицировать. Огрублять даты до недели/месяца, подавлять ячейки с малым N, не публиковать редкие комбинации (subject + class + tutor) |
| **Демография** | `gender` чувствителен (несовершеннолетние) → исключить или огрублять; использовать только если методологически необходимо |
| **Разделение проектов** | Дубай и Кипр делят **только** обезличенные агрегаты по общему `student_anon`; ни один проект не получает ПДн другого |

Реализация ретроспективного экспорта (с хешами по этой схеме) — `export-hw-check-anonymized.sql`.
Телеметрия — `telemetry-hw-ai-check-events.md` (та же соль на экспорте → стабильная связка).

## 3. Сводная схема потока данных

```
[ Сократ БД + телеметрия hw_ai_check_events ]
        │  (внутренний экспорт под NDA: хеширование + дроп free-text/фото)
        ▼
[ Слой А · Квалити проверки (Дубай) ]            [ Слой Б · Прогресс/активность (Кипр) ]
  grain: ученик × задача × ДЗ                      grain: ученик × окно времени
  → агрегация до student_anon × неделя                   │
        │                                                 │
        └───────────────►  student_anon  ◄───────────────┘
                                   │
                                   ▼
                 [ Предиктор «риск отстать» (Кипр) ]
                   фичи прогресса + активности + квалити-слоя
```

## 5. Принципы (одной строкой)

- Общий ключ соединения — **хеш `auth.users.id`**, не сырой UUID.
- Квалити-слой **агрегируется до (ученик × неделя)** перед подачей в предиктор.
- Наружу — **только числа/категории/огрублённые даты** (+ free-text комментарии под NDA по решению владельца); фото, имена, сырые UUID не покидают периметр.
- Каждый проект видит **только обезличенные агрегаты** другого.
