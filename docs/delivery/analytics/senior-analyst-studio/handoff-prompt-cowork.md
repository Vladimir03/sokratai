## Жёсткие ограничения (не нарушать)
- Ученики несовершеннолетние → только обезличенные данные под NDA, никаких ПДн в выгрузках/отчётах.
- AI-проверка ДЗ работает на **Gemini** (Lovable AI Gateway). Агент отчётов родителям задуман на **Claude**.
- Не выдумывать данные/метрики, которых нет. Чего нет — помечать «собрать/подтвердить».

## Два проекта
- **Катя-Дубай** — «Качество и ценность AI-проверки ДЗ».
- **Катя-Кипр** — «Успеваемость + отчёты родителям» (Claude-агент отчётов + предиктор «риск отстать»).

## Находки — проверка ДЗ (для Кати-Дубай)
- Гранулярность проверки: одна строка = (ученик × задача). ✓
- Хранится и запрашиваемо: `ai_score` (балл Gemini, шаг 0.5), `ai_score_comment` (tutor-only),
  `ai_criteria_json` (покритериально, ТОЛЬКО языки DELF/ЕГЭ EN/IELTS; физика = null), `earned_score`/
  `available_score`, `hint_count`/`wrong_answer_count`/`attempts`, полный текст разбора AI (как
  сообщение), сдача ученика (numeric/photos/text/voice). ✓
- НЕ хранится как колонка, исторически только в логах: `verdict` (CORRECT/ON_TRACK/INCORRECT/
  CHECK_FAILED), `confidence`, `error_type`, `failure_reason`. С 2026-06-30 пишется ВПЕРЁД в таблицу
  `hw_ai_check_events` (ретроспективы по этому слою нет). ⚠
- **OCR рукописи НЕ существует**: фото уходит в Gemini мультимодально в реальном времени,
  распознанный текст нигде не сохраняется. Legacy-таблицы classic-режима (с `recognized_text`) ДРОПНУТЫ
  — на них не закладываться. ✓
- Главный готовый сигнал ОШИБКИ AI = правка репетитора: `tutor_score_override` ≠ `ai_score` (вниз =
  ложно-верно, вверх = ложно-ошибка) + `tutor_force_completed` + `tutor_reviewed`. Единственный
  персистентный «лейбл правды». ✓
- Уже есть код-защиты + телеметрия по: утечке решения, бесполезной подсказке, «фото условия не
  загрузилось» (fail-closed), инфра-сбою → их частоту можно мерить из логов. ✓
- НЕ отлавливается автоматически (нужна ручная аннотация): ложно-верно/ложно-ошибка по сути,
  неверный пошаговый разбор, сбой распознавания рукописи, тон/сложность для возраста. ✚

## Находки — успеваемость и отчёты (для Кати-Кипр)
- Прогресс (edge `tutor-progress-api`): `pct_to_goal`, `reviewed_pct`, сигналы `review_backlog`/
  `overdue`/`behind_goal`/`declining`; per-student `works[]`, `current_level`, `trend[]`. ✓
- Балл: приоритет `tutor_override → earned → ai → status`. Шкалы ЕГЭ физика 2026 (первичный 45 →
  тестовый 100, порог 36/«вуз» 39) и ОГЭ (2–5) реализованы. ✓
- Активность: `student_opened_at`, лента `recent-dialogs` (opened/wrote/submitted/stuck/completed;
  stuck = wrong/hint ≥ 3), `total_time_minutes`. Пробники `mock_exam_attempts` (total_score, status). ✓
- Чего НЕТ: посещаемость занятий (только booked/completed/cancelled, без attended/no-show); готовый
  risk-score; топик-теги «что западает» для ДЗ (есть subject/topic задания; по № КИМ — только пробник
  Часть 1); вероятно нет даты экзамена ученика. ✚/⚠
- Отчёты родителям УЖЕ существуют, но детерминированные: вердикт-чип + комментарий репетитор
  ставит/пишет руками; числа + шаблонное «что требует внимания»; AI/LLM не используется. Anti-leak
  уже встроен (без решений/критериев/имён/uuid). Claude-агент отчётов = greenfield (его и проектирует
  Кипр; модель работы — AI генерит ЧЕРНОВИК → репетитор подтверждает/правит перед отправкой). ✓
- Метка «отстал» — 3 кандидата: A динамика баллов (declining / падение HW success); B активность (нет
  сдач M дней, рост stuck/overdue) — ранний сигнал; C цель/темп (behind_goal / траектория не дойдёт до
  target к экзамену). Рекомендация: any-of + ручная валидация. Ground truth метки пока НЕ определён. ✚

## Находки — связка проектов + обезличивание (общее)
- Общий ключ соединения двух проектов — хеш `auth.users.id` ученика (к нему сходятся обе цепочки).
  FK-дрейф: `tutor_students.tutor_id → tutors.id` (нужна конверсия через `tutors.user_id`), а
  `homework`/`mock assignments.tutor_id → auth.users.id` напрямую. ✓
- Квалити-слой (ученик × задача) агрегируется до (ученик × неделя) перед подачей фичами в предиктор
  риска. Передаются агрегаты (override_rate, check_failed/image_missing_rate, verdict_mix,
  mean_confidence, доля detailed_solution/фото), не сырьё. ✚
- Обезличивание: HMAC-хеш id; удалить весь free-text сообщений/имён → заменить структурными фичами
  (комментарии проверки `ai_score_comment` — по решению владельца включаются под NDA); не
  экспортировать фото/URL; k-анонимность (огрублять даты, подавлять малые ячейки); gender
  исключить/огрублять. ✓ (принципы)
