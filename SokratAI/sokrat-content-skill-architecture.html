3 · Pipeline: 7 этапов работы скилла

Один полный запуск (weekly cycle) занимает ~45-60 минут вычислительной работы скилла + 5-10 минут пользовательского контроля через AskUserQuestion gates.

1
Bootstrap
~5 сек · автоматический
Проверяет state-файлы. Если brief.md существует → subsequent run, переход к §3. Если нет → §2 (первый запуск).
2
Discovery (first run only)
~3 мин · 1 AskUserQuestion call
5 вопросов в одном button-сообщении: KPI ("trial signups"), traffic ("TG Егора + lead-magnet"), production ("solo + Higgsfield"), audience split ("40/35/25"), Egor Soul status ("trained / training / N/A").
3
Auto-research
~5 мин · parallel
Seeded competitors: Школково, Долгих, Бакаев, Гущин, Решу ЕГЭ. Plus shadow hunt + Higgsfield capability probe (cache hook/setting UUIDs).
4
Collection
~15 мин · parallel subagents
Pull 20-30 top videos per competitor. Filters: placement + size <500k + recency <60d. Compute engagement_ratio, vs_account_median, save rate.
5
Analysis
~10 мин
7-axis decomposition top 5-10 видео. Synthesize trend report (trends/YYYY-MM-DD.md): top hooks, formats, sounds, audience pains from comments.
6
Ideation
~10 мин · 1 AskUserQuestion gate
10 hook ideas mapped to 5 Sokrat formats with Higgsfield preset+hook+setting+avatar resolved. Каждая идея: IG-version + TG-version (cross-post).
7
Production handoff
~5 мин · 1-2 AskUserQuestion gates
Top 4-5 ideas → Higgsfield generation queue (с per-batch permission gates). Image pack 40/20/20/20 для landing/social. Final output: видео + post copy + CTA.
8
Log experiments
~2 мин · async (после публикации)
После публикации — пользователь вызывает "log experiment". Метрики + verdict (SCALE/ITERATE/KILL) + learning. Winner ≥1000 views на cold аккаунте → auto-prompt variant generation в следующем запуске.
Production handoff (Step 7) — ключевое отличие от обоих родителей

trendwatch выдаёт ideas/YYYY-MM-DD.md и останавливается. Higgsfield content factory сразу пытается генерить 50-200 видео в Meta Ads. sokrat-content делает середину: 4-5 идей в неделю, генерация с per-batch permission, output ready для manual posting в IG/TG. Контроль остаётся у Vladimir.

Student-AI dialogue showcase
Pillar C · ~15-20% (overlap with Format 3)
Higgsfield presetugc split-screen

Save rate target5-10% (research-backed = high credibility)

Hook example"Что мой ученик пишет AI в 23:00. Скриншоты с согласия."

Почему 5, и почему именно эти

Distribution следует IG Playbook v1: A (40%) + B (30%) + C (20%) + D (10%). Format 5 — overlap с C, специально для maximum save rate (research-backed contentinent). Higgsfield presets выбраны по преcedent map (см. Higgsfield SKILL §"Preset Relevance Map"). Никаких UGC форматов (Unboxing/ASMR/Street Interview) — они не транслируются на EdTech.

Самый важный файл — experiments.md

Здесь компаундируется вся ценность скилла. Каждый постнутый Reel логируется с метриками. Через 3 месяца у Sokrat будет уникальный dataset: "что работает для AI-инструмента для физика-репов в РФ". Этого никто не повторит.

7 · Before / After: что меняется в твоей работе

❌ Без custom skill (текущее состояние)

  • Vladimir вручную пишет hook'и и сценарии для 4-5 Reels каждую неделю (5-7 часов)
  • Trend-watching конкурентов делается ad-hoc, без записи
  • Higgsfield prompts строятся "на глаз", иногда выходит UGC mode когда нужно tutorial
  • Каждый Reel — separate decision: какой формат, какой hook, какой CTA
  • Опыт не накапливается: что сработало неделю назад — Vladimir может забыть
  • IG-version и TG-version пишутся отдельно (~2x работа)
  • Avatar в Higgsfield каждый раз рандомный → нет визуальной консистентности
  • Total Vladimir time per week: ~20-25 часов

✓ С custom skill (target state)

  • Vladimir запускает скилл раз в неделю. Получает 10 ideas с готовыми Higgsfield prompts + slug + avatar + hook UUIDs.
  • Trend-watching автоматический, 5 конкурентов мониторятся постоянно
  • Higgsfield prompts строятся через cached capability map → 0 slug-mismatch errors
  • Каждая idea — pre-mapped на 1 из 5 Sokrat formats с правильным CTA
  • experiments.md накапливает знание. Winners auto-trigger variant generation
  • Каждая idea имеет BOTH IG-version + TG-version (cross-post ready)
  • Egor Soul Character locked → визуальная консистентность через все production
  • Total Vladimir time per week: ~6-8 часов (3-4x экономия)
Где скилл НЕ помогает

Скилл не снимает реальный footage Егора (Sunday recording сессия остаётся за ним). Не пишет финальные captions для постов (требует human voice). Не решает за тебя стратегические вопросы (audience split, pricing, продуктовые приоритеты). Не отвечает на DMs от подписчиков. Это amplifier твоих решений, не замена.

8 · Build steps: как имплементировать

Скилл — это один markdown файл SKILL.md + state-files. Без code, без npm install. Создаётся итеративно.

Фазы имплементации

  1. Фаза 0 — Установить trendwatch как baseline (15 мин). Скопировать SKILL.md в ~/.claude/skills/trendwatch/. Запустить на @grand_phys, ответить на 5 discovery questions, получить первый brief.md + ideas/. Это baseline, дальше расширяем.
  2. Фаза 1 — Создать sokrat-content/SKILL.md (1-2 часа Vladimir + Claude). Скопировать trendwatch's SKILL.md, переименовать. Добавить:
    • Section "5 Sokrat formats" (заменяет UGC formats)
    • Section "Higgsfield production layer" (cherry-picked из higgsfield-content-factory)
    • Section "Cross-post handoff" (новое)
    • Updated "Tactical playbook" с EdTech-specific patterns
  3. Фаза 2 — Pre-seed state files (2-3 часа).
    • brief.md — заполнить вручную из Marketing Strategy v1.1
    • competitors/ — 5 файлов с initial research (Школково, Долгих, Бакаев, Гущин, Решу ЕГЭ)
    • formats.md — детальный mapping 5 форматов
    • audience-personas.md — JTBD для tutors/students/parents
  4. Фаза 3 — Train Egor Soul Character (~1 час Егор + setup). Сессия фотографирования Егора: 10-15 портретов в разных ракурсах/освещении/одежде. Upload в Higgsfield Soul training. Сохранить master.png в mascots/01-egor-soul/.
  5. Фаза 4 — Запись первой партии screencasts (1 час Vladimir + Egor). 4 ключевых scenario:
    • Создание ДЗ (Sokrat constructor)
    • AI проверяет ответ ученика
    • Диалог ученика с AI (с согласия)
    • Mock exam parent report
    Save в screencasts/.
  6. Фаза 5 — Build lead-magnets (1-2 дня).
    • 50-ege-tasks.pdf (Vladimir + Claude assemble из KB)
    • tutor-kit-26-tasks.xlsx (export из Sokrat KB)
  7. Фаза 6 — First run end-to-end (~60 мин). Запустить sokrat-content. Прогнать full pipeline на real data. Опубликовать первые 4-5 Reels. Залогировать через "log experiment".
  8. Фаза 7 — Iterate (continuous). Каждую неделю — один запуск. После 4 weeks — Vladimir + Claude ревью experiments.md, добавляет cross-vertical learnings в Tactical playbook.

Estimated calendar

  • День 1 (сегодня): Фаза 0 (trendwatch install) + Фаза 1 (sokrat-content SKILL.md draft)
  • День 2-3: Фаза 2 (pre-seed) + Фаза 3 (Egor Soul training)
  • День 4-5: Фаза 4 (screencasts) + Фаза 5 (lead-magnets)
  • День 6-7: Фаза 6 (first run, publish, log)
  • Неделя 2+: Фаза 7 (continuous iteration)
Total upfront investment

~15-20 часов distributed по 7 дням (~3 часа/день). После этого — 6-8 часов в неделю на actual content production. ROI: после 4-х недель скилл начнёт компаундировать через experiments.md, и weekly time снизится до 4-5 часов.
