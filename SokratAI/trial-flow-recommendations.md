# Trial-флоу для репетиторов sokratai.ru — рекомендации

**Контекст:** добавляем 7-дневный бесплатный trial с полным доступом к AI перед существующим тарифом «AI-старт за 200 ₽ первый месяц». Лендинг и инфраструктура существующая, надо доточить.

**Документ к макетам:** `landing-trial-mockup.html` рядом с этим файлом — открыть в браузере, смотреть параллельно с этим текстом.

---

## TL;DR

1. **Формула победителя в RU EdTech: 7 дней бесплатно (без карты) → 99–200 ₽ первый месяц → стандартный тариф.** CloudText.ru делает ровно это: 7 дней full + 99 ₽ первый месяц вместо 899 ₽. Это твоя референсная модель.
2. **Trial без карты — обязательно для пилотного продукта.** RU-аудитория не привыкла привязывать карту «на пробу». Ты теряешь больше signup-ов, чем conversion-ов от auto-charge. Через 7 дней — явная просьба продолжить за 200 ₽, не списываем сами.
3. **После trial — soft demotion на free-tier**, не lockout. У тебя уже есть бесплатная базовая платформа (оплаты + расписание + профили). Soft demotion = trial-юзер не теряет всё, остаётся в системе → его легче вернуть позже.
4. **Dual-CTA уже есть, копию надо обновить.** Hero CTA меняется на «🎁 Попробовать 7 дней бесплатно» (не «за 200 ₽»). 200 ₽ становится <em>следующим шагом после trial</em>, не первым препятствием. Telegram-кнопка остаётся, но визуально стилизуется под фирменный TG-blue, чтобы её сразу узнавали.
5. **Pricing-блок упрощается с 5 карточек до 3 + 2 дополнительных.** Карточка «AI-старт» теперь рассказывает <em>всю воронку</em> trial → 200 ₽ → 1000 ₽/мес как лестницу.

## Что нашёл в исследовании конкурентов

### CloudText.ru — точный аналог
- **7 дней бесплатно с полным функционалом, без ограничений**
- **99 ₽ первый месяц вместо 899 ₽** (anchor-скидка −89%)
- Альтернативное предложение: **«1 ₽ первый месяц»** для тех, кто не верит в trial — психологически платный, но почти бесплатно
- Цена не зависит от числа учеников
- Источник: [cloudtext.ru/pay](https://cloudtext.ru/pay)

### Мой Класс (moyklass.com) — паттерн soft demotion
- 7 дней пробного периода, потом **переход на бесплатный тариф** (до 5 групп)
- Данные сохраняются — пользователь не теряет работу
- Это <em>не</em> trial с hard cutoff, это freemium с trial-окном на платные функции
- **Это паттерн, который ты должен копировать** — у тебя уже есть free-tier с базой, тебе осталось добавить «AI на 7 дней»
- Источник: [moyklass.com](https://moyklass.com/)

### RubiTime — формулировка «без карты»
- 7 дней бесплатно, **«без карты и обязательств»** — буквально в копии
- Это psychological barrier, который снижает abandonment в воронке
- Источник: [rubitime.ru](https://rubitime.ru/crm-dlya-repetitorov)

### OkoCRM — продление по запросу
- 7 дней + ещё неделя по запросу
- Низкая стоимость, повышает вероятность активации в неделе 2

### Антитренинги — щедрее
- 14 дней с полным функционалом
- Это работает для CRM, но для AI-продукта (где compute-стоимость высокая) 7 дней — оптимально

### YCLIENTS, BigBen, Umai — все 7 дней
- 7 дней — индустриальный стандарт RU EdTech CRM. Не 14, не 30. Не отклоняйся.

## Рекомендуемый флоу

> Полная диаграмма в макете — Surface 7. Ниже — текстовое описание.

### Path A — «Готов попробовать» (CTA «7 дней бесплатно»)

```
Лендинг (Hero/Pricing/FinalCTA)
    ↓ клик «🎁 7 дней бесплатно»
Signup-форма (single-page)
    email + пароль (или TG OAuth)  ←  карты НЕТ
    «Что преподаёте?»  ←  для onboarding
    оферта + чекбокс
    ↓
[День 0] Welcome: онбординг-тур, «создай первое ДЗ за 3 минуты»
[День 1] Push: «Создал первое ДЗ?»
[День 3] Email: кейс репетитора-донора
[День 5] Email + push: «Осталось 2 дня — попробуй ещё фичу X»
    ↓
[День 7] GATE: модалка при первом входе
    «Trial завершён. Спасибо. Вот что вы получили: N ДЗ, M часов сэкономлено»
    [Primary CTA] Продолжить за 200 ₽ первый месяц → ЮKassa
    [Secondary CTA] Остаться на бесплатной (только оплаты + расписание)
    ↓
Если ДА → AI активен ещё месяц по 200 ₽ → потом autocharge по числу учеников (1000/2000/3000)
Если НЕТ → AI выключается, free-tier остаётся
    [День 14] Email: «N учеников ждут AI-проверки — вернуться?»
    [День 30] Email: финальный nudge
    Дальше — пассивно: видит «Активировать AI» в кабинете
```

### Path B — «Пока изучаю» (CTA «Канал Егора»)

```
Лендинг
    ↓ клик «Канал Егора →»
Telegram канал @sokrat_rep
    ↓ /start (если бот ещё не знаком)
Welcome-серия от бота с inline-кнопками:
    [🎁 Попробовать в кабинете 7 дней]  →  редирект на /signup?ref=tg-bot
    [📚 3 поста о методе]                 →  показывает pinned посты
    [❓ Задать вопрос Егору]               →  переадресация в личку Егора
```

### Принципиальные решения

| Решение | Выбор | Почему |
|---|---|---|
| Карта на trial | **Нет** | RU-юзеры не привязывают карту «на пробу». Конверсия signup×3 важнее, чем conversion paid×0.7 |
| Длительность | **7 дней** | Индустриальный стандарт RU CRM. Не 14, не 30 |
| Что включено | **Полный AI, до 20 учеников** | CloudText/moyklass дают full. Limited trial = anti-pattern |
| После trial | **Soft demotion на free** | У тебя уже есть free. Lockout = lost touchpoint навсегда |
| Auto-charge через 7 дней | **Нет** | Без карты невозможно технически, и это нам выгодно |
| Цена после trial | **200 ₽ первый месяц → 1000/2000/3000** | Уже есть, оставляем. 200 ₽ становится anchor-скидкой |
| Extension по запросу | **Phase 2** | OkoCRM делает, но это операционная нагрузка. Запустим без, добавим если будут запросы |

## Backend / DB (новое)

Существует только примитивный trial. Нужно добавить:

### Таблица `tutor_subscriptions` (если ещё нет)

```sql
create table tutor_subscriptions (
  user_id uuid primary key references auth.users(id) on delete cascade,
  status text not null check (status in ('trial_active','trial_expired','paid','cancelled','free')),
  trial_started_at timestamptz,
  trial_expires_at timestamptz,  -- trial_started_at + 7 days
  paid_until timestamptz null,
  current_tier text null check (current_tier in ('ai-start-200','ai-plus-1000','ai-pro-2000','ai-team')),
  trial_decision text null check (trial_decision in ('paid','declined','no_action')),
  trial_decision_at timestamptz null,
  -- AI-feature gating: edge-functions проверяют это поле
  ai_features_active boolean generated always as (
    status in ('trial_active','paid')
  ) stored,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

create index idx_tutor_subs_trial_expiring on tutor_subscriptions(trial_expires_at)
  where status = 'trial_active';
```

### Cron-напоминалки (новый edge function `trial-reminders`)

Запускать ежедневно. Логика:

```ts
// День 1, 3, 5, 7 после trial_started_at — каскад push → email (по правилам 70-notifications.md)
// День 7 — переключить status → trial_expired, отправить дайджест-email "вот что вы получили"
// День 14, 30 — nudge для тех, кто не оплатил (status=trial_expired, decision=declined|no_action)
```

### AI-feature gating

Все edge-функции, использующие AI (`chat`, `homework-api/handleCheckAnswer`, `handleRequestHint`), должны:

```ts
const sub = await db.from('tutor_subscriptions').select('ai_features_active').eq('user_id', tutor_id).single();
if (!sub?.ai_features_active) {
  return { error: 'ai_disabled', message: 'Trial завершён. Активируйте подписку для AI-функций.' };
}
```

Текущий free-tier (оплаты, расписание, профили) **не должен** упасть при `ai_features_active=false` — это точка soft demotion.

## Открытые вопросы — что мне нужно от тебя

> Это блокирующие вопросы перед написанием кода. Ответ на каждый — ровно один абзац.

### 1. **Карта или без карты на trial?**
   Я рекомендую без карты. Но если ты хочешь conversion×1.5 за счёт auto-charge — нужно интегрировать ЮKassa токенизацию (она уже есть для учеников, можно переиспользовать). Если без карты — никакого бэкенда платежей на trial, только тег в `tutor_subscriptions.status='trial_active'`.

### 2. **Что делать с уже зарегистрированными tutor-аккаунтами без подписки?**
   У тебя уже есть репетиторы в системе, которые пользуются продуктом бесплатно (пилот). Им trial задним числом? Им ничего? Им письмо «у вас осталось N дней trial»? Я бы пошёл по варианту: всех существующих → migrate в `status='paid_grandfathered'` (бесплатный legacy), новых → честный trial. Иначе сломаешь отношения с пилотными.

### 3. **Какая длительность trial — точно 7 дней?**
   Я рекомендую 7 (стандарт RU). Но возможно ты захочешь:
   - 14 дней — если AI «раскрывается» дольше (например, репетитор должен накопить минимум 5–10 ДЗ, чтобы понять value)
   - 3 дня — если хочешь агрессивную воронку и не боишься потерять часть signup-ов
   Решение зависит от того, как быстро репетитор видит value.

### 4. **На trial — лимит учеников или нет?**
   CloudText/moyklass дают full. Я рекомендую **до 20 учеников на trial** (как тариф AI-про). Если делать unlimited, то крупные онлайн-школы могут зарегистрировать trial-аккаунт и не платить. Лимит 20 = honest для индивидуальных репетиторов, не позволяет abuse.

### 5. **Telegram-канал — что в нём сейчас?**
   Я предположил, что Welcome-сообщение бота нужно настроить — но нужно знать, кто владелец. Канал @sokrat_rep — это Егора? Pinned posts уже есть? Я могу набросать копию для welcome-серии и для pinned posts, но нужны: статус канала + кто будет писать контент.

### 6. **Куда вести «Канал Егора →» — на канал или сразу в бот?**
   Сейчас линк ведёт на t.me/sokrat_rep (канал). Альтернатива — t.me/sokrat_rep_bot (бот). Бот → welcome-серия → конверсия. Канал → пассивное чтение. Я бы вёл на бот: первый contact-point должен быть интерактивным.

### 7. **Подписка после первого месяца — auto-renewal?**
   200 ₽ первый месяц — это discount. Но что после? У тебя должна быть card привязана, чтобы списывать 1000/2000/3000 со 2-го месяца. Если репетитор привязывает карту в момент оплаты 200 ₽, то да. Это нормальная практика — главное явно сказать «следующий месяц 1000 ₽, отмена в клик».

### 8. **Срочность — когда нужно в прод?**
   Если до конца мая 2026 — успеем сделать P0 (Hero/Pricing copy + signup-страница без backend trial-логики, фейковый bypass). Полный flow с cron-напоминалками и DB-таблицей — июнь. Хочешь приоритизировать что-то одно?

## Anti-patterns — чего НЕ делать

1. **Не делать trial с обязательной картой и auto-charge.** Это российский анти-паттерн. RU-юзеры распознают и закрывают вкладку. Особенно для нового бренда.
2. **Не делать «trial с лимитами»** (3 ученика, 5 ДЗ, 1 предмет). Лимиты на trial = у юзера нет шанса понять value. CloudText/moyklass дают full — ты должен тоже.
3. **Не делать hard lockout после trial.** Базовая платформа должна остаться. Иначе репетитор уносит данные на коленке и больше не вернётся.
4. **Не показывать «*потом 1000 ₽/мес»** мелким шрифтом. RU-юзеры это считывают как обман. Карточка «AI-старт» в pricing должна показывать всю лестницу 0 → 200 → 1000 явно.
5. **Не убирать «Отмена в один клик».** Это уже на лендинге, не теряй фразу — она снимает 50% сомнений.
6. **Не делать 5+ CTA «Попробовать» на одной странице.** Сейчас на лендинге их 6+. Ограничь до 3 (Hero / Pricing / FinalCTA), все ведут на одну signup-страницу. Иначе аналитика смешается.
7. **Не делать «Telegram канал» visualно как зелёную outline-кнопку.** Делай TG-blue. Юзер должен сразу понять, что это другой канал, не дубль primary CTA.

## Кратко: ROI для бизнеса

Если эта механика работает как у CloudText (плюс-минус) — вот что ожидать:

- **Signup conversion** на лендинге: +50% (с card-up-front до без-card-trial)
- **Trial-to-paid conversion**: 15–25% (RU EdTech baseline)
- **First-month retention**: 70–80% (хорошие 200 ₽ anchor-скидки → юзер не сильно жалеет)
- **Second-month retention**: 50–60% — это критично проверить, потому что 1000 ₽ vs 200 ₽ это серьёзная разница в восприятии

Главный риск — **trial-juicing**: репетиторы регистрируются с разных email, чтобы получить 7 дней × N. Митигировать можно жёсткой линковкой по телефону или TG-attribute, но это снова поднимает friction. Стандартный приём — линкуем по `auth.users.email` + IP fingerprint в `trial_signups` audit log, и блокируем повторные trial-ы для одного fingerprint.
