## Что важно по тону

- **Не извиняться** за переход на платный тариф. Ты уже подарил месяц free — это адекватный пилот, дальше нормально брать деньги. Извинительный тон сигналит «я не уверен, что мой продукт стоит денег» → юзер тоже не будет уверен.
- **Подсветить конкретный фактик**, как они пользовались. Это показывает, что ты следил за ними как за человеком, а не за email-адресом.
- **Назвать дату cutoff** прямо. «Скоро» = размыто. «До 18 мая включительно» = решение.
- **Уважать опт-аут.** Соц-давление работает плохо в long-run. «Если решишь не платить — без проблем» снимает напряжение и оставляет дверь открытой через месяц.
- **Личная подпись.** Не «Команда Сократа AI», а «Володя» / «Егор». Чем меньше дистанция — тем выше conversion.

## Технически — что делать с легаси-аккаунтами в БД

В P0 — **ничего**. AI продолжает работать у всех (нет gating). Письма работают как маркетинговый туф, не как technical cutoff.

В P1 — добавляешь поле `profiles.paid_grandfathered_until` (default `null`). Для legacy ставишь `'2026-05-31T23:59:59Z'` (или твой реальный cutoff). AI-gating-проверка станет:

```ts
const sub = await db.from('tutor_subscriptions').select('ai_features_active').eq('user_id', tutor_id).single();
const profile = await db.from('profiles').select('paid_grandfathered_until').eq('user_id', tutor_id).single();
const grandfathered = profile.paid_grandfathered_until && new Date(profile.paid_grandfathered_until) > new Date();

if (!sub?.ai_features_active && !grandfathered) {
  return { error: 'ai_disabled' };
}
```

Это даёт чистую миграцию: до даты — AI работает, после — выключается, репетитор видит модалку «активируй подписку» (та же что для post-trial).
