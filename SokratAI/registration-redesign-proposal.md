*Дата: 13 апреля 2026*

## 1. Что делают конкуренты

### ProgressMe (платформа для репетиторов по языкам)

ProgressMe решает ту же задачу: репетитор добавляет своих учеников на платформу.

**Как репетитор добавляет ученика:**

1. Переходит в раздел «Ученики» → «Добавить ученика»
2. Три способа:
   - **Вручную** (основной): вводит email + имя ученика → нажимает «Создать» → система генерирует пароль → данные приходят на email ученика И дублируются репетитору
   - **По ссылке**: копирует invite-ссылку → отправляет ученику → ученик сам регистрируется
   - **Из существующих**: если ученик уже на платформе — просто добавляет в свой класс

**Что получает ученик:** письмо с логином (email) и сгенерированным паролем. Репетитор тоже видит эти данные и может передать их лично (как на твоём скриншоте — кнопка «Данные для входа» + «Скопировать для отправки»).

**Что хорошо:**
- Репетитор контролирует процесс на 100% — ученику не нужно ничего делать кроме «войти»
- Кнопка «Скопировать для отправки» — репетитор вставляет готовый текст в WhatsApp/Telegram
- Данные доступны в профиле ученика — можно отправить повторно

**Что плохо:**
- Ученик получает чужой пароль (безопасность)
- Нет Google OAuth — только email+password

### Khan Academy (учителя + ученики + родители)

Khan Academy решает масштабнее: школьные классы по 30 человек.

**Как учитель добавляет учеников:**

1. Создаёт «класс» в дашборде
2. Получает **class code** (короткий буквенно-цифровой код, например `ABC123`)
3. Ученик заходит на `khanacademy.org/join` → вводит код → создаёт аккаунт (или входит в существующий) → автоматически привязывается к учителю
4. Альтернатива: импорт из Google Classroom (массовый)

**Что хорошо:**
- Минимум действий от учителя (дал код — и забыл)
- Ученик сам контролирует свой аккаунт (свой пароль, Google login)
- Масштабируется на класс из 30 человек

**Что плохо:**
- Ученик может забыть/потерять код
- Требует от ученика инициативы (а 16-летние не всегда мотивированы)
- Для репетитора 1:1 — оверхед (создавать «класс» ради одного ученика)

## 2. Сравнение подходов для Сократа

| Критерий | ProgressMe (репетитор создаёт) | Khan Academy (ученик сам) |
|---|---|---|
| **Скорость для репетитора** | Быстро: ввёл email → готово | Быстро: скопировал код → отправил |
| **Скорость для ученика** | Мгновенно: получил данные → вошёл | 2-3 мин: перешёл по ссылке → регистрация → пароль |
| **Контроль репетитора** | Полный: видит данные, может отправить повторно | Средний: не знает пароль ученика |
| **Безопасность** | Ниже: репетитор знает пароль | Выше: ученик сам задаёт пароль |
| **Ученик 16 лет** | Идеально: минимум самостоятельных действий | Хуже: может забить на регистрацию |
| **Масштаб (5-15 учеников)** | Нужно добавлять каждого вручную | Один код/ссылка на всех |
| **Telegram/WhatsApp передача** | Отлично: «Скопировать для отправки» | Нормально: отправить ссылку |
| **Передача лично на уроке** | Отлично: показать данные с экрана | Хуже: ученик должен при тебе регистрироваться |
| **Разработка** | Проще: ~2-3 дня | Текущий invite flow уже есть |

### Рекомендация: гибрид (ProgressMe primary + invite link secondary)

**Почему именно так для Сократа:**

1. **Твои репетиторы работают 1:1 или в мини-группах (3-5 чел.)** — им не нужен «class code» на 30 человек. Им нужно быстро добавить конкретного Ваню.

2. **Каналы коммуникации — Telegram и личные встречи.** Формат ProgressMe идеален: репетитор нажимает «Скопировать данные для отправки» → вставляет в Telegram → ученик входит за 10 секунд. Или показывает на уроке с экрана.

3. **Ученики 16-18 лет.** Чем меньше от них требуется — тем лучше. «Вот тебе логин и пароль, зайди» работает надёжнее чем «перейди по ссылке, зарегистрируйся, придумай пароль».

4. **Invite-ссылка уже есть в коде** — оставляем как secondary path для случаев «у ученика уже есть аккаунт» или «репетитор не знает email».

## 3. Текущее состояние регистрации в Сократе

### Что уже работает:

**Для репетитора:**
- Вход по email+пароль (`/tutor/login`)
- Вход через Telegram (но сейчас сломан из-за блокировки)
- Роль присваивается через edge function `assign-tutor-role`

**Для добавления ученика (AddStudentDialog):**
- Вкладка «По ссылке»: QR-код + invite URL + копирование ссылки
- Вкладка «Вручную»: имя + email/telegram + цель + ставка + доп. поля
- Backend: `tutor-manual-add-student` создаёт auth user с рандомным паролем + профиль + связку

**Что НЕ работает / отсутствует:**
- Нет страницы регистрации для репетиторов (только вход)
- Нет Google OAuth
- При ручном добавлении ученика — ученик получает `@temp.sokratai.ru` email и рандомный UUID-пароль, которым невозможно войти
- Нет механизма «показать данные для входа ученика» как в ProgressMe
- Нет кнопки «Скопировать для отправки»
- Telegram auth сломан (блокировка РФ)

## 4. Предложение: Парето-решение для регистрации

### 4.1 Регистрация репетитора (новый flow)

**Текущая проблема:** нет самостоятельной регистрации, роль присваивается вручную.

**Предлагаемый flow:**

```
sokratai.ru (лендинг) → «Я репетитор» → /register-tutor
```

**Страница /register-tutor (3 шага, один экран):**

**Шаг 1 — Вход/регистрация (выбор метода):**
- **Google OAuth** (primary CTA) — один клик, без пароля
- **Email + пароль** — классика, fallback
- Telegram — secondary, мелким текстом «Войти через Telegram» (для тех у кого есть VPN)

**Шаг 2 — Мини-анкета (5 полей, один экран):**
- Имя (предзаполнено из Google)
- Предмет: [Физика] [Математика] [Другой] — pills, один клик
- Формат работы: [Частные уроки] [Школа/курсы] — pills
- Откуда узнали: [Рекомендация] [Поиск] [Соцсети] — pills
- Телефон/Telegram (опционально) — для связи

**Шаг 3 — Дашборд с подсказкой:**
- Перенаправление на `/tutor/dashboard`
- Пустое состояние: «Добавьте первого ученика» с большой зелёной кнопкой
- Короткий тултип: «Вы можете добавить ученика за 30 секунд»

**Что под капотом:**
- Google OAuth: `supabase.auth.signInWithOAuth({ provider: 'google' })`
- После auth: create profile + assign tutor role автоматически (если пришёл через `/register-tutor`)
- Мини-анкета: upsert в `profiles` + `tutors` таблицу

**Объём работы: ~2-3 дня**

### 4.2 Добавление ученика репетитором (редизайн AddStudentDialog)

**Новый flow — вдохновлён ProgressMe, адаптирован под Сократ:**

**Вкладка 1: «Быстрое добавление» (primary, как ProgressMe)**

Минимальная форма:
- **Имя ученика** (обязательно)
- **Email ученика** (обязательно) — если есть
- **Telegram ученика** (альтернатива email) — @username

→ Кнопка «Добавить»

**Что происходит после нажатия:**

1. Backend создаёт аккаунт ученика (или находит существующий)
2. Генерирует **читаемый пароль** (4-6 цифр, как ProgressMe: `1529`, `8472`)
3. Сохраняет пароль hash в auth, plain-text нигде не хранится
4. **Показывает модалку «Данные для входа»** (как на скриншоте ProgressMe):

```
┌──────────────────────────────────┐
│     Данные для входа ученика     │
│                                  │
│  Страница входа:                 │
│  sokratai.ru/login               │
│                                  │
│  Почта:                          │
│  ivan@gmail.com                  │
│                                  │
│  Пароль:                         │
│  4827                            │
│                                  │
│  [Скопировать для отправки]      │
│                                  │
│  [Закрыть]    [Отпр. на почту]   │
└──────────────────────────────────┘
```

**«Скопировать для отправки»** формирует текст:

```
Страница входа в платформу:
https://sokratai.ru/login

Почта от аккаунта:
ivan@gmail.com

Пароль для входа:
4827
```

Репетитор вставляет это в Telegram ученику — ровно как на скриншоте 2.

**«Отпр. на почту»** — автоматическое письмо ученику с данными (опционально).

**Вкладка 2: «По ссылке» (secondary, текущий flow)**

Оставляем как есть: QR-код + invite URL. Для случаев когда репетитор не знает email ученика.

**Что под капотом:**
- Генерация пароля: `Math.floor(1000 + Math.random() * 9000).toString()` (4 цифры) — как ProgressMe
- Пароль показывается ОДИН раз при создании, потом не доступен (кроме кнопки «Показать данные» в профиле ученика — генерирует новый)
- Если ученик уже существует — показать «Ученик уже зарегистрирован, привязан к вашему кабинету»
- Email-письмо: через существующий email queue (`process-email-queue`)

**Объём работы: ~3-4 дня**

### 4.3 Первый вход ученика

**Flow (максимально простой):**

```
Ученик получает данные от репетитора (Telegram/лично)
  → Заходит на sokratai.ru/login
  → Вводит email + пароль (4 цифры)
  → Попадает на /student/homework (список ДЗ от репетитора)
```

**Дополнительно:**
- При первом входе — краткий welcome-экран: «Привет, [Имя]! Тебя добавил репетитор [Имя репетитора]» + «Начать»
- Google OAuth на странице входа — если ученик хочет войти через Google (email должен совпадать)
- Смена пароля: в настройках профиля (не обязательно, но хорошо бы)

**Объём работы: ~1-2 дня**

### 4.4 Вход через Google OAuth (общий для всех)

**Зачем:** Google OAuth — это один клик без пароля. Работает и для репетиторов, и для учеников.

**Flow:**
1. Кнопка «Войти через Google» на `/login` и `/register-tutor`
2. Supabase OAuth → Google consent screen → callback
3. По email определяем: есть ли аккаунт? Какая роль?
4. Redirect на нужный дашборд

**Важно:**
- Если ученик создан вручную с email `ivan@gmail.com`, а потом Ivan входит через Google с тем же email — аккаунты связываются автоматически (Supabase linking)
- Это означает: репетитор добавил ученика → ученик нажал «Войти через Google» → всё работает, пароль не нужен

**Объём работы: ~1-2 дня** (Supabase Google provider + UI кнопки)

| Приоритет | Задача | Усилие | Ценность |

## 6. User Stories (конкретные сценарии)

### Сценарий A: Репетитор добавляет ученика перед уроком

```
1. Репетитор в /tutor/dashboard → «Добавить ученика»
2. Вводит: Иван Петров, ivan@gmail.com
3. Нажимает «Добавить»
4. Видит модалку с данными: email + пароль 4827
5. Нажимает «Скопировать для отправки»
6. Вставляет в Telegram ученику
7. Готово. 30 секунд.
```

### Сценарий B: Репетитор добавляет ученика лично на уроке

```
1. На уроке: «Давай зарегистрирую тебя в Сократе»
2. Открывает AddStudentDialog → вводит имя + email
3. Показывает ученику с экрана: «Вот, зайди на sokratai.ru/login»
4. Ученик тут же на телефоне входит с показанными данными

```

### Сценарий C: Ученик входит впервые

```
1. Получает от репетитора в Telegram:
   "Страница входа: sokratai.ru/login
    Почта: ivan@gmail.com
    Пароль: 4827"

```

### Сценарий D: Новый репетитор регистрируется сам

```
1. Заходит на sokratai.ru → «Я репетитор» → /register-tutor
2. Нажимает «Войти через Google» → авторизуется

4. Попадает на пустой дашборд: «Добавьте первого ученика»
5. Добавляет ученика по сценарию A
```

## 7. Безопасность

**Пароль 4 цифры — это безопасно?**

Для контекста репетитора 1:1 — достаточно:
- Rate limiting на login (5 попыток → блокировка на 15 мин)
- Аккаунт ученика не содержит платёжных данных
- Ученик может сменить пароль на сложный в настройках
- Google OAuth позволяет вообще не использовать пароль

ProgressMe использует 4-значные пароли годами и это работает для их сегмента.

## 8. Статус активации и каналы связи в кабинете репетитора

### 8.1 Статус «Вошёл / Не входил» на карточке ученика

Репетитор должен видеть в списке учеников, кто уже активировал аккаунт (хотя бы раз залогинился), а кто нет.

**Реализация (минимальная):**

На карточке ученика в `/tutor/students` — бейдж статуса:
- **«Не входил»** — серый бейдж + иконка AlertCircle. Показывается если `profiles.last_sign_in_at IS NULL`
- **«Активирован»** — зелёный бейдж + иконка CheckCircle. Показывается если `last_sign_in_at IS NOT NULL`
- Дополнительно: дата последнего входа мелким текстом (`Последний вход: 12 апр`)

**Откуда брать данные:**
- `auth.users.last_sign_in_at` — Supabase хранит это автоматически
- Для tutor-запроса: edge function `handleGetStudents` должен возвращать `last_sign_in_at` (через `auth.admin.getUserById` или добавить триггер, который копирует в `profiles`)
- Рекомендация: добавить `last_sign_in_at` в `profiles` таблицу и обновлять при каждом входе — проще для RLS и запросов

**Действие на «Не входил»:**
- Кнопка «Отправить данные повторно» — открывает модалку «Данные для входа» (та же что при создании, но с кнопкой «Сбросить пароль» вместо показа старого)

**Объём работы: ~1 день**

### 8.2 Каналы связи (Telegram + Email) на карточке ученика

Репетитор видит, какие каналы привязаны у ученика:

```
┌─────────────────────────────────────────────┐
│  Иван Петров                    Активирован │
│  ЕГЭ Физика · 10 класс                     │
│                                             │
│  📧 ivan@gmail.com              ✅          │
│  ✈️ @ivan_petrov                ✅          │
│                                             │
│  [Данные для входа]  [Напомнить]            │
└─────────────────────────────────────────────┘
```

**Состояния каналов:**
- **Email**: ✅ (есть реальный email, не `@temp.sokratai.ru`) или ❌ (нет / temp email)
- **Telegram**: ✅ (есть `telegram_user_id` в profiles — ученик привязал бота) или ⚠️ (есть `telegram_username` но нет `telegram_user_id` — username указан, но бот не привязан) или ❌ (ничего нет)

**Откуда данные (уже есть в БД):**
- `profiles.telegram_username` — заполняется при ручном добавлении
- `profiles.telegram_user_id` — заполняется когда ученик пишет боту `/start`
- `auth.users.email` — email аккаунта
- Для определения temp email: проверка `email.endsWith('@temp.sokratai.ru')`

**Ценность для репетитора:**
- Видит сразу: «Иван не привязал Telegram → уведомления о ДЗ не дойдут → нужно попросить привязать»
- Видит: «У Маши только temp email → нужно попросить настоящий email»
- Каналы связи = каналы доставки уведомлений о ДЗ (уже используются в cascade delivery)

**Объём работы: ~1 день** (данные уже есть, нужен только UI)

## 9. Групповые занятия — добавление учеников в группу

### Текущее состояние

В AddStudentDialog уже есть мини-группы: toggle «Добавить в группу» → выбор существующей или создание новой. Это работает для добавления одного ученика в группу.

### Что нужно для сценария «создаю группу из 5 человек»

**Два пути:**

**Путь A: Последовательное добавление (текущий + улучшение)**

```
1. Репетитор нажимает «Добавить ученика»
2. Включает toggle «Добавить в группу» → создаёт группу «ЕГЭ Физика, вт/чт»
3. Вводит данные ученика 1 → «Добавить» → видит данные для входа → «Скопировать»
4. Диалог НЕ закрывается — очищаются поля, группа остаётся выбранной
5. Вводит данные ученика 2 → «Добавить» → и т.д.
6. После последнего → «Закрыть»
```

Изменение: после успешного добавления — **не закрывать диалог**, а очистить поля и показать toast. Кнопка «Закрыть» (вместо auto-close).

**Путь B: Batch-добавление (новый UI)**

```
1. Репетитор нажимает «Добавить группу»
2. Вводит название группы
3. Добавляет строки учеников:
   [Имя] [Email] [+]
   [Имя] [Email] [+]
   [Имя] [Email] [+]
4. «Создать группу и добавить учеников»
5. Видит сводку: 5 учеников добавлены, данные для входа
```

**Рекомендация:** Путь A (минимальное изменение) — 0.5 дня. Путь B — 2-3 дня и не критичен для 3-5 учеников.

## 11. Решённые вопросы (финал)

| # | Вопрос | Решение |
|---|---|---|
| 1 | Формат пароля | **4 цифры** (как ProgressMe: `1529`). Легко диктовать на уроке, копировать в Telegram |

| 3 | Email confirmation | **НЕТ**. Репетитор верифицировал ученика лично. Лишний шаг = потеря конверсии |
| 4 | Лендинг sokratai.ru | **Уже готов.** Dropdown «Войти» → «Я ученик» / «Я репетитор» есть и на десктопе, и на мобиле. Entry point для `/register-tutor` = пункт «Я репетитор» в этом dropdown |
| 5 | `last_sign_in_at` | **Триггер** на `auth.users` → копирует в `profiles.last_sign_in_at` при каждом входе. Проще для RLS и tutor-запросов |
